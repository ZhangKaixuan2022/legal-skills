# 庭审与庭后工作Skill

## 功能概述

本Skill定位在民事一审诉讼体系的第5环节（承接"庭前准备"Skill），系统化支持庭审全流程工作，包括开庭前检查提醒、庭审纪要记录、庭审复盘分析、庭后代理词撰写、补充材料管理、与当事人和法官沟通等。

### 核心功能

1. **开庭前最终检查**：生成提醒清单，提示律师逐项检查开庭材料、着装仪态、庭审注意事项
2. **庭审纪要记录**：将律师庭后提供的庭审要点整理为结构化《庭审纪要》
3. **庭审复盘分析**：基于庭审纪要，按五个维度生成《案件庭审复盘报告》
4. **庭后代理词撰写**：基于庭审实际情况，独立撰写回应性代理词及思维导图
5. **庭后补充材料管理**：生成补充材料清单，提示提交期限和逾期风险
6. **与当事人沟通**：生成向当事人汇报的要点清单
7. **与法官沟通记录**：提供沟通记录模板和技巧提示
8. **多次开庭支持**：自动编号、关联前次记录、累积复盘

### 模块2：庭审纪要记录

**工作流程**：
1. 律师庭后将庭审要点（文字/语音）发送给AI
2. AI根据模板整理为结构化《庭审纪要》
3. 输出Word文档

#### 2.1 信息收集

**请律师提供以下信息**：

**基本信息**：案件名称、案号、开庭时间、法庭、审判人员、书记员、各方当事人及代理人

**庭审要点**：
- 原告诉讼请求及事实理由
- 被告答辩意见
- 举证质证情况（证据名称、证明目的、质证意见）
- 法庭归纳的争议焦点
- 法庭询问及各方回答
- 法庭辩论要点
- 最后陈述内容
- 法庭交待事项（补充材料、下次开庭等）

#### 2.2 庭审纪要结构

**模板路径**：`templates/庭审纪要模板.md`

**输出格式**：
```
庭审纪要

一、基本信息
二、法庭调查阶段（诉讼请求、答辩、举证质证、争议焦点、法庭询问）
三、法庭辩论阶段
四、最后陈述
五、法庭交待事项
六、庭审笔录签署情况
七、待办事项
八、备注
```

##### 法院释明的识别与应对

**释明的两种类型**：
1. **法律关系性质释明**：法院认为当事人主张的法律关系性质与认定不一致时，告知可变更诉讼请求
2. **举证责任释明**：告知举证责任及举证不能的后果

**庭审应对策略**：
- 法院释明变更法律关系性质时：评估是否需要当庭变更诉讼请求
- 法院释明举证责任时：记录内容，评估是否需要补充举证
- 法院释明不当时：在庭审笔录中如实记录，保留异议权利

> **法律依据**：《民事证据规定》第53条、《民诉解释》第226条
> 📖 完整释明权应对策略 → [references/释明权应对与逾期举证策略.md](释明权应对与逾期举证策略.md)

##### 当庭质证实战要点

核心技巧：质证方式选择（一证一质/分组质证）、抓薄弱环节突破、关键证据书面质证
📋 详细说明参见 [references/庭审质证实战技巧.md](庭审质证实战技巧.md)

### 模块3：庭审复盘

**工作流程**：
1. 读取庭审纪要
2. 按五个维度进行分析
3. 生成《案件庭审复盘报告》

#### 3.1 复盘维度

**维度1：庭审表现评价**
- 己方在各环节的表现分析
- 优点、不足、改进建议

**维度2：举证质证效果**
- 己方证据被采信的可能性评估
- 对方证据的质证效果评估
- 风险点和补强建议

**维度3：法庭辩论效果**
- 核心观点是否充分表达
- 是否有效回应对方
- 辩论策略调整建议

**维度4：法官态度与倾向判断**
- 根据法官发问分析倾向性
- 根据归纳焦点的方式推断
- 整体倾向性判断

**维度5：策略执行与偏差**
- 实际庭审与预设策略的偏差
- 偏差原因分析
- 需调整的方向

#### 3.2 复盘报告结构

**模板路径**：`templates/庭审复盘报告模板.md`

**输出格式**：
```
案件庭审复盘报告

一、案件基本信息
二、庭后需完善的工作（补充意见、核实问题、补充材料）
三、庭审效果分析（五个维度）
四、多次开庭对比分析（如适用）
五、与当事人沟通要点
六、总结（胜诉概率、风险点、后续建议、时间节点）
```

### 模块4：庭后代理词

**重要原则**：庭后代理词完全独立于庭前代理词，基于实际庭审情况重新撰写。

#### 4.1 与庭前代理词的区别

| 对比项 | 庭前代理词 | 庭后代理词 |
|-------|-----------|-----------|
| 性质 | 预判性 | 回应性 |
| 争议焦点 | 基于预判 | 基于法庭实际归纳 |
| 对方观点 | 基于预估 | 基于实际庭审 |
| 法官关注 | 基于经验 | 基于实际发问 |
| 新证据 | 未涉及 | 需回应 |

#### 4.2 代理词撰写要点

1. **针对庭审实际归纳的争议焦点逐一回应**：每个焦点按"事实依据→法律依据→分析论证→结论"展开
2. **针对对方庭审中提出的新观点/新证据进行反驳**
3. **回应法官关注的重点问题**
4. **运用三段论逻辑结构**：大前提（法律规范）→ 小前提（案件事实）→ 结论

#### 4.3 代理词结构

**模板路径**：`templates/庭后代理词模板.md`

**输出格式**：
```
【案件名称】一审代理词

首部（致法院、案号、代理律师）
前言（委托情况、开庭情况）
正文（逐个争议焦点展开）
结语（总结核心观点、提出请求）
附件（证据目录、法规汇编、案例报告、思维导图）
```

**提交提醒**：庭后代理词一般在庭审后3-5个工作日内提交，无论法官是否要求均应主动提交

## 第四部分：交互规范

### 4.1 智能追问机制

**追问原则**：每次只追问一个关键问题，语言简洁专业，说明追问原因，等待用户回复后再继续

| 触发情形 | 追问示例 |
|---------|---------|
| 需要确认开庭次数 | "这是第几次开庭？" |
| 需要庭审要点 | "请提供庭审要点，包括争议焦点、举证质证情况、法官发问等" |
| 需要确认诉讼地位 | "请确认己方是原告还是被告？" |
| 需要前次记录 | "请提供前次庭审纪要的路径，以便进行对比分析" |

### 4.2 沟通风格

- 专业但不生硬
- 简洁明了
- 问题导向
- 选项式提问便于回复

## Wiki 知识库查阅规则

当处理的案件涉及具体罪名或法律领域时，必须查阅 Wiki 知识库：

1. **检查知识库**：检查 `~/wiki/wiki/` 下是否有对应知识页（刑事：`books/criminal/{罪名}.md`，民事：`books/civil/{领域}/{具体问题}.md`）
2. **读取知识卡片**：若有对应页面，读取页面顶部 `<!-- CARD_START -->` 到 `<!-- CARD_END -->` 之间的知识卡片（约800字浓缩摘要）
3. **按需深入**：卡片信息足够则直接引用；需要深入了解具体辩护策略、案例细节、司法解释原文时，读取全文对应板块
4. **融入输出**：将 Wiki 知识自然融入当前工作输出，引用时标注来源页面路径
5. **案例知识页**：同时检查 `~/wiki/wiki/cases/` 下是否有相关判例知识页可供参考
