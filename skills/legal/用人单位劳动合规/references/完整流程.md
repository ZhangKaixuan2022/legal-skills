## 旧规则废止（强制）

- 旧文中直接写死的客户目录、阶段目录、旧式台账写入、旧本地读取协议均不作为执行规则。
- 事项路径、当前事项、系统记录、业务文件区和复盘台账统一以法律工作总控 `matter-workspace-protocol.md` 为准。
- 不得静默写入复盘台账；确需更新时，先确认属于复盘台账更新并向用户说明。


## 文件与法规核验

处理劳动合同、员工手册、规章制度、薪酬制度、考勤制度、绩效制度、解除/处罚材料、工资表、考勤表等文件时，必须：
1. 按总控文件读取规范完整读取文件。
2. 对姓名、公司名称、岗位、日期、金额、工资基数、工时、发薪日、制度版本、生效日期、签收日期做读取复查。
3. 引用中国大陆劳动法律法规、地方规则、最低工资、社平工资、特殊工时审批规则时，必须联网核验名称、条文、时效和适用地区。
4. 未读取、未核验或仅来自用户口述的信息，不得包装成确定事实。

## 适用边界

优先处理以下公司端非诉或争议前任务：
- 公司遇到员工投诉、解除、调岗、降薪、处罚、待岗、停职、绩效处理前的风险预判。
- 员工手册、规章制度、劳动合同、薪酬制度、考勤制度、绩效制度、奖惩制度的审查、修改和生效流程设计。
- 工资工时问答，包括加班费、发薪日、特殊工时、年休假工资、工资结构、绩效工资、扣款规则。
- 公司内部劳动政策文件起草。

已经进入仲裁、诉讼、仲裁答辩、裁决后衔接的，转回「劳动争议诉讼」和「劳动争议仲裁程序管理」。需要经济补偿、加班费、双倍工资等金额测算的，调用「劳动关系认定与经济补偿计算」。

## 事项路径与归档

本 Skill 的事项类型为 `劳动合规`，不归入 `诉讼/民事`。只有已经进入仲裁、诉讼、劳动监察、律师函、仲裁答辩或裁决后衔接的劳动争议，才转入诉讼路径。

### 建档规则

1. 任务开始前读取 `_系统记录/当前事项.md`。
2. 劳动合规按企业客户建立长期档案，一个公司一个总文件夹；不得把每次咨询都建成独立顶层事项。
3. 新公司首次处理时，创建 `<企业客户>` 公司档案，并写入 `公司基本信息.md`。
4. 员工手册修订、工资工时问答、制度更新、专项风险评估等，作为公司档案下的 `06-专项合规事项/<YYYYMMDD-事项简称>/` 和系统记录 `专项事项/<YYYYMMDD-事项简称>/` 管理。
5. 收到客户材料，原文件放业务文件区；读取复查、OCR、来源验证和缺口归档放系统记录区。
6. 正式交付给客户的报告、制度草案、修订稿、流程清单放业务文件区；内部风险推演、让步建议和工作笔记只放系统记录区。

### 劳动争议衔接

同一企业客户发生劳动争议、劳动仲裁或诉讼时，诉讼事项另按 `诉讼/民事/<企业客户>/<案件简称或案号>/` 建档。本 Skill 维护的公司劳动合规档案可作为背景资料供劳动争议 Skill 读取：
- 优先提供公司基本信息、公司材料清单、制度版本台账、变更记录、生效流程记录。
- 按争议类型提供相关制度版本和专项事项记录。
- 输出时必须标注来源为“公司劳动合规档案”，不得把公司档案记录直接当作争议事实而不复核。

## 模块一：公司端争议事实收集与风险预判

适用于：准备辞退员工、员工投诉公司、调岗降薪、违纪处理、绩效淘汰、待岗停职、仲裁风险评估。

### 必问事实

| 类别 | 核心问题 |
|---|---|
| 劳动者基本信息 | 姓名、岗位、入职时间、合同期限、工作地点、工资结构、社保缴纳 |
| 特殊状态 | 是否孕期/产期/哺乳期、工伤、医疗期、残疾、距退休不足5年且本单位满15年 |
| 近期敏感行为 | 近期是否投诉、举报、申请劳动监察、主张加班费/社保/工资、拒绝调岗降薪 |
| 公司文件 | 劳动合同、员工手册、规章制度、薪酬制度、考勤制度、绩效制度、奖惩制度 |
| 拟处理动作 | 解除、调岗、降薪、处罚、待岗、停职、绩效改进、协商解除 |
| 证据链 | 事实材料、调查记录、员工申辩、制度依据、工会通知、送达记录 |

### 风险旗标

出现以下情形时，风险等级原则上不得低于“中”；多个叠加时标为“高”：
- 特殊保护状态未查明。
- 员工近期有投诉、举报、劳动监察、仲裁威胁或权利主张。
- 解除/处罚依据来自未生效或未公示的规章制度。
- 缺少事实调查、员工申辩、工会通知或送达记录。
- 同类行为处理不一致，存在选择性执法风险。
- 调岗、降薪、绩效淘汰实质降低劳动条件但缺少协商一致或制度依据。

### 输出结构

```markdown
### 公司端风险预判
- 拟处理事项：
- 初步风险等级：
- 风险原因：
- 现有证据：
- 缺失材料：
- 建议动作：可继续 / 先补证 / 暂停处理 / 建议和解 / 需客户判断
- 来源：
```

## 模块二：工资工时问答

适用于：加班费、工资发放日、工资结构、特殊工时、年休假工资、绩效工资、扣款、调薪。

### 必读材料

- 劳动合同工资与工时条款。
- 员工手册、薪酬制度、考勤制度、加班审批制度。
- 工资表、工资条、银行流水、考勤记录、排班表。
- 综合计算工时制或不定时工作制审批文件。
- 工作地和用人单位所在地。

### 基础公式

```text
工作日加班费 = 月工资基数 ÷ 21.75 ÷ 8 × 加班小时 × 150%
休息日加班费 = 月工资基数 ÷ 21.75 ÷ 8 × 加班小时 × 200%
法定节假日加班费 = 月工资基数 ÷ 21.75 ÷ 8 × 加班小时 × 300%
日工资 = 月工资收入 ÷ 21.75
小时工资 = 月工资收入 ÷ 21.75 ÷ 8
```

地方规则影响加班费基数、最低工资、特殊工时审批或发薪周期时，必须按适用地区联网核验。

### 输出结构

```markdown
### 工资工时结论
- 问题：
- 结论：
- 依据：
- 计算公式：
- 代入数据：
- 风险等级：
- 缺失材料：
- 需要客户判断：
- 来源：
```

## 模块三：劳动合同/规章制度更新

适用于：修改员工手册、劳动合同、薪酬制度、考勤制度、绩效制度、奖惩制度。

### 更新流程

1. 读取现有文件和拟修改文本。
2. 标注旧条款、拟修改条款、涉及人员范围和生效时间。
3. 判断是否涉及劳动报酬、工作时间、休息休假、保险福利、劳动安全卫生、劳动纪律、定额管理等直接涉及劳动者切身利益的事项。
4. 检查民主程序、协商程序、公示告知、签收确认是否需要补足。
5. 检查与其他文件是否冲突。
6. 输出修订文本、生效流程和风险提示。

### 冲突检查矩阵

| 文件A | 文件B | 检查重点 |
|---|---|---|
| 劳动合同 | 员工手册 | 工资、岗位、地点、工时、解除条件 |
| 员工手册 | 薪酬制度 | 发薪日、绩效、奖金、扣款规则 |
| 员工手册 | 考勤制度 | 迟到、旷工、加班审批、请假规则 |
| 绩效制度 | 解除/调岗规则 | 不胜任认定、PIP、末位淘汰风险 |
| Offer/补充协议 | 劳动合同 | 工资、岗位、试用期、福利承诺 |

### 输出结构

```markdown
### 制度更新审查
- 拟修改事项：
- 是否涉及切身利益：
- 是否需要民主程序/协商程序：
- 是否需要公示告知/签收：
- 冲突条款：
- 建议修改文本：
- 生效流程：
- 风险等级：
- 来源：
```

## 模块四：公司内部劳动政策起草

适用于：考勤制度、薪酬制度、绩效制度、奖惩制度、调岗制度、离职交接制度、员工手册条款。

### 起草前必须确认

- 制度目的和希望达到的管理效果。
- 适用对象和适用范围。
- 是否降低既有劳动条件或改变既有权益。
- 是否作为处分、解除、调岗、降薪依据。
- 现有员工手册、劳动合同、薪酬制度、考勤制度是否已有相关条款。
- 是否已有工会、职代会或员工代表机制。

### 输出结构

```markdown
### 内部劳动政策草案
一、适用范围
二、具体规则
三、流程与责任人
四、员工权利与申诉机制
五、生效与解释

### 合规说明
- 合法性依据：
- 与现有文件冲突：
- 生效流程：
- 员工告知/签收建议：
- 需要客户确认事项：
- 来源：
```

## 禁止事项

- 不得未读取公司文件就判断制度合法。
- 不得未核验适用地区规则就给出特殊工时、最低工资、社平工资、加班费基数的确定结论。
- 不得把英美法项目中的 FLSA、州法、at-will、EEOC 等规则直接套入中国大陆劳动法。
- 不得只给结论不列来源、公式、缺口和客户判断事项。
