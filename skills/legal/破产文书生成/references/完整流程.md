# 破产文书生成Skill

## 功能概述

基于《破产程序法律文书样式》（最高人民法院）中的165份文书模板，生成各类破产程序文书，包括：
- **法院破产案件文书**（106份样式）：决定书、通知书、裁定书、公告、裁判文书、移送函
- **管理人破产程序工作文书**（59份）：通知书类、告知函类、报告书类

本Skill作为破产文书的统一生成工具，供其他破产Skill（破产申请与受理、管理人工作、财产调查与管理等）调用，也可由用户直接调用。

支持与复盘台账衔接，读取当前事项系统记录并按需同步文书路径。

### 步骤2：确定文书类型与收集必要信息

#### 2.1 文书类型确认

根据用户指令或调用参数，从以下分类中确定具体文书类型：

**法院文书类型**：
- 决定书：管理人指定、债权人会议主席指定、债权人委员会认可等
- 通知书：告知义务、通知债权人、通知债务人等
- 裁定书：受理/不予受理、宣告破产、批准重整计划等
- 公告：受理公告、宣告破产公告等
- 判决书：各类破产衍生诉讼裁判文书
- 移送函：执行转破产移送

**管理人文书类型**：
- 通知书：清偿债务、交付财产、解除合同/继续履行、追回财产等
- 告知函：解除保全、中止执行、恢复程序等
- 报告书：各类管理人工作报告、方案报告、审查报告等

#### 2.2 交互收集文书特定信息

根据选定文书类型，一次性列出所有需要补充的信息字段：

**通用信息**（所有文书必需）：
- 破产案号
- 受理法院名称
- 文书生成日期

**特定信息**（根据文书类型）：
- 决定书：决定事项、决定理由、法律依据
- 通知书：通知对象、通知事项、履行期限
- 裁定书：申请事项、查明事实、裁判理由、主文
- 公告：公告事项、公告期限
- 管理人文书：管理人名称、具体工作内容、相关期限

#### 3.3 生成技术方案

**使用python-docx库生成Word文档**：

```python
# 使用python-docx创建新文档，应用标准字体格式
from docx import Document
from docx.shared import Pt, Cm
from docx.enum.text import WD_ALIGN_PARAGRAPH

doc = Document()

# 设置页边距
sections = doc.sections
for section in sections:
    section.top_margin = Cm(2.8)
    section.bottom_margin = Cm(2.6)
    section.left_margin = Cm(2.6)
    section.right_margin = Cm(2.5)

# 添加标题
title = doc.add_paragraph()
title_run = title.add_run('民事裁定书')
title_run.font.name = '宋体'
title_run.font.size = Pt(22)
title_run.bold = True
title.alignment = WD_ALIGN_PARAGRAPH.CENTER

# 添加正文...

doc.save("输出路径.docx")
```

**字体规范**：
- 标题（文书名称）：宋体二号加粗，居中
- 案号：仿宋三号，右对齐
- 正文：仿宋三号，首行缩进2字符，两端对齐
- 落款：仿宋三号，右对齐

## 文书生成规范

### 格式要求

- **纸张规格**：统一使用A4纸张
- **页边距**：上2.8cm、下2.6cm、左2.6cm、右2.5cm
- **标题**：宋体二号加粗，居中
- **正文**：仿宋三号，首行缩进2字符，两端对齐
- **案号格式**：按（年份）+法院简称+破+程序类型+序号
- **编号规则**：管理人文书编号按案号+破管字第X号

### 内容规范

- **法条引用准确**：引用条文序号及内容完整准确
- **当事人信息完整**：名称、住所、法定代表人等信息完整
- **事实叙述客观**：基于案件材料客观陈述，不添加主观评价
- **法律论证严密**：逻辑清晰，法律依据充分
- **程序合规**：符合破产法及相关司法解释的程序要求

### 案号编制规则

**法院案号格式**：
```
（20XX）X破X号
（年份）+法院简称+破+程序类型+序号
```

程序类型标识：
- 清算：破
- 重整：破重
- 和解：破和
- 预重整：破预重

**管理人文书编号格式**：
```
（20XX）X破管字第X号
（年份）+法院简称+破管字第+序号
```

## 注意事项

1. **程序合规性**：破产程序有严格的法定程序要求，文书生成必须符合《企业破产法》及相关司法解释的规定
2. **信息准确性**：当事人信息、案号、日期等关键信息必须准确无误
3. **时效性注意**：注意各类期限规定（如债权申报期限、异议期限等）
4. **送达要求**：注意不同文书的送达对象和送达方式要求
5. **审批流程**：部分管理人文书需经法院批准或债权人会议决议
6. **档案管理**：生成的文书应妥善归档，便于后续查阅

## Wiki 知识库查阅规则

当处理的案件涉及具体罪名或法律领域时，必须查阅 Wiki 知识库：

1. **检查知识库**：检查 `~/wiki/wiki/` 下是否有对应知识页（刑事：`books/criminal/{罪名}.md`，民事：`books/civil/{领域}/{具体问题}.md`）
2. **读取知识卡片**：若有对应页面，读取页面顶部 `<!-- CARD_START -->` 到 `<!-- CARD_END -->` 之间的知识卡片（约800字浓缩摘要）
3. **按需深入**：卡片信息足够则直接引用；需要深入了解具体辩护策略、案例细节、司法解释原文时，读取全文对应板块
4. **融入输出**：将 Wiki 知识自然融入当前工作输出，引用时标注来源页面路径
5. **案例知识页**：同时检查 `~/wiki/wiki/cases/` 下是否有相关判例知识页可供参考
