# 刑事案件承接特殊程序与联动说明

---

## 一、认罪认罚从宽程序

### 1.1 初始判断流程

在案件初步评估中嵌入以下判断：

**判断条件**：
1. **当事人意向**：当事人是否有认罪认罚意向
2. **案件条件**：
   - 案件事实清楚
   - 证据确实、充分
   - 当事人自愿如实供述
   - 真诚悔罪

### 1.2 适用标记

如判断适用认罪认罚从宽程序：
1. 在案件档案中标记"认罪认罚类型"
2. 后续阶段Skill自动激活认罪认罚相应流程
3. 在审查起诉阶段启用量刑协商程序

### 1.3 告知事项

向委托人告知认罪认罚的法律后果、权利保障和风险：

1. **法律后果**：可以依法从宽处理（从轻、减轻或免除处罚）
2. **权利保障**：
   - 有权获得律师帮助
   - 有权知悉指控事实和证据
   - 有权了解量刑建议
   - 签署认罪认罚具结书须自愿
3. **风险提示**：
   - 认罪认罚后反悔的法律后果
   - 法院对量刑建议的采纳情况
   - 被害人谅解对量刑的影响

**法律依据**：《刑事诉讼法》第十五条、第一百七十四条至第一百七十六条

## 二、与其他Skill的联动

### 2.1 输入来源

**本Skill为刑事辩护全流程的起点，无强制前置依赖。**

可选输入来源：
- **法律咨询助手Skill**：读取咨询记录中的初步案件信息
- **复盘台账Excel**：读取已有客户基本信息

### 2.2 输出去向

| 后续Skill | 读取内容 |
|-----------|---------|
| 侦查阶段辩护Skill | 案件基本信息、委托手续、当事人信息 |
| 审查起诉阶段Skill | 案件信息、认罪认罚标记 |
| 一审阶段辩护Skill | 案件全部信息 |
| 刑事辩护总控Agent | 更新案件状态 |

### 2.3 独立启动机制

- 本Skill可独立启动，通过交互问答完整采集案件信息
- 也可被总调度Skill调用，接收已有的部分案件信息

### 2.4 数据流转

```
案件承接与委托Skill
    ↓ （输出：刑事法律服务方案、案件档案、委托文书）
侦查阶段辩护Skill
    ↓ （输出：会见记录、取保申请、辩护意见）
审查起诉阶段Skill
    ↓ （输出：阅卷笔录、辩护意见、认罪认罚文书）
一审/二审阶段Skill
```

### 2.5 服务方案与委托手续衔接

**衔接机制**：
1. 服务方案文档（步骤3）在风险告知（步骤4）和委托手续办理（步骤5）之前生成
2. 服务方案中的收费金额作为委托合同中律师费用条款的参考依据
3. 委托人基于服务方案确认委托意向后，方进入正式委托手续办理流程

**引用关系**：
- 委托合同中的律师费用金额应与服务方案中律师确认的收费金额一致
- 委托合同中的委托事项范围应与服务方案中的法律服务内容对应

### 2.6 案件状态枚举值

| 状态值 | 含义 | 对应阶段 |
|-------|------|---------|
| 已接案 | 完成委托手续 | 案件承接 |
| 侦查中 | 侦查阶段辩护进行中 | 侦查阶段 |
| 审查起诉中 | 审查起诉阶段进行中 | 审查起诉 |
| 一审中 | 一审阶段进行中 | 一审 |
| 二审中 | 二审阶段进行中 | 二审 |
| 已结案 | 案件终结 | 任何阶段 |

### 2.7 认罪认罚标记

- **存储位置**：复盘台账.xlsx "认罪认罚"字段
- **取值**："是" / "否"
- **标记时机**：案件承接阶段评估后写入
- **后续联动**：后续阶段Skill读取此字段，自动激活认罪认罚从宽相关流程

### 2.8 职责边界说明

- 本Skill（案件承接与委托）为刑事辩护体系的独立接案Skill，负责刑事案件的委托手续文书生成
- 委托合同管理 Skill适用于民事案件的委托合同管理，不适用于刑事案件
- 刑事辩护全流程的文书生成由各阶段Skill独立完成，不依赖民事诉讼Skill体系
