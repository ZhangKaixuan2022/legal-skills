# 独立起草指引

> 本文件为合同起草Skill提供独立构建合同的方法论指引。当合同库中无匹配模板时启用本指引，依据"三观四步法"方法论独立构建合同。

---

## 一、适用场景

本指引适用于以下情形：
1. 合同库中无匹配或近似模板
2. 用户需求高度定制化，现有模板无法满足
3. 新型交易模式，尚无成熟模板可参考
4. 用户明确要求从零起草

**前置检查**：启用本指引前，应确认已穷尽合同库搜索，确无可用模板。

---

## 二、合同整体框架设计

### 2.1 标准合同结构（三点一线法）

"三点一线法"是合同条款结构的推荐方法，将全部合同条款分为三个版块：

```
┌─────────────────────────────────────┐
│           合同标题（居中）            │
│           合同编号：                  │
├─────────────────────────────────────┤
│ 【合同首部】                          │
│   · 合同主体信息                      │  ← 三点之一：主体
│   · 鉴于条款（背景说明、缔约目的）    │
├─────────────────────────────────────┤
│ 【交易结构】                          │
│   第一条 定义与解释                   │
│   第二条 合同标的                     │  ← 三点之二：标的/价款
│   第三条 价款与支付                   │
│   第四条 履行方式、时间与地点         │
│   第五条 双方权利义务                 │
│   第六条 陈述与保证                   │
├─────────────────────────────────────┤
│ 【配套条款】                          │
│   第七条 违约责任                     │  ← 三点之三：违约救济
│   第八条 合同的变更、解除与终止       │
│   第九条 争议解决                     │
│   第十条 不可抗力                     │
│   第十一条 保密条款                   │
│   第十二条 通知送达                   │
│   第十三条 其他条款                   │
├─────────────────────────────────────┤
│ 【签署部分】                          │
│   签署日期、签署地点                  │
│   各方签字盖章                        │
├─────────────────────────────────────┤
│ 【合同附件】（如有）                  │
└─────────────────────────────────────┘
```

**"一线"含义**：
1. 条款按上述顺序从上到下排列
2. 交易条款适当参照交易流程顺序或按重要性排序

### 2.2 简易合同框架（5-10条款）

适用于金额小、关系简单的交易：

```
合同名称
甲方：________ 乙方：________
经双方协商一致，达成如下协议：
一、标的与数量
二、价款与支付
三、履行方式与时间
四、质量要求
五、违约责任
六、争议解决
七、其他约定
甲方（签章）：        乙方（签章）：
日期：                日期：
```

### 2.3 复杂合同框架（含附件体系）

适用于金额大、多方参与的交易：

```
正文部分：
  第一章 定义与解释
  第二章 交易标的
  第三章 价款与支付
  第四章 交割与验收
  第五章 陈述与保证
  第六章 各方权利义务
  第七章 违约责任
  第八章 合同变更与终止
  第九章 争议解决
  第十章 其他条款

附件体系：
  附件一：标的物清单
  附件二：付款计划表
  附件三：验收标准
  附件四：保密协议
  附件五：担保函（如适用）
```

## 三、条款起草方法论

### 3.1 双扣合理论

合同条款设计应遵循"双扣合"原则：

**（一）双方权利义务一环扣一环**

正确示例：甲投入A → 乙投入B → 甲投入C → 乙投入D → 合同履行完毕

错误示例：甲投入A → 甲投入B → 甲投入C → 甲投入D → 乙投入E → 合同履行完毕

**改进方法**：设置阶段性确认节点
- 甲投入A → 乙确认 → 甲投入B → 乙确认 → 甲投入C → 乙确认 → 甲投入D → 乙确认付款 → 合同履行完毕

**（二）权利义务与责任相扣合**

有义务，则应附上"如果不履行义务应承担的责任"。

示例：
> 承租方应在租赁结束一个月内将公司注册地址迁出。**如承租方逾期未迁出，则应按每月5000元向出租方赔偿损失。**

### 3.2 程序理论

合同如同程序，主要环节都有"是/否"两种可能：
- 考虑"是"（正常履行）的情况下如何处理
- 考虑"否"（违约、情况变化）的情况下如何处理

**沙盘推演**：
1. 正常交易情况：推演环环相扣的交易步骤
2. 违约/特殊情况：推演"应急预案"

示例推演：
> 条款："租赁解除或终止后，承租方应在三日内搬出并退房。"
> 推演："如果对方就是不搬怎么办？"
> 补充："如果承租方拒绝退房，则应按原租金标准的150%支付房屋占用费，直至退房之日为止。"

## 四、必备条款清单

无论何种合同类型，以下条款原则上不可缺失：

| 序号 | 条款类型 | 核心要点 | 重要程度 |
|-----|---------|---------|---------|
| 1 | **主体条款** | 当事人信息完整准确、签约资格确认 | ★★★ |
| 2 | **标的条款** | 交易对象具体描述、品名规格数量 | ★★★ |
| 3 | **价格/对价条款** | 金额、计算方式、税费承担、发票 | ★★★ |
| 4 | **履行条款** | 方式、时间、地点、验收标准 | ★★★ |
| 5 | **违约责任条款** | 违约金、损害赔偿、继续履行 | ★★★ |
| 6 | **期限与解除终止** | 合同期限、解除条件、终止后果 | ★★☆ |
| 7 | **争议解决条款** | 管辖法院或仲裁机构 | ★★☆ |
| 8 | **不可抗力条款** | 定义、通知、后果处理 | ★★☆ |
| 9 | **通知与送达条款** | 联系方式、送达地址、变更通知 | ★☆☆ |
| 10 | **签署生效条款** | 生效条件、份数、附件效力 | ★☆☆ |

## 五、风险防控条款设计

### 5.1 执行人员风险防范

| 风险类型 | 防范条款设计 |
|---------|-------------|
| 收款风险 | 明确收款账户，禁止现金或非指定账户收款 |
| 商业贿赂 | 禁止经办人员收受任何形式的回扣、佣金 |
| 越权风险 | 超额交易须经授权审批 |
| 账目风险 | 双方定期核对账目 |

### 5.2 相关方同意/优先权处理

- **共有人同意**：不动产、共同财产处分需全体共有人同意
- **配偶同意**：涉及夫妻共同财产的重大处分
- **优先购买权人通知**：承租人、股东等法定优先权人
- **用益物权人、担保物权人**：需征得权利人同意或保障其权利

### 5.3 信用额度控制

```
示例条款：
1. 乙方的赊销额度上限为人民币____万元。
2. 乙方累计欠款达到上限的80%时，甲方有权暂停发货并发出预警通知。
3. 乙方累计欠款达到上限时，甲方有权暂停供货，直至乙方将欠款降至上限的50%以下。
4. 双方应于每月__日前进行对账确认。
```

### 5.4 担保措施选择指引

| 担保方式 | 法律要点 | 注意事项 |
|---------|---------|---------|
| **保证** | 未约定保证方式时推定为一般保证（《民法典》第686条） | 需明确约定"连带保证" |
| **抵押** | 不动产抵押需登记生效，动产抵押登记对抗 | 查验权属、是否已设定其他担保 |
| **质押** | 动产质押需交付，权利质押需登记 | 确保交付或完成登记 |
| **定金** | 不超过主合同标的额20%（《民法典》第586条） | 超过部分不产生定金效力 |
| **保证金** | 性质取决于约定 | 明确返还条件和时间 |

## 六、SMART原则条款检验

每条关键条款须满足SMART原则：

| 原则 | 含义 | 检验标准 |
|-----|-----|---------|
| **S - Specific** | 具体的 | 权利义务描述是否足够具体明确？ |
| **M - Measurable** | 可衡量的 | 义务履行标准是否可量化？验收标准是否明确？ |
| **A - Attainable** | 可达到的 | 约定的义务是否现实可行？对方能否实际履行？ |
| **R - Relevant** | 有关联性 | 条款是否与交易目的相关？是否环环相扣？ |
| **T - Time-bound** | 有期限的 | 是否有明确的时间节点？起止期限是否清楚？ |

**检验示例**：

❌ 不合格条款："乙方应尽快交货。"
- 不具体（什么货？多少？）
- 不可衡量（怎样算"尽快"？）
- 无期限（没有明确时间）

✅ 合格条款："乙方应于2024年3月15日前，将本合同附件一所列货物（共100件）送达甲方指定仓库（地址：XX市XX区XX路XX号），并经甲方验收合格。"

## 七、正反两面检查

每条条款完成后执行双向检查：

### 正面检查
正常履行情形下，该条款是否清晰可执行？
- 权利义务主体是否明确？
- 履行内容是否具体？
- 履行时间、地点、方式是否清楚？

### 反面检查
如果该条款涉及的义务无法履行或被违反，合同中是否有对应的救济方案？
- 违约情形是否有定义？
- 违约责任是否有约定？
- 救济措施是否充分？

**检查模板**：

| 条款内容 | 正面检查 | 反面检查 |
|---------|---------|---------|
| 甲方应于每月15日前支付租金 | ✓ 支付主体：甲方<br>✓ 支付时间：每月15日前<br>✓ 支付对象：租金 | ? 逾期如何处理？<br>→ 需补充违约金条款 |

**补充后**：
> 甲方应于每月15日前向乙方指定账户支付当月租金。逾期支付的，乙方有权按日加收未付金额万分之五的违约金；逾期超过30日的，乙方有权解除本合同并要求甲方承担违约责任。

## 八、管道理论自检

合同如同管道，好的合同要把"水"全部送到：
- 对方找不到什么漏洞可以使我方的权利大打折扣
- 标的、质量、服务标准约定清楚
- 有相应的违约责任促使各方履行

**自检清单**：
- [ ] 标的物/服务内容是否描述完整？
- [ ] 质量/验收标准是否可操作？
- [ ] 付款条件与节点是否与履行进度挂钩？
- [ ] 核心义务是否都有对应的违约责任？
- [ ] 是否存在"权利真空"地带？
