# 快速产品问答清单

用于客户提出简短产品法律问题时快速判断风险和下一步。

## 执行步骤

1. 先做最小产品事项画像：公司做什么、产品是什么、用户是谁、拟实施动作是什么、动作发生在哪里、是否涉及数据/资质。
2. 判断是否涉及高风险触发器：收费、默认开启、个人信息、敏感个人信息、未成年人、医疗/金融/教育等强监管行业、客户 Logo/客户案例、广告宣传、自动续费、算法推荐。
3. 只基于已知信息给出初步结论；材料不足时说明缺口。
4. 重要法律依据必须联网核验后引用；未核验的只能标注为待核验方向。

## 快速追问模板

用户没有给足背景时，优先追问会直接影响结论的 2-5 个问题：

```markdown
我先确认几个会影响结论的点：
1. 公司主要做什么业务？
2. 这个产品/功能具体是什么，面向谁使用？
3. 这个动作发生在哪里：官网、App、小程序、销售材料还是其他渠道？
4. 是否涉及个人信息、收费、默认开启、客户 Logo/客户案例或强监管行业？
5. 有没有合同授权、页面截图、产品说明或文案原文？
```

## 常见问题判断方向

### 客户 Logo / 客户案例

- 是否有书面授权或合同条款支持。
- 授权范围是否覆盖官网、销售材料、公开展示、期限、地域和撤回机制。
- 是否可能构成客户推荐、背书或合作关系暗示。
- 是否涉及商标、企业名称、肖像、作品或商业秘密。
- 建议使用中性表述，如“服务客户”“客户案例”，避免暗示官方认证、独家合作或效果承诺。

### 功能默认开启

- 是否影响用户权利、费用、隐私、通知、定位、推荐、自动续费或交易结果。
- 是否需要用户主动选择、单独同意、显著提示或便捷取消。
- 是否涉及未成年人、敏感个人信息、营销推送、画像推荐、自动化决策。
- 默认开启的商业收益不能替代合规必要性判断。

### 数据、权限和推送

- 数据收集是否与功能直接相关、最小必要。
- 权限申请是否提前、过度或捆绑。
- 营销短信、邮件、App 推送是否有同意、退订和关闭入口。
- 隐私政策、弹窗、页面文案和实际行为是否一致。

### 交易、会员和自动续费

- 是否明确价格、计费周期、扣费方式、取消路径、退款规则。
- 自动续费是否显著提示，是否提供便捷取消。
- 会员权益是否与宣传一致，是否存在格式条款限制消费者权利。
