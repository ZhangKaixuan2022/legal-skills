# 产品变更与版本记录

用于产品多轮修改、上线版本、最终版确认和后续争议衔接。

## 需要记录的变更

- 新功能上线、功能下线、默认设置变更。
- 用户协议、隐私政策、会员规则、平台规则修改。
- 页面文案、权限弹窗、营销素材、客户案例修改。
- 收费方式、自动续费、退款、注销、投诉流程修改。
- 高合规行业资质、备案、审批状态变化。

## 记录字段

```markdown
## 版本记录
- 版本编号/日期：
- 变更内容：
- 影响用户：
- 影响材料：
- 法律风险变化：
- 需通知/同意事项：
- 已核验来源：
- 未核验/缺口：
- 最终确认人：
- 归档位置：
```

## 变更影响检查

每次变更后检查：

- PRD、页面、协议、隐私政策、FAQ、客服话术是否一致。
- 旧版本用户是否需要通知或重新同意。
- 是否影响既有合同、承诺、广告宣传或客户案例。
- 是否需要重新进行行业资质、广告、数据或消费者权益评估。
- 是否需要保留发布记录、用户同意记录和回滚方案。

## 最终版确认

收到客户最终版、上线版或发布版时：

1. 与上一版比对关键变更。
2. 记录最终版文件路径、确认时间、确认人和适用范围。
3. 如发现新增风险，提示用户确认是否重新审查。
4. 将最终版放入业务文件区，内部比对和来源记录放入系统记录区。
