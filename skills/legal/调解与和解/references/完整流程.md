# 调解与和解Skill

## 功能概述

本Skill定位在民事一审诉讼体系的全流程环节，可在诉前调解、立案后调解、庭中调解、庭后调解任何阶段触发，为律师提供调解与和解的决策支持、方案起草、协议审查和撤诉管理。

### 核心功能

1. **调解/和解可行性分析**：基于案件材料分析调解利弊，对比诉讼与调解的预期收益、时间成本、执行风险
2. **调解方案起草**：生成调解方案确认函，梳理可让步空间与不可让步底线
3. **调解书/和解协议审查**：提供审查要点提示和关键条款核查清单
4. **撤诉管理**：生成撤诉申请书，提示代理权限和法律后果
5. **系统记录同步**：记录调解状态、日期、结果等信息

## 第二部分：核心工作流程

### 模块1：调解/和解可行性分析

#### 1.1 自动分析维度

基于读取的案件材料，AI从以下维度进行分析：

| 分析维度 | 分析内容 |
|---------|---------|
| **案件事实** | 事实是否清楚、证据是否充分 |
| **法律适用** | 法律依据是否明确、有无争议 |
| **对方情况** | 对方履行能力、诚信度、调解意愿 |
| **时间成本** | 诉讼周期预估 vs 调解周期 |
| **执行风险** | 判决执行难度 vs 调解履行保障 |
| **关系维护** | 是否需要维持商业/合作关系 |
| **费用成本** | 诉讼费、律师费、保全费等对比 |

#### 1.2 对比分析表

**输出格式**：

| 对比项 | 继续诉讼 | 接受调解 | 备注 |
|-------|---------|---------|------|
| **预期收益** | 全额支持/部分支持/存在风险 | 可协商金额 | |
| **时间成本** | 一审X个月+二审X个月+执行 | 即时或短期履行 | |
| **执行风险** | 需申请强制执行，存在执行不能 | 可约定履行保障 | |
| **费用支出** | 诉讼费+律师费+保全费+执行费 | 可协商分担 | |
| **关系影响** | 对抗性，关系破裂 | 和解性，关系可修复 | |
| **确定性** | 判决结果不确定 | 结果可预期 | |

#### 1.3 建议结论

根据分析结果，给出明确建议：

| 建议类型 | 适用情形 |
|---------|---------|
| **推荐调解** | 证据存在瑕疵、执行风险高、需维持关系、对方有履行意愿 |
| **不推荐调解** | 证据充分、对方无履行能力、恶意拖延、原则性案件 |
| **视条件而定** | 需看对方提出的具体条件、需补充担保措施 |

**输出示例**：
> **建议结论**：推荐调解
> 
> **理由**：
> 1. 本案证据存在瑕疵，【具体瑕疵】，诉讼存在败诉风险
> 2. 对方有履行意愿，已主动提出调解意向
> 3. 双方存在长期合作关系，诉讼将彻底破坏合作基础
> 4. 可要求对方提供【担保方式】作为履行保障

### 模块2：调解方案起草

#### 2.1 调解条件输入

律师提供以下信息：
- 对方提出的调解条件
- 我方可接受的底线
- 必须坚持的条款
- 可让步的空间

#### 2.2 底线分析

**帮助律师梳理**：

| 类别 | 梳理内容 |
|-----|---------|
| **不可让步底线** | 最低可接受金额、必须确认的违约责任、必须保留的权利 |
| **可让步空间** | 金额让步幅度、履行期限延长、部分请求放弃 |
| **交换条件** | 让步可换取的利益（如提前履行、担保、违约金等） |

#### 2.3 调解三原则提示

在调解过程中，提示律师注意：

1. **自愿原则**
   - 调解必须基于双方自愿
   - 不得强迫或变相强迫当事人接受调解
   - 律师应充分尊重当事人意愿

2. **合法原则**
   - 调解协议内容不得违反法律强制性规定
   - 不得损害国家利益、社会公共利益或第三人合法权益
   - 不得规避法律、逃避债务

3. **不公开原则**
   - 调解过程不公开进行
   - 调解协议内容原则上不公开
   - 保护当事人商业秘密和个人隐私

#### 2.4 重要法律提示

> **⚠️ 调解中的让步不构成自认**
> 
> 根据《最高人民法院关于适用〈中华人民共和国民事诉讼法〉的解释》第107条：
> "在诉讼中，当事人为达成调解协议或者和解协议作出妥协而认可的事实，不得在后续的诉讼中作为对其不利的根据，但法律另有规定或者当事人均同意的除外。"
> 
> **实务要点**：
> - 调解过程中的让步和妥协，不得在后续诉讼中作为不利证据
> - 建议在调解开始时明确告知对方此规则
> - 如调解失败，律师应注意防止对方不当使用调解中的陈述

### 模块3：调解书/和解协议审查

#### 3.1 审查要点提示

当对方提出调解方案时，AI辅助审查要点：

| 审查项目 | 审查要点 |
|---------|---------|
| **主体准确性** | 当事人名称/姓名、身份证号/统一社会信用代码、法定代表人/代理人信息是否准确 |
| **金额/标的明确性** | 金额是否具体、计算方式是否清晰、币种是否明确、是否含税 |
| **履行期限合理性** | 履行时间是否明确、是否分期、每期金额和时间节点 |
| **违约责任约定** | 逾期履行的违约金或利息、违约责任的触发条件、救济方式 |
| **争议解决条款** | 调解不成时的处理方式、管辖约定、法律适用 |
| **遗漏事项** | 诉讼费承担、已履行部分确认、其他关联事项处理 |

#### 3.3 诉讼调解 vs 庭外和解的效力区别

**重要提示**：

| 对比项 | 法院调解书 | 庭外和解协议 |
|-------|-----------|-------------|
| **性质** | 司法文书 | 民事合同 |
| **生效时间** | 经双方签收即生效 | 经双方签署即生效 |
| **法律效力** | 具有生效判决同等效力 | 具有合同约束力 |
| **强制执行** | 可直接申请强制执行 | 需先起诉确认效力或申请仲裁 |
| **反悔后果** | 不得反悔，可申请执行 | 可起诉要求履行或承担违约责任 |
| **制作主体** | 法院制作，双方签收 | 双方自行协商签署 |
| **费用** | 诉讼费可减半或协商承担 | 无需诉讼费 |

**实务建议**：
- 如对方履行能力存疑，建议选择法院调解，取得调解书以获得强制执行力
- 如对方诚信度较高且希望保密，可选择庭外和解
- 庭外和解可约定"本协议经双方签署后，任何一方均可向法院申请司法确认"

### 模块4：撤诉管理

#### 4.1 撤诉条件提示

**法律依据**：《民事诉讼法》第148条

> "宣判前，原告申请撤诉的，是否准许，由人民法院裁定。人民法院裁定不准许撤诉的，原告经传票传唤，无正当理由拒不到庭的，可以缺席判决。"

**适用条件**：
- 原告自愿申请
- 须在宣判前提出
- 不得损害国家利益、社会公共利益或他人合法权益
- 不得违反法律禁止性规定

#### 4.2 撤诉申请书

**文档结构**：

**模板路径**：`templates/撤诉申请书.md`

#### 4.3 代理权限提醒

> **⚠️ 重要提示：撤诉需要特别授权**
> 
> 根据《民事诉讼法》及相关司法解释：
> - **一般授权**：仅包括代为起诉、应诉、提供证据、发表代理意见等程序性权利
> - **特别授权**：包括代为承认、放弃、变更诉讼请求，进行和解，提起反诉或上诉，**以及撤诉**
> 
> **实务操作**：
> 1. 撤诉前务必确认授权委托书中包含"代为撤诉"或"特别授权"的明确表述
> 2. 如为一般授权，需联系当事人补充签署特别授权或由其亲自申请撤诉
> 3. 建议在撤诉申请书中附上授权委托书，证明代理权限

#### 4.4 撤诉后的法律后果

| 后果类型 | 具体说明 |
|---------|---------|
| **诉讼费用** | 减半收取，由原告承担（双方可协商分担） |
| **再次起诉** | 可以就同一事实和理由再次起诉（离婚案件除外） |
| **诉讼时效** | 撤诉视为未起诉，诉讼时效从撤诉之日起重新计算 |
| **证据保全** | 诉前保全措施在撤诉后解除 |
| **既判力** | 不产生既判力，不约束后续诉讼 |

**特殊提示**：
- **离婚案件**：原告撤诉后6个月内无新情况、新理由不得再次起诉
- **诉讼时效**：注意撤诉后诉讼时效的计算，避免超过时效

## 第三部分：交互规范

### 3.1 智能追问机制

**追问原则**：
- 每次只追问一个关键问题
- 追问语言简洁专业
- 说明追问的原因
- 等待用户回复后再继续

**常见追问情形**：

| 触发情形 | 追问示例 |
|---------|---------|
| 需要确认调解阶段 | "本案目前处于哪个阶段？A.诉前 B.立案后 C.庭中 D.庭后" |
| 需要确认调解意向 | "对方是否已提出具体调解方案？如有，请提供方案内容。" |
| 需要确认底线 | "本案您方可接受的最低金额是多少？" |
| 需要确认担保 | "是否需要对方提供履行担保？A.需要 B.不需要" |
| 需要确认撤诉授权 | "请确认授权委托书中是否包含'代为撤诉'的特别授权？" |

### 3.2 沟通风格

- 专业但不生硬
- 简洁明了
- 问题导向
- 选项式提问便于回复
- 重要法律提示用⚠️标识

## 第五部分：注意事项

### 5.1 调解注意事项

1. **充分沟通**：调解前与当事人充分沟通，了解其真实意愿和底线
2. **书面确认**：调解方案需经当事人书面确认后方可提出或接受
3. **保留证据**：调解过程中的重要沟通应保留记录
4. **防范风险**：对对方履行能力存疑时，应要求提供担保

### 5.2 和解协议注意事项

1. **条款明确**：避免使用模糊表述，金额、时间、方式必须具体
2. **违约责任**：必须约定违约责任，增加违约成本
3. **争议解决**：约定调解不成时的处理方式
4. **全面性**：确保所有争议事项均在协议中处理，避免遗漏

### 5.3 撤诉注意事项

1. **授权审查**：撤诉前务必审查代理权限
2. **后果告知**：向当事人充分说明撤诉的法律后果
3. **履行确认**：如对方承诺履行后撤诉，应确认已履行或落实担保
4. **时效关注**：注意撤诉后诉讼时效的计算

### 5.4 执业风险提示

1. **不得强迫调解**：尊重当事人意愿，不得强迫或变相强迫接受调解
2. **利益冲突**：注意调解方案是否存在利益冲突
3. **保密义务**：调解过程中的信息应严格保密
4. **记录保存**：妥善保存调解过程中的重要文件和沟通记录

## Wiki 知识库查阅规则

当处理的案件涉及具体罪名或法律领域时，必须查阅 Wiki 知识库：

1. **检查知识库**：检查 `~/wiki/wiki/` 下是否有对应知识页（刑事：`books/criminal/{罪名}.md`，民事：`books/civil/{领域}/{具体问题}.md`）
2. **读取知识卡片**：若有对应页面，读取页面顶部 `<!-- CARD_START -->` 到 `<!-- CARD_END -->` 之间的知识卡片（约800字浓缩摘要）
3. **按需深入**：卡片信息足够则直接引用；需要深入了解具体辩护策略、案例细节、司法解释原文时，读取全文对应板块
4. **融入输出**：将 Wiki 知识自然融入当前工作输出，引用时标注来源页面路径
5. **案例知识页**：同时检查 `~/wiki/wiki/cases/` 下是否有相关判例知识页可供参考
