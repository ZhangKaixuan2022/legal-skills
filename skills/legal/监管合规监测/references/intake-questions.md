# 启动询问

监管合规监测开始前，必须先确认监测对象和客户文件位置。

## 必问问题

```markdown
我先确认监管监测范围：
1. 这次监测对象是行业、具体客户、具体产品/业务线，还是单一主题？
2. 具体行业/主题是什么：数据合规、广告、金融、医疗、食品、劳动、AI、平台、电商、消费者权益等？
3. 如果是客户：客户名称、主营业务、地区、产品/服务、目标用户是什么？
4. 监管地域是全国，还是吉林/广东/长春/深圳等特定地区？
5. 监测期间是本周、本月、最近 90 天，还是指定日期后？
6. 客户现有合规文件放在哪里：制度、隐私政策、用户协议、广告规则、合同模板、员工手册、流程图、培训记录、备案/许可材料等？
7. 你要的输出是内部简报、客户报告、整改清单、政策修改稿，还是正式法律意见？
```

## 演示或测试

用户要求“测试一个行业”但未指定行业时，可默认选择 `数据合规/个人信息保护`，并声明：

- 本次为流程测试，不是正式客户合规意见。
- 未读取客户现有文件，只能演示法规检索、分级和缺口核查模板。
- 北大法宝关键词检索不是全量监管更新监测。
