# 客户合规缺口核查

用于将新规要求与客户现有制度、流程、合同、产品、广告、隐私政策等材料进行比对。

## 核查步骤

1. 建立客户画像：主营业务、产品/服务、地区、用户、行业资质、业务模式。
2. 建立适用规则清单：法律、法规、规章、规范性文件、标准、监管口径。
3. 读取客户现有合规文件，执行文件读取复查。
4. 将新规要求拆成义务项。
5. 逐项匹配客户现有文件和流程。
6. 标注缺口、风险等级、整改建议、责任部门和期限建议。
7. 输出给客户看的整改清单，系统记录保留完整来源和工作笔记。

## 缺口卡片

```markdown
### 合规缺口 001｜缺口标题
- 新规要求：
- 客户现有文件/流程：
- 已读取材料：
- 缺口判断：
- 风险等级：L1/L2/L3/L4
- 整改建议：
- 建议修改文本：
- 责任部门/期限建议：
- 需要客户确认：
- 来源：
```

## 常见客户文件

- 公司合规制度、数据合规制度、个人信息保护制度。
- 隐私政策、用户协议、儿童隐私政策、Cookie/SDK 清单。
- 产品流程、权限弹窗、告知同意页面、注销/撤回同意流程。
- 广告审查制度、营销文案规则、客户案例授权。
- 合同模板、供应商协议、委托处理协议、数据共享协议。
- 员工手册、规章制度、培训记录、审批记录。

材料不足时，只能输出基于现有材料的初步缺口，必须写入缺口归档并提示用户。
