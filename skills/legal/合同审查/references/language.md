# 多语言输出规范

使用合同的**主要语言**进行所有输出：批注标签、概要标题、审核意见、流程图节点文字。

---

## 语言检测规则

1. **检测主要语言**（按合同正文字符比例）：
   - 中文字符（CJK）占比 ≥ 60% → 使用中文
   - 拉丁字母占比 ≥ 60% → 使用英文
2. **双语情况**：以章节标题使用的语言为准
3. **仍不明确时**：询问用户；无回应则默认中文

**禁止在同一输出中混用语言**

### 中文
- `【问题类型】`

- `【修订建议】`

### 英文
- `[Issue Type]`
- `[Risk Reason]`

## 合同概要标题映射（中文）

中文合同使用以下标题/标签：

- `一、合同基本信息`
  - `项目` / `内容`
  - `合同名称` / `合同类型` / `合同当事人`（含主体核查结果） / `签订时间` / `合同期限` / `合同金额`
- `二、业务模式概述`
  - `简要描述：...`
- `三、核心条款要素`
  - `3.1 交易要素`
    - `要素` / `具体内容`
    - `标的物/服务内容` / `数量规格` / `价格构成` / `支付方式` / `交付方式`
  - `3.2 权利义务分配`
    - `甲方主要权利义务：...`
    - `乙方主要权利义务：...`
  - `3.3 履行保障条款`
    - `条款类型` / `具体约定`
    - `违约责任` / `担保措施` / `验收标准` / `质量保证`
  - `3.4 风险分担与特殊约定`
    - `风险分担：...`
    - `特殊约定：...`
  - `3.5 争议解决与合同终止`
    - `项目` / `约定内容`
    - `争议解决方式` / `合同变更` / `合同解除` / `适用法律`
- `四、关键时间节点`

## 审核意见措辞（中文）

第二段开头使用：
- `经审核，本合同存在以下几项主要风险需提请关注：`

然后按处理建议和业务影响排序列出编号项目（1. 2. 3.），不得输出红黄蓝绿标识或单独风险等级标签。

---

## 问题类型枚举（中文）

| 类型代码 | 说明 |
|---------|------|
| 条款矛盾 | 合同不同条款之间存在冲突 |
| 条款缺失 | 缺少必备或重要条款 |
| 主体风险 | 合同相对方存在资信或履约风险 |
| 表述模糊 | 条款表述不明确，容易产生歧义 |
| 违法条款 | 违反法律强制性规定 |
| 数字错误 | 金额、日期、数量等数据有误 |
| 程序缺陷 | 履行程序约定不完整或不可操作 |
| 违约责任 | 违约责任约定不合理 |
| 争议解决 | 管辖或仲裁条款存在问题 |
| 知识产权 | 知识产权归属或授权约定不清 |
| 保密条款 | 保密义务约定不完整 |
