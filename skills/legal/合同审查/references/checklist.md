# 合同审查检查清单（通用版）

> **重要说明**：此为通用检查清单，适用于所有合同类型的基础审查。具体合同类型请结合 `合同类型专项审查规则.md` 进行专项审查。

---

## Layer 0：主体核查（主体真实性）

### 1. 主体提取
- [ ] 识别所有签约方（完整法定名称）
- [ ] 提取统一社会信用代码（如有）
- [ ] 记录法定代表人姓名（如有载明）
- [ ] 识别主体类型（公司、个体工商户、自然人等）

### 2. 名称准确性
- [ ] 注册名称完全匹配（无错别字、无遗漏/多余字符）
- [ ] 主体类型后缀正确（有限公司 vs 有限责任公司等）
- [ ] 如商号与注册名称不同，需同时核实两者

### 3. 真实性与状态
- [ ] 主体在工商登记记录中存在
- [ ] 主体未被吊销/注销/列入异常状态
- [ ] 经营范围涵盖合同标的（如可核实）

### 4. 核查方法
- [ ] 使用 qcc-company 工具查询企业主体信息（工商登记、统一社会信用代码、法定代表人、经营状态、经营范围等）
- [ ] 使用 qcc-risk 工具进行企业风险穿透预警（失信信息、被执行人、经营异常、股权冻结、司法诉讼等18类风险扫描）
- [ ] 在批注中记录核查来源（qcc-company / qcc-risk）

**处理优先级指引**：
- 🔴 重大风险：主体不存在、名称严重错误、已吊销/注销
- 🟡 一般风险：名称轻微差异、状态未能完全核实
- 🔵 注意事项：名称格式差异（如标点符号）

## Layer 1：基础审查（文本质量）

### 1. 文本准确性
- [ ] 关键术语和拼写正确
- [ ] 数字、金额、比例准确
- [ ] 大写金额与数字一致
- [ ] 日期精确（避免"尽快"等模糊表述）

### 2. 格式一致性
- [ ] 标点符号正确
- [ ] 条款编号连续
- [ ] 无重复编号
- [ ] 版面整洁
- [ ] 签章区域预留足够空间

### 3. 表达清晰性
- [ ] 无语法错误
- [ ] 无模糊表述
- [ ] 时间/数量/质量无歧义
- [ ] 术语使用正确

### 4. 内部一致性
- [ ] 同一概念使用一致的命名
- [ ] 交叉引用正确
- [ ] 条款间无逻辑冲突
- [ ] 附件与正文一致

**处理优先级指引**：
- 🔴 重大风险：核心条款（价格、标的、权利义务）存在歧义
- 🟡 一般风险：非核心条款存在歧义
- 🔵 注意事项：实际影响极小

## Layer 2：商务条款

### 1. 目的与期限
- [ ] 目的明确
- [ ] 背景说明清晰
- [ ] 起止日期明确
- [ ] 续约条款清晰（如有）

### 2. 标的
- [ ] 数量具体
- [ ] 类别/品牌/型号/规格清晰
- [ ] 质量标准明确
- [ ] 验收条款可操作
- [ ] 合法性/权属状态清晰

### 3. 价格与付款
- [ ] 价格结构清晰
- [ ] 计价方式明确
- [ ] 币种明确
- [ ] 含税/不含税区分清晰
- [ ] 税负承担明确
- [ ] 付款方式明确
- [ ] 付款节点与履行进度匹配
- [ ] 付款条件可操作
- [ ] 涉及收付款的合同已明确收款账户/收款方式、收款主体和到账认定规则
- [ ] 发票/收据条款清晰

### 4. 履行
- [ ] 履行时间具体
- [ ] 履行地点具体
- [ ] 履行方式详细
- [ ] 履行流程结构化
- [ ] 所有权转移点明确
- [ ] 风险转移点明确
- [ ] 通知义务明确

### 5. 权利义务
- [ ] 主要权利完整
- [ ] 无默示放弃
- [ ] 免责条款合理
- [ ] 主要义务完整
- [ ] 义务标准明确
- [ ] 义务可履行
- [ ] 后合同义务明确
- [ ] 附随权利义务明确

### 6. 知识产权
- [ ] 既有知识产权归属明确
- [ ] 履行过程中产生的知识产权归属明确
- [ ] 知识产权使用范围/目的/期限明确
- [ ] 知识产权转让/许可条款清晰
- [ ] 保护责任明确
- [ ] 保密/竞业限制条款合理

**处理优先级指引**：
- 🔴 重大风险：核心业务因矛盾/歧义/缺漏无法开展
- 🟡 一般风险：存在实质性争议风险但业务仍可开展
- 🔵 注意事项：对业务影响极小

## Layer 3：法律条款

### 1. 生效
- [ ] 成立与生效区分明确
- [ ] 生效条件清晰
- [ ] 条件可行性已考虑
- [ ] 生效前法律责任已约定

### 2. 责任/违约
- [ ] 违约类型明确定义
- [ ] 救济方式清晰
- [ ] 违约金比例合理
- [ ] 责任分配平衡
- [ ] 计算方式明确

### 3. 变更/解除
- [ ] 变更条件明确
- [ ] 变更程序清晰
- [ ] 解除条件合理
- [ ] 解除程序可操作
- [ ] 期满条件明确
- [ ] 存续条款合理
- [ ] 解除后义务明确

### 4. 适用法律
- [ ] 适用法律明确约定
- [ ] 法律选择合理
- [ ] 不违反强制性规定
- [ ] 可执行性已考虑

### 5. 保密
- [ ] 保密信息已定义
- [ ] 保密期限明确
- [ ] 例外情形有限且合理
- [ ] 违约责任明确

### 6. 不可抗力
- [ ] 事件定义合理
- [ ] 通知义务明确
- [ ] 免责条件公平
- [ ] 后续措施明确

### 7. 争议解决
- [ ] 争议解决方式明确
- [ ] 管辖/仲裁机构明确
- [ ] 仲裁与诉讼无冲突
- [ ] 适用法律与争议解决地匹配

### 8. 通知
- [ ] 通知方式明确
- [ ] 地址/联系方式完整
- [ ] 生效时间/条件明确
- [ ] 地址变更通知义务明确

### 9. 授权
- [ ] 授权人员明确
- [ ] 范围/权限明确
- [ ] 期限合理
- [ ] 撤销/变更机制明确

### 10. 其他法律条款
- [ ] 解释规则明确
- [ ] 签署时间/地点明确
- [ ] 可分割性条款明确

**处理优先级指引**：
- 🔴 重大风险：法律条款缺失或责任不合理
- 🟡 一般风险：其他实质性问题
- 🔵 注意事项：轻微问题

## 使用说明

### 审查流程
1. 按顺序审查：Layer 0 → Layer 1 → Layer 2 → Layer 3
2. 评估每个检查项
3. 记录发现和处理建议
4. 向文档添加结构化批注

### 批注格式

每个批注必须包含：
- **问题类型**（对应检查项）
- **修改位置**（精确到"第X条第Y款"或"建议在第X条后新增"）
- **风险原因**（影响和理由）
- **修订建议**（可操作的修复方案，必须包含**修改后条款的完整全文**及精确的修改/插入位置）

**修订建议格式要求**：
- 修改现有条款：需显示原文 + 建议修改后的完整条款全文（含条款号、条款名和完整表述）
- 新增条款：需给出完整新条款全文及建议插入位置

**示例**：
```
【问题类型】违约金条款约定过高
【修改位置】第8条第2款
【风险原因】违约金比例超过《民法典》第585条规定的合理范围，法院可能调减
【修订建议】
原文："第8条第2款 违约金
乙方逾期交货的，每逾期一日按合同总金额的30%向甲方支付违约金。"

建议修改为：
"第8条第2款 违约金
乙方逾期交货的，每逾期一日按逾期交货部分货款的万分之五向甲方支付违约金；违约金总额不超过合同总金额的10%。"
```

内部可按处理优先级排序；**不要**在批注正文或飞书问题卡中添加"风险等级"行。

### 优先级
- **重大风险**：优先处理
- **一般风险**：其次处理
- **注意事项**：可选清理
