# 刑事案件档案目录结构

> **用途**：标准化刑事案件电子档案管理，确保文件有序归档
> **使用时机**：作为刑事事项阶段目录参考；实际建档和路径维护由法律工作总控按 `matter-workspace-protocol.md` 执行
> **新路径规则**：刑事事项必须遵循法律工作总控的 `matter-workspace-protocol.md`。本文件中的阶段目录作为业务文件区使用；案件信息卡、阶段进展、材料清单、来源验证、工作笔记、OCR 文本等系统记录应放入 `_系统记录/诉讼/刑事/<客户或委托人>/<案件简称或案号>/`。

## 二、目录命名规范

### 2.1 案件根目录命名

**格式**：`[当事人姓名]-[罪名]-刑事案件`

**示例**：
- `张某某-诈骗罪-刑事案件`
- `李某某-故意伤害罪-刑事案件`
- `王某某-职务侵占罪-刑事案件`

**特殊情况**：
- 涉及多罪名：使用主要罪名，如 `张某某-诈骗罪等-刑事案件`
- 涉及多名当事人：分别建立目录，或使用 `张某某等-诈骗罪-刑事案件`

### 2.2 阶段子目录命名

| 序号 | 目录名称 | 创建时机 | 说明 |
|-----|---------|---------|------|
| 00 | 案件总览 | 首次启动时 | 法律工作总控创建或确认 |
| 01 | 案件承接 | 进入案件承接Skill时 | 案件承接与委托创建 |
| 02 | 侦查阶段 | 进入侦查阶段Skill时 | 侦查阶段辩护创建 |
| 03 | 审查起诉阶段 | 进入审查起诉Skill时 | 审查起诉阶段辩护创建 |
| 04 | 一审阶段 | 进入一审Skill时 | 一审阶段辩护创建 |
| 05 | 二审阶段 | 进入二审Skill时 | 二审阶段辩护创建 |
| 06 | 特殊程序 | 涉及特殊程序时 | 相关Skill创建 |
| 07 | 归档总结 | 案件结案时 | 法律工作总控创建或确认 |

### 2.3 文件命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 会见笔录 | `第X次会见笔录_YYYYMMDD.docx` | `第一次会见笔录_20240315.docx` |
| 法律文书 | `[文书名称].docx` | `取保候审申请书.docx` |
| 司法文书 | `[文书名称].pdf` | `逮捕通知书.pdf` |
| 工作记录 | `[描述].md` | `案件法律分析报告.md` |
| 证据材料 | `证据[序号]_[名称].[格式]` | `证据1_银行流水.pdf` |

## 三、目录创建规则

### 3.1 首次启动时创建

法律工作总控在首次启动案件时，创建或确认业务文件区和系统记录区。业务文件区创建阶段目录；系统记录区创建核心记录文件。本文件只提供刑事阶段目录参考。

```bash
mkdir -p "<LEGAL_WORKSPACE>/诉讼/刑事/[客户或委托人]/[案件简称或案号]/00-案件总览"
mkdir -p "<LEGAL_WORKSPACE>/_系统记录/诉讼/刑事/[客户或委托人]/[案件简称或案号]"
touch "<LEGAL_WORKSPACE>/_系统记录/诉讼/刑事/[客户或委托人]/[案件简称或案号]/案件基本信息.md"
touch "<LEGAL_WORKSPACE>/_系统记录/诉讼/刑事/[客户或委托人]/[案件简称或案号]/阶段记录.md"
touch "<LEGAL_WORKSPACE>/_系统记录/诉讼/刑事/[客户或委托人]/[案件简称或案号]/材料清单.md"
```

### 3.2 阶段进入时创建

各阶段Skill在首次进入时，创建对应阶段子目录：

```bash
# 示例：进入侦查阶段时
mkdir -p "[案件根目录]/02-侦查阶段/{会见记录,法律文书,案件分析,司法文书}"
```

### 3.3 按需创建

以下目录仅在实际需要时创建：

- `06-特殊程序/` 及其子目录
- `辩护证据/` 目录（当有辩护证据时）
- `新证据/` 目录（当发现新证据时）

## 四、文件存放规则

### 4.1 各阶段输出物归档位置

| Skill | 主要输出物 | 存放目录 |
|------|-----------|---------|
| 案件承接与委托 | 委托合同、授权委托书、律所函 | 01-案件承接/ |
| 侦查阶段辩护 | 会见笔录、取保候审申请书、辩护意见书 | 02-侦查阶段/ |
| 审查起诉阶段辩护 | 证据审查报告、辩护意见、不起诉申请书 | 03-审查起诉阶段/ |
| 一审阶段辩护 | 辩护词、质证意见、量刑意见书 | 04-一审阶段/ |
| 二审阶段辩护 | 上诉状、二审辩护词 | 05-二审阶段/ |
| 未成年人案件 | 社会调查报告、附条件不起诉申请 | 相关阶段目录 |
| 死刑案件 | 死刑复核辩护意见 | 06-特殊程序/死刑复核/ |
| 特殊程序 | 各类特殊程序文书 | 06-特殊程序/ |
| 简易速裁程序 | 认罪认罚相关材料 | 相关阶段目录 |

### 4.2 跨阶段文件处理

| 文件类型 | 处理方式 |
|---------|---------|
| 案件信息卡/案件基本信息 | 始终存放在系统记录区，由法律工作总控统一维护 |
| 累进更新的报告 | 各版本存放在产生该版本的阶段目录 |
| 司法文书（收到的） | 存放在收到时所处阶段目录 |
| 工作总结 | 每阶段目录下独立存放 |

## 五、档案目录使用指引

### 5.1 总调度Skill使用方式

1. **首次启动**：
   - 根据当事人姓名和罪名创建案件根目录
   - 创建00-案件总览/目录
   - 初始化案件信息卡.md

2. **阶段转换时**：
   - 检查下一阶段目录是否存在，不存在则创建
   - 更新案件信息卡中的档案路径信息
   - 更新阶段进展记录.md

3. **案件结案时**：
   - 创建07-归档总结/目录
   - 生成案件全程总结报告

### 5.2 各阶段Skill使用方式

1. **进入阶段时**：
   - 从案件信息卡读取档案根目录路径
   - 检查并创建本阶段子目录
   - 按规范创建子文件夹

2. **生成文书时**：
   - 将文书保存至对应子目录
   - 使用标准命名格式

3. **阶段结束时**：
   - 整理本阶段所有文件
   - 生成工作总结.md

## 六、目录创建命令模板

### 6.1 完整目录创建（一次性）

```bash
# 替换变量后执行
案件目录="张某某-诈骗罪-刑事案件"
基础路径="<LEGAL_WORKSPACE>"

mkdir -p "$基础路径/$案件目录/00-案件总览"
mkdir -p "$基础路径/$案件目录/01-案件承接/{委托材料,律所函件,案件评估}"
mkdir -p "$基础路径/$案件目录/02-侦查阶段/{会见记录,法律文书,案件分析,司法文书}"
mkdir -p "$基础路径/$案件目录/03-审查起诉阶段/{案卷材料,案卷材料/电子卷宗,证据审查,会见记录,法律文书,案件分析}"
mkdir -p "$基础路径/$案件目录/04-一审阶段/{起诉材料,庭审准备,法律文书,辩护证据,庭审记录,裁判文书}"
mkdir -p "$基础路径/$案件目录/05-二审阶段/{上诉材料,法律文书,会见记录,新证据,裁判文书}"
mkdir -p "$基础路径/$案件目录/06-特殊程序"
mkdir -p "$基础路径/$案件目录/07-归档总结"
```

### 6.2 按阶段创建

```bash
# 首次启动（总调度）
mkdir -p "$案件目录/00-案件总览"

# 案件承接阶段
mkdir -p "$案件目录/01-案件承接/{委托材料,律所函件,案件评估}"

# 侦查阶段
mkdir -p "$案件目录/02-侦查阶段/{会见记录,法律文书,案件分析,司法文书}"

# 审查起诉阶段
mkdir -p "$案件目录/03-审查起诉阶段/{案卷材料/电子卷宗,证据审查,会见记录,法律文书,案件分析}"

# 一审阶段
mkdir -p "$案件目录/04-一审阶段/{起诉材料,庭审准备,法律文书,辩护证据,庭审记录,裁判文书}"

# 二审阶段
mkdir -p "$案件目录/05-二审阶段/{上诉材料,法律文书,会见记录,新证据,裁判文书}"
```
