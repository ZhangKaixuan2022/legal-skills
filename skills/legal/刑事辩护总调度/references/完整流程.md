# 刑事辩护总调度Skill

## 模块一：功能概述与Skill体系全景

### 1.1 总调度Skill定位

本 Skill 是刑事辩护 Skill 体系的**专业流程编排器**，承担以下职责：

| 职责 | 说明 |
|-----|------|
| **阶段路由** | 根据刑事诉讼阶段路由至对应专业 Skill |
| **特殊标签分流** | 根据未成年人、死刑、认罪认罚、特殊程序等标签叠加或替代路由 |
| **阶段转换提示** | 识别阶段转换标志，提示下一阶段关键动作 |
| **期限提示** | 提醒刑事程序关键时间节点 |
| **下一步建议** | 根据当前阶段推荐优先工作 |

**启动方式**：用户提供案件基本信息（当事人姓名、涉嫌罪名、当前阶段）即可启动。

### 1.2 刑事辩护Skill体系全景表

本体系包含9个专业Skill，覆盖刑事辩护全流程：

| 编号 | Skill名称 | 目录名 | 定位 | 主要功能 |
|-----|----------|-------|------|---------|
| 1 | 案件承接 | 案件承接与委托 | 刑事委托起点 | 接待、评估、风险告知、委托手续、档案建立 |
| 2 | 侦查阶段 | 侦查阶段辩护 | 侦查期辩护 | 会见、取保候审、不予批捕意见、量刑预判 |
| 3 | 审查起诉 | 审查起诉阶段辩护 | 审查起诉期辩护 | 阅卷、证据审查、辩护意见、不起诉申请 |
| 4 | 一审阶段 | 一审阶段辩护 | 一审庭审辩护 | 庭前准备、质证、辩论、辩护词、量刑辩护 |
| 5 | 二审阶段 | 二审阶段辩护 | 上诉审辩护 | 上诉、开庭申请、二审辩护、改判策略 |
| 6 | 未成年人 | 未成年人案件 | 特殊主体保护 | 社会调查、附条件不起诉、犯罪记录封存 |
| 7 | 死刑案件 | 死刑案件 | 死刑案件专项 | 避免死立执、死刑复核、被害人和解 |
| 8 | 特殊程序 | 特殊程序 | 特殊程序处理 | 违法所得没收、强制医疗、再审申诉 |
| 9 | 简易/速裁 | 简易速裁程序 | 简化程序分支 | 简易程序、速裁程序、认罪认罚 |

### 1.3 主线流程图示

刑事诉讼主线流程如下：

```
案件承接 ──→ 侦查阶段 ──→ 审查起诉 ──→ 一审阶段 ──→ 二审阶段
    │            │            │            │            │
    ▼            ▼            ▼            ▼            ▼
委托手续      会见会谈      全面阅卷      庭审辩护      上诉辩护
风险评估      取保申请      证据审查      质证辩论      争取改判
档案建立      批捕意见      辩护意见      辩护词        
```

### 1.4 分支关系图示

特殊标签触发的分支Skill：

```
主线任意阶段
    │
    ├──→ 检测到"未成年人" ──→ 附加 未成年人案件
    │
    ├──→ 检测到"可能死刑" ──→ 附加 死刑案件
    │
    ├──→ 检测到"简易/速裁程序" ──→ 替代一审主线 简易速裁程序
    │
    └──→ 检测到"特殊程序" ──→ 附加 特殊程序
         （违法所得没收/强制医疗/再审申诉）
```

## 模块二：路由信息读取

### 2.1 信息来源

本 Skill 不独立创建或维护案件信息卡。刑事事项的基本信息、当前状态、阶段记录、材料清单和工作笔记由法律工作总控的系统记录区维护。

本 Skill 只读取与专业路由直接相关的字段，并据此判断主线路由和附加路由。

> 📋 字段参考 → skills/legal/刑事辩护总调度/reference/案件信息卡模板.md

### 2.2 核心路由字段

#### 一、基本信息（必填）

| 字段 | 说明 | 示例 |
|-----|------|------|
| 当事人姓名 | 犯罪嫌疑人/被告人姓名 | 张某某 |
| 案件编号 | 公安/检察/法院编号（动态更新） | 沪公X（刑）字[2024]XXX号 |
| 罪名 | 涉嫌/被指控罪名 | 诈骗罪 |
| 委托人 | 委托人姓名及与当事人关系 | 李某某（配偶） |
| 承办律师/律所 | 主办律师及所属律所 | 潘睿/广东广和（长春）律师事务所 |
| 委托日期 | 签订委托合同日期 | 2024年X月X日 |

#### 二、案件状态标记（核心路由字段）

| 字段 | 选项 | 说明 |
|-----|------|------|
| 当前诉讼阶段 | 案件承接/侦查/审查起诉/一审/二审/特殊程序 | 决定主线路由目标 |
| 当前阶段启动日期 | 日期 | 计算时间节点依据 |
| 关键时间节点 | 下次开庭/审限到期/上诉期限等 | 预警提示依据 |

#### 三、特殊标签（分支路由字段）

| 标签 | 选项 | 路由影响 |

| 是否未成年人案件 | 是/否 | 是→附加未成年人案件 |
| 是否可能判处死刑 | 是/否 | 是→附加死刑案件 |
| 是否认罪认罚 | 是/否 | 是→启用认罪认罚模块 |
| 是否涉及特殊程序 | 无/违法所得没收/强制医疗/再审申诉 | 有→附加特殊程序 |
| 适用审理程序 | 普通程序/简易程序/速裁程序 | 简易/速裁→附加简易速裁程序 |

#### 四、事项路径

| 字段 | 说明 |
|-----|------|
| 业务文件路径 | 由法律工作总控维护 |
| 系统记录路径 | 由法律工作总控维护 |
| 已完成阶段记录 | 各阶段完成情况及主要输出物 |

## 模块四：智能路由引擎

### 4.1 路由机制说明

智能路由引擎根据案件信息卡中的**当前诉讼阶段**和**特殊标签**，确定路由目标。

**路由逻辑**：
1. 首先根据"当前诉讼阶段"确定**主线路由目标**
2. 然后根据"特殊标签"确定**附加路由目标**
3. 输出路由指令供用户参考

### 4.2 主线路由规则

| 当前诉讼阶段 | 主线路由目标 | Skill路径 |
|------------|------------|----------|
| 案件承接 | 案件承接Skill | skills/legal/案件承接与委托/SKILL.md |
| 侦查阶段 | 侦查阶段Skill | skills/legal/侦查阶段辩护/SKILL.md |
| 审查起诉 | 审查起诉Skill | skills/legal/审查起诉阶段辩护/SKILL.md |
| 一审阶段 | 一审阶段Skill | skills/legal/一审阶段辩护/SKILL.md |
| 二审阶段 | 二审阶段Skill | skills/legal/二审阶段辩护/SKILL.md |
| 特殊程序 | 特殊程序Skill | skills/legal/特殊程序/SKILL.md |

> **说明**：当"当前诉讼阶段"为"特殊程序"时，主线路由直接指向特殊程序Skill，同时仍可依据特殊标签叠加路由其他子Skill。

### 4.3 分支路由规则（叠加触发）

| 特殊标签条件 | 附加路由目标 | Skill路径 | 触发方式 |
|------------|------------|----------|---------|
| 未成年人案件=是 | 未成年人Skill | skills/legal/未成年人案件/SKILL.md | 叠加 |
| 可能判处死刑=是 | 死刑案件Skill | skills/legal/死刑案件/SKILL.md | 叠加 |
| 适用程序=简易/速裁 | 简易速裁Skill | skills/legal/简易速裁程序/SKILL.md | 替代一审 |
| 涉及特殊程序≠无 | 特殊程序Skill | skills/legal/特殊程序/SKILL.md | 叠加 |

### 4.4 路由输出格式

```
═══════════════════════════════════════════════════════════
📋 智能路由结果
═══════════════════════════════════════════════════════════

【主线路由】
> 📋 当前阶段：[阶段名称]
> 📋 路由目标：skills/legal/[目录]/SKILL.md

【附加路由】（如有特殊标签）
> 📋 特殊标签：[标签名称]
> 📋 附加Skill：skills/legal/[目录]/SKILL.md

═══════════════════════════════════════════════════════════
```

## 模块五：阶段转换管理

### 5.1 阶段转换标志事件

| 转换 | 标志事件 | 典型文书/情形 |
|-----|---------|-------------|
| 承接→侦查 | 签订委托协议 | 刑事委托代理合同 |
| 侦查→审查起诉 | 侦查终结，移送起诉 | 移送审查起诉通知书 |
| 审查起诉→一审 | 提起公诉 | 起诉书 |
| 一审→二审 | 上诉或抗诉 | 上诉状/抗诉书 |

### 5.2 阶段转换时律师关键动作

| 转换 | 关键动作 | 时限要求 |
|-----|---------|---------|
| 承接→侦查 | 递交委托手续、安排首次会见、了解基本案情 | 签约后24-48小时内 |
| 侦查→审查起诉 | 获取起诉意见书、**全面阅卷**、制作证据分析表 | 移送后尽快 |
| 审查起诉→一审 | 获取起诉书、**对比起诉书与起诉意见书差异**、庭前会议准备 | 收到起诉书后立即 |
| 一审→二审 | 案卷交接、**半月内提交初步辩护意见**、争取开庭审理 | 上诉后尽快 |

### 5.3 阶段转换执行流程

```
检测到阶段转换标志事件
    │
    ├─→ 1. 更新案件信息卡
    │       - 当前诉讼阶段
    │       - 当前阶段启动日期
    │       - 关键时间节点
    │
    ├─→ 2. 执行阶段转换检查清单
    │       - 确认前阶段必须完成事项
    │       - 列出转换后关键动作
    │
    ├─→ 3. 提示法律工作总控同步阶段记录
    │
    ├─→ 4. 向用户确认转换
    │
    └─→ 5. 推荐下一步工作
```

> 📋 按需读取详细检查清单 → skills/legal/刑事辩护总调度/checklists/阶段转换检查清单.md

## 模块六：案件状态追踪与主动推进

### 6.1 阶段完成度检查

根据各Skill的核心输出物清单，检查当前阶段工作完成情况：

| 阶段 | 核心输出物 | 完成标准 |
|-----|-----------|---------|
| 案件承接 | 委托合同、授权委托书、律所函、案件档案 | 4项全部完成 |
| 侦查阶段 | 会见笔录（至少1次）、量刑预判（初版） | 会见完成+初步分析 |
| 审查起诉 | 证据审查报告、辩护意见 | 阅卷完成+意见提交 |
| 一审阶段 | 辩护词、出庭辩护 | 完成庭审 |
| 二审阶段 | 上诉状、二审辩护意见 | 完成二审程序 |

### 6.2 时间节点预警

各阶段关键时间节点及预警规则：

| 阶段 | 时间节点 | 法定期限 | 预警时机 |
|-----|---------|---------|---------|
| 侦查阶段 | 拘留期限 | 30+7天 | 批捕审查前5日 |
| 侦查阶段 | 逮捕后侦查期限 | 2+3个月 | 期限届满前15日 |
| 审查起诉 | 审查期限 | 1+0.5个月 | 期限届满前7日 |
| 一审阶段 | 一审审限 | 3个月（可延长） | 期限届满前15日 |
| 二审阶段 | 二审审限 | 2+2个月 | 期限届满前15日 |
| 速裁程序 | 审结期限 | 10日 | 开庭前3日 |
| 一审判决后 | 上诉期限 | 10日 | 判决送达后立即 |

### 6.3 时间节点预警输出格式

```
═══════════════════════════════════════════════════════════
⏰ 时间节点预警
═══════════════════════════════════════════════════════════

【即将到期】
⚠️ 批捕审查截止日期：2024年X月X日（还剩5天）
   → 建议立即：提交不予批准逮捕律师意见书

【已过期】
🔴 审查起诉期限已届满
   → 请关注检察院是否延期或退回补充侦查

═══════════════════════════════════════════════════════════
```

### 6.4 智能推荐下一步

基于案件当前状态，推荐最优先的工作项：

| 当前状态 | 推荐下一步 | 优先级 |
|---------|-----------|-------|
| 刚签订委托 | 递交委托手续、安排首次会见 | 高 |
| 首次会见完成 | 制作会见笔录、初步量刑预判 | 高 |
| 批捕前5日 | 提交不予批捕意见书 | 紧急 |
| 刚移送审查起诉 | 联系检察官、申请阅卷 | 高 |
| 阅卷完成 | 制作证据审查报告、核实性会见 | 高 |
| 收到起诉书 | 对比差异分析、庭前准备 | 高 |
| 开庭前3日 | 会见被告人、最终庭审准备 | 高 |
| 判决后 | 分析判决、协助决定是否上诉 | 紧急（上诉期10日） |

### 6.5 委托人沟通提醒

在以下节点提醒律师进行委托人沟通：

| 节点 | 沟通内容 |
|-----|---------|
| 阶段转换时 | 通报阶段变化、说明下一阶段工作重点 |
| 重要时间节点前 | 提醒关键期限、确认工作安排 |
| 收到重要文书时 | 通报文书内容、分析影响 |
| 重大决策前 | 上诉/认罪认罚等重大决策需委托人同意 |

## 模块八：十大基本原则（全局适用版）

### 8.1 原则概述

依据《刑事辩护实务操作技能与执业风险防范》（徐宗新著）第一章"基本原则"，刑事辩护律师应遵循以下十大原则，**贯穿刑事辩护全流程所有阶段**：

### 8.2 十大原则精要

| 原则 | 核心要义 | 实务要点 |
|-----|---------|---------|
| **原则一：以事实为依据** | 律师观点必须以事实为依据 | 不以当事人辩解或道听途说为唯一依据，需审查证据后准确提出观点 |
| **原则二：以法律为准绳** | 每一步工作都应有法律依据 | 主动寻找法律、司法解释、会议纪要、司法政策作为依据 |
| **原则三：恪守职业道德和执业纪律** | 严格遵守执业规范 | 诚实守信、勤勉尽责、保守秘密、不违规会见传递信息 |
| **原则四：认真维护当事人合法权益** | 维护的是**合法**权益 | 用正当合法手段维护，抓住有利机会，"律师画地图，当事人选路" |
| **原则五：独立辩护** | 依法独立进行，不受委托人意志限制 | 与当事人意见不一致时妥善处理，坚持原则但有效沟通 |
| **原则六：保守秘密** | 保守国家秘密、商业秘密和委托人隐私 | 例外：知悉危害国家安全、公共安全及严重危害他人人身安全的犯罪应告知 |
| **原则七：有理有利有节** | 与司法机关建立良性业务互动 | 不请客送礼贿赂司法人员，不以特殊关系招揽业务或承诺 |
| **原则八：不打包票** | 不对案件结果作明确承诺 | 只承诺专业能力和敬业态度，分析可能性而非承诺结果 |
| **原则九：不贬损同行** | 尊重其他律师、律师事务所 | 客观评价同行意见，不诋毁贬低 |
| **原则十：不诋毁司法机关** | 维护司法机关形象 | 客观分析案件结果，不将败诉责任全归咎于司法机关 |

### 8.3 保密义务的例外

根据《刑事诉讼法》第四十八条：

> 辩护律师知悉委托人或其他人**准备或正在实施**危害国家安全、公共安全以及**严重危害他人人身安全**的犯罪的，应当**及时告知**司法机关。

**操作要点**：
1. 准确把握"准备或正在实施"的现实可能性
2. 先进行劝导制止
3. 无法把握时向律师事务所、律师协会报告

### 8.4 与当事人意见不一致的处理

| 情形 | 处理方式 |
|-----|---------|
| 律师认为有罪，当事人认为无罪 | 充分分析说服，无法说服可解除委托或建议再委托一名无罪辩护律师 |
| 律师认为无罪，当事人认为有罪 | 让当事人实事求是陈述，律师依法独立发表辩护意见 |
| 律师认为必须说，当事人不让说 | 充分考虑当事人意见，妥善处理 |
| 律师认为不必说，当事人坚持说 | 充分沟通，既满足合理要求又依法依规 |
| 当事人要求走关系 | 坚持原则，提醒防止诉讼诈骗 |

## 附录六：常见问题处理指引

### Q1：案件刚接受委托，从哪里开始？

**回答**：
1. 首先通过总调度Skill采集基本信息，创建案件信息卡
2. 确定当前阶段为"案件承接"
3. 路由至 案件承接与委托 完成委托手续
4. 委托手续完成后，阶段自动转换至"侦查阶段"

### Q2：案件已经在审查起诉阶段，如何快速接入？

**回答**：
1. 提供案件基本信息（当事人、罪名、委托人等）
2. 明确当前阶段为"审查起诉"
3. 总调度将直接路由至 审查起诉阶段辩护
4. 如前阶段无工作记录，该Skill将以独立模式启动

### Q3：案件涉及多个特殊标签如何处理？

**回答**：
特殊标签可以叠加触发，例如：
- 未成年人 + 可能死刑 → 同时路由至 未成年人案件 和 死刑案件
- 认罪认罚 + 速裁程序 → 路由至 简易速裁程序（包含认罪认罚模块）

建议优先处理主线工作，在需要时按需读取附加Skill的专项指导。

### Q4：阶段判断不确定怎么办？

**回答**：
根据以下标志判断当前阶段：

| 标志 | 对应阶段 |
|-----|---------|
| 刚签订委托协议，当事人被拘留/逮捕中 | 侦查阶段 |
| 收到移送审查起诉通知书 | 审查起诉阶段 |
| 收到起诉书副本 | 一审阶段 |
| 已提起上诉或收到抗诉通知 | 二审阶段 |

如仍不确定，请提供案件当前的具体情况，总调度将协助判断。

### Q5：如何查看案件整体进展？

**回答**：
1. 查看 `00-案件总览/案件信息卡.md` 获取案件基本信息和当前状态
2. 查看 `00-案件总览/阶段进展记录.md` 获取各阶段工作进展
3. 各阶段目录下的 `工作总结.md` 记录该阶段详细工作

### Q6：时间节点如何计算？

**回答**：
关键时间节点计算规则：

| 节点 | 计算起点 | 期限 |
|-----|---------|------|
| 批捕审查截止 | 刑事拘留日期 | +37天（30+7） |
| 侦查羁押届满 | 逮捕日期 | +5个月（2+3） |
| 审查起诉届满 | 移送日期 | +1.5个月 |
| 一审审限届满 | 法院受理日期 | +3个月 |
| 上诉期限截止 | 判决送达日期 | +10日 |
| 二审审限届满 | 二审受理日期 | +4个月（2+2） |

## Wiki 知识库查阅规则

当处理的案件涉及具体罪名或法律领域时，必须查阅 Wiki 知识库：

1. **检查知识库**：检查 `~/wiki/wiki/` 下是否有对应知识页（刑事：`books/criminal/{罪名}.md`，民事：`books/civil/{领域}/{具体问题}.md`）
2. **读取知识卡片**：若有对应页面，读取页面顶部 `<!-- CARD_START -->` 到 `<!-- CARD_END -->` 之间的知识卡片（约800字浓缩摘要）
3. **按需深入**：卡片信息足够则直接引用；需要深入了解具体辩护策略、案例细节、司法解释原文时，读取全文对应板块
4. **融入输出**：将 Wiki 知识自然融入当前工作输出，引用时标注来源页面路径
5. **案例知识页**：同时检查 `~/wiki/wiki/cases/` 下是否有相关判例知识页可供参考
