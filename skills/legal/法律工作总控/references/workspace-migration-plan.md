# 旧文件迁移计划

本文件记录 `【自定义工作目录】/` 下历史客户目录迁移到“业务文件区 + 系统记录区”双路径结构的方案和后续安排。

## 一、迁移目标

最终目录：

```text
【自定义工作目录】/
  合同/
  劳动合规/
  诉讼/
    民事/
    刑事/
  _系统记录/
    当前事项.md
    总索引.md
    合同/
    劳动合规/
    诉讼/
      民事/
      刑事/
    合同偏好库/
```

## 二、迁移步骤

1. 扫描旧目录，生成《旧文件迁移清单.md》。
2. 对每个旧客户目录判断业务类型：合同、民事诉讼、劳动争议、刑事、破产、其他。
3. 一个客户多个案件或多个合同的，拆成多个事项；不得只按客户名整体迁移。
4. 不确定分类的，先列入“需用户确认”，不得自行移动。
5. 用户确认迁移清单后，创建新业务文件区和系统记录区。
6. 给用户看的材料、文书、交付文件和最终版移入业务文件区。
7. 材料清单、事实时间线、来源验证、OCR、缺口归档、工作笔记、版本记录等移入 `_系统记录`。
8. 每个事项生成 `迁移记录.md`，记录旧路径、新路径、迁移时间、文件数量和存疑项。
9. 迁移完成后校验旧路径引用。
10. 用户确认迁移无误后，删除 Skill 中所有历史兼容描述。

## 三、诉讼总调度降级安排

以下 Skill 已按降级目标处理：

- `民事一审诉讼`
- `劳动争议诉讼`
- `刑事辩护总调度`

降级目标：

1. 保留为专业流程编排 Skill。
2. 不再承担法律工作总控的职责。
3. 不再维护通用路径规则、来源边界、文件读取、OCR、当前事项等共享规则。
4. 只保留阶段流转、状态判断、专业任务清单、子 Skill 调用顺序和领域特有规则。
5. 三个 Skill 均必须引用 `法律工作总控` 的共享协议，不复制通用规则。
