# 合同偏好学习协议

本协议用于合同审查、合同起草、合同问答、版本比对和最终版归档中的“自我学习”。它只学习潘睿律师在具体业务中的接受偏好、谈判底线和处理习惯，不自动改变法律强制规则或合同专项审查标准。

## 一、基本原则

1. 只生成提案，不静默改规则。
2. 用户明确确认后，才写入已确认偏好。
3. 法律风险判断与业务接受偏好必须分开。
4. 已接受的条款不能当然视为无风险，只能说明“历史上曾接受”。
5. 未读取原合同、未完成读取复查、OCR 存疑或来源不清的内容，不得作为学习样本。
6. 法律法规、强制性规范、无效条款、违法条款、主体资格问题、授权瑕疵，不因历史接受而降低法律风险。
7. 偏好适用必须标注来源，不得靠模型记忆。

## 二、偏好库位置

按需创建，不批量改造历史案件：

```text
<LEGAL_WORKSPACE>/_系统记录/合同偏好库/
  合同偏好索引.md
  偏好样本记录.md
  偏好更新提案.md
  已确认偏好规则.md
  偏好变更历史.md
```

偏好记录统一写入 `_系统记录/合同偏好库/`。

## 三、可学习样本

合同审查或起草过程中，出现以下情形时，记录到 `偏好样本记录.md`：

1. 用户明确表示接受某条款、接受某项风险、放弃修改或按某个底线谈判。
2. 最终版比对显示某项审查意见未被采纳，但用户仍确认定稿、签署或归档。
3. 起草合同多轮修改后，用户反复保留或要求加入某类条款。
4. 用户明确说“以后这类合同可以这样处理”“这个条款我一般接受”。

样本记录格式：

```text
| 日期 | 合同编号 | 合同类型 | 我方立场 | 条款类型 | 条款位置 | 原处理建议 | 最终处理 | 接受原因 | 是否仅业务偏好 | 来源 |
|---|---|---|---|---|---|---|---|---|---|---|
```

## 四、触发偏好提案

满足任一条件时，生成 `偏好更新提案.md`，并询问用户是否确认：

1. 同一合同类型、同一条款类型，在最近 5 个可比样本中有 3 次以上被同样接受。
2. 同一客户或项目中，某类条款连续 2 次以上被同样接受。
3. 用户明确要求“以后这样处理”“把这个作为默认偏好”。

可比样本必须同时满足：

- 合同类型相同或高度近似。
- 我方立场相同或可比。
- 条款功能相同。
- 接受原因相同或相近。
- 来源合同编号可追溯。

不可比样本不得强行合并。

## 五、偏好提案格式

```markdown
# 合同偏好更新提案

- 提案编号：
- 触发原因：
- 涉及合同类型：
- 涉及条款类型：
- 适用范围：全局 / 合同类型 / 客户或项目
- 最近样本：
- 原审查规则或默认处理：
- 实际处理结果：
- 建议偏好：
- 对法律风险判断的影响：
- 对业务风险提示的影响：
- 对处理建议的影响：
- 不适用情形：
- 来源：
- 是否需要潘睿确认：是
```

## 六、确认与写入

用户明确确认后，写入 `已确认偏好规则.md`，并同步更新 `合同偏好索引.md` 和 `偏好变更历史.md`。

已确认偏好格式：

```markdown
## 偏好编号：

- 确认日期：
- 适用范围：全局 / 合同类型 / 客户或项目
- 合同类型：
- 我方立场：
- 条款类型：
- 已确认偏好：
- 默认处理建议调整：
- 法律风险是否改变：否 / 仅限已验证的业务判断
- 仍需提示的风险：
- 不适用情形：
- 样本来源：
- 确认方式：
```

用户只确认偏好时，默认只修改偏好库；只有用户明确要求“修改合同专项审查规则”或“上升为通用审查规则”时，才修改 Skill 或专项审查 reference。

## 七、使用偏好

合同审查或起草时，如命中已确认偏好：

1. 先按法律规则和合同专项审查规则完成独立判断。
2. 再读取相关偏好规则，作为处理建议和谈判底线的叠加层。
3. 输出中必须区分：
   - 法律风险：来自合同原文、法规案例检索或模型分析。
   - 业务偏好：来自合同偏好库。
   - 处理建议：在法律风险和业务偏好之间做出的本次建议。
4. 不得因为偏好存在而省略风险提示。

输出示例：

```markdown
- 法律风险：付款条件与验收节点绑定不清，可能引发履行争议。
- 业务风险：我方可能先履行但回款周期拉长。
- 潘睿历史偏好：同类服务合同中曾接受“验收后30日付款”，来源为合同偏好库。
- 处理建议：需客户确认
- 来源：合同原文第X条；合同偏好库《已确认偏好规则.md》偏好编号XXX。
```

## 八、禁止自动学习的事项

以下事项不得自动降低审查意见：

- 违反法律、行政法规强制性规定。
- 主体不存在、无授权、资质不适格。
- 管辖、仲裁、送达等条款明显无效或不可执行。
- 涉及个人信息、数据出境、劳动用工、金融监管、食品广告等强监管事项。
- 依据 OCR 存疑文本、未读附件、未核验法规案例得出的判断。
- 单次临时让步、价格换风险、特殊客户关系导致的个案接受。

## 九、驳回与回滚

用户不同意提案时，在 `偏好更新提案.md` 中标注“已驳回”，不得再次以同一批样本重复提案；除非后续新增样本或用户重新要求。

用户撤销已确认偏好时，更新 `偏好变更历史.md`，并在 `已确认偏好规则.md` 中标注“已停用”。不得删除历史记录。
