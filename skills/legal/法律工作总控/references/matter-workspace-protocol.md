# 事项工作区协议

本协议用于区分“给用户看的业务文件”和“系统需要记录的内部文件”。所有事项按本协议的新路径建档和读取。

## 一、分层原则

- Excel 总台账只做复盘索引。
- 给用户看的合同、劳动合规材料、产品法务材料、监管合规材料、诉讼材料、交付文书、最终版，放在业务文件区。
- 系统内部需要读取、复查、溯源、OCR 校正、缺口归档、事实时间线、工作笔记的内容，放在 `_系统记录`。
- 当前正在处理的事项由文件记录，不靠模型记忆。
- 一个客户可以有多个合同或多个案件；不得只靠客户名判断当前事项。

## 二、根目录

统一根目录：

```text
【自定义工作目录】/
```

新结构：

```text
【自定义工作目录】/
├── 产品法务/
├── 监管合规/
├── 合同/
├── 劳动合规/
├── 诉讼/
│   ├── 民事/
│   └── 刑事/
└── _系统记录/
    ├── 当前事项.md
    ├── 总索引.md
    ├── 诉讼案件总览.md
    └── 案件复盘台账说明.md
```

如出现非诉、合规、破产等事项，可在业务文件区按需新增同级目录；系统记录仍放入 `_系统记录/<业务类型>/...`。

## 三、业务文件区

业务文件区只放用户可见、便于查找和交付的文件，不放系统内部推理、来源验证、OCR 文本、缺口归档和工作笔记。

### 4.3A 劳动合规与劳动争议诉讼衔接

同一企业客户发生劳动争议、劳动仲裁或诉讼时，诉讼事项仍按 `诉讼/民事/<企业客户>/<案件简称或案号>/` 建档；但可以将该企业的劳动合规档案作为公司端背景资料读取。

读取范围遵循最小必要原则：
- 先读取 `_系统记录/劳动合规/<企业客户>/公司基本信息.md`、`当前状态.md`、`公司材料清单.md`、`制度版本台账.md` 和 `变更记录.md`。
- 根据争议类型再定向读取相关制度：劳动合同模板、员工手册、薪酬制度、考勤制度、绩效制度、奖惩制度、社保福利制度、对应专项事项记录。
- 只把与本争议有关的材料写入诉讼事项的 `材料清单.md`、`事实时间线.md`、`来源验证记录.md` 或 `工作笔记.md`，并标注“来源于公司劳动合规档案”。
- 不得默认读取无关员工、无关专项事项或全量公司内部资料。
- 如劳动合规档案中的制度版本、OCR 文本或记录被校正，应同步提示是否影响当前劳动争议诉讼事项。

## 五、当前事项规则

1. 具体事项任务开始前，先读取 `_系统记录/当前事项.md`。
2. 用户指定新事项或切换事项时，更新 `_系统记录/当前事项.md`。
3. 当前事项不明确时，询问用户。
4. 默认不得读取其他客户或其他事项文件夹。
5. 跨事项比较、复盘或经验迁移，必须由用户明确提出。

## 五A、正式交付硬门槛

凡是法律文书、Word、本地 `.docx` 或其他正式交付文件：

1. 第一动作必须读取 `_系统记录/当前事项.md`。
2. 当前事项与用户任务不匹配时，必须先按本协议新建或切换业务文件区和系统记录区。
3. 正式交付的 `matter_path` 必须位于 `【自定义工作目录】/` 下的业务文件区。
4. 正式交付的 `system_record_path` 必须位于 `【自定义工作目录】/_系统记录/` 下的系统记录区。
5. `.cache`、`/tmp`、下载目录、微信临时目录仅能作为读取或导出的临时中间目录，不得作为正式事项路径、系统记录路径或最终交付目录。
6. `法律文书出稿前审查` 应校验上述路径和当前事项记录；未通过时不得进入正式 Word 导出。

`当前事项.md` 建议格式：

```text
# 当前事项

- 业务类型：合同 / 劳动合规 / 产品法务 / 监管合规 / 诉讼-民事 / 诉讼-刑事 / 咨询 / 其他

- 当前子任务：

- 业务文件路径：
- 系统记录路径：
```

## 六、文件职责

### 案件基本信息.md / 公司基本信息.md / 产品基本信息.md

记录客户、对方、案由/事项、管辖或适用地区、律师角色、关键事实、特殊规则、利益冲突状态等相对稳定信息。劳动合规按公司长期归档，使用 `公司基本信息.md`；产品法务按企业客户和产品/项目归档，使用 `产品基本信息.md`。

### 当前状态.md

记录当前阶段、正在处理的任务、调用的 Skill、下一步动作、关键期限。

### 事件记录.md / 阶段记录.md

记录办案动作和程序进展：收到材料、完成读取、完成检索、生成文书、提交材料、开庭、调解、会见、阅卷、OCR 校正等。

### 材料清单.md

记录材料名称、业务文件路径、系统记录路径、类型、来源、形成时间、读取方式、读取覆盖范围、复查状态、关键字段、存疑项。

### 事实时间线.md

记录从材料和用户口述中提取的案件事实。每条事实必须带来源。

### 程序时间线.md

记录诉讼程序推进：立案、送达、举证、答辩、开庭、调解、裁判、上诉、执行、刑事阶段转换、会见、阅卷等程序事件。每条程序事件必须带来源。

### 期限台账.md

记录开庭、举证、答辩、上诉、缴费、提交材料、会见、阅卷等期限，包含起算日、截止日、来源、风险等级、飞书提醒状态和存疑项。

### 传票与通知台账.md

记录收到的传票、开庭通知、举证通知、应诉通知、缴费通知、裁判文书通知、公安/检察/法院程序通知及其触发的行动事项。

### 案件简报.md

记录内部简报、客户简报和庭前简报的系统版。正式给客户交付的文件应另存业务文件区。

### 组合状态.md

记录单个案件在多案件组合中的位置、风险等级、下一步动作和与其他案件的关联。多案件总览同步到 `_系统记录/诉讼案件总览.md`。

### 飞书同步记录.md

记录飞书日历、任务、文档、思维导图或画板的创建/更新时间、链接或 ID、对应本地记录和来源材料。

### 结案归档记录.md

记录案件关闭检查、未处理期限、未归档材料、客户移交、费用结算提示、飞书提醒处理和最终归档状态。

### 工作笔记.md

记录内部分析、策略思考、待办、用户偏好和不适合放入正式文书的工作内容。

### 来源验证记录.md

记录法规、案例、Wiki、网页搜索、QCC/MCP 等外部来源的检索和验证状态。

### 缺口归档.md

记录材料缺口、事实缺口、验证缺口、影响范围和补充建议。写入缺口时必须同时提示用户。

### OCR校正记录.md

记录 OCR 或读取错误的校正过程和同步更新范围。OCR 文本和校正文只放系统记录区。

## 七、Excel 复盘台账

继续使用：

```text
【客户台账文件】
```

新增或维护 sheet：`案件复盘台账`。

字段：

```text
事项编号
事项名称
客户/委托人
对方当事人
业务类型
案由/事项
当前阶段
事项状态
下一步动作
关键期限
业务文件夹路径
系统记录路径
最后更新时间
备注
```

旧 sheet 保留，不删除旧数据。日常只维护简化台账。

劳动合规台账填写方式：
- `事项编号` 可留空或使用企业简称，不强制生成独立编号。
- `事项名称` 填企业客户劳动合规档案，例如 `某公司劳动合规档案`。
- `业务类型` 填 `劳动合规`。
- `案由/事项` 记录当前子任务摘要，例如 `员工手册修订`、`工资工时问答`、`解除风险预判`。
- 不因每次制度修改或问答新建一条长期事项；具体变更写入该公司系统记录的 `变更记录.md` 和 `专项事项/`。

产品法务台账填写方式：
- `事项编号` 可留空或使用企业简称加产品简称，不强制生成独立编号。
- `事项名称` 填企业客户和产品/项目，例如 `某公司某产品法务`。
- `业务类型` 填 `产品法务`。
- `案由/事项` 记录当前子任务摘要，例如 `默认开启评估`、`客户 Logo 使用`、`产品材料审查`、`医疗产品深度评估`。
- 同一产品的多轮审查和上线变更，优先写入该产品系统记录的 `版本记录.md` 和 `变更记录.md`，不因每次问答新建长期事项。

监管合规台账填写方式：
- `事项编号` 可留空或使用行业/客户简称，不强制生成独立编号。
- `事项名称` 填行业、客户或产品监管合规事项，例如 `数据合规监管动态监测`、`某公司产品监管合规核查`。
- `业务类型` 填 `监管合规`。
- `案由/事项` 记录当前子任务摘要，例如 `新规更新分级`、`客户合规缺口核查`、`政策修改建议`。
- 同一行业或客户的连续监管监测，优先写入该监管合规系统记录的 `监管动态台账.md`、`合规缺口台账.md` 和 `整改跟踪.md`，不因每次监测新建长期事项。

## 八、记录规则

- 收到新材料：材料文件放入业务文件区；同步更新系统记录区的 `事件记录.md` 和 `材料清单.md`。
- 劳动合规收到新材料：材料文件放入该公司劳动合规业务文件区；同步更新 `_系统记录/劳动合规/<企业客户>/事件记录.md` 和 `公司材料清单.md`。
- 劳动合规制度变更：同步更新该公司的 `制度版本台账.md`、`变更记录.md`、`制度冲突检查.md` 和对应 `专项事项/`。
- 产品法务收到新材料：材料文件放入对应产品法务业务文件区；同步更新 `_系统记录/产品法务/<企业客户>/<产品或项目>/事件记录.md` 和 `材料清单.md`。
- 产品功能、页面、协议、隐私政策、营销物料或上线版本变更：同步更新该产品项目的 `版本记录.md`、`变更记录.md`、`功能风险台账.md`、`数据处理与授权记录.md` 或 `营销审查记录.md`。
- 监管合规收到新材料：材料文件放入对应监管合规业务文件区；同步更新 `_系统记录/监管合规/<行业或企业客户>/客户材料清单.md` 和 `监管对象画像.md`。
- 发现监管更新或完成法规核验：同步更新 `监管动态台账.md`、`法规政策卡片/`、`适用规则清单.md`、`来源验证记录.md` 和 `北大法宝核验记录.md`。
- 发现客户合规缺口或提出整改建议：同步更新 `合规缺口台账.md`、`整改跟踪.md` 和 `缺口归档.md`。
- 诉讼案件收到传票、通知或裁判文书：同步更新 `传票与通知台账.md`、`期限台账.md`、`程序时间线.md` 和 `当前状态.md`。
- 诉讼案件产生明确期限：同步更新 `期限台账.md`；需要提醒时调用飞书日历，并更新 `飞书同步记录.md`。
- 诉讼案件生成时间线、思维导图或飞书文档：本地事实源仍为 `事实时间线.md`、`程序时间线.md` 和 `材料清单.md`，飞书链接写入 `飞书同步记录.md`。
- 诉讼案件关闭或结案检查：同步更新 `结案归档记录.md`、`组合状态.md` 和 `_系统记录/诉讼案件总览.md`。
- 提取新事实：更新系统记录区的 `事实时间线.md`。
- 完成法规/案例/Wiki 检索：更新系统记录区的 `来源验证记录.md`。
- 材料不足：提示用户并更新系统记录区的 `缺口归档.md`。
- OCR 校正：更新系统记录区的 `OCR校正记录.md`，并按影响范围同步其他系统记录 md。
- 正式文书、合同终稿、审查报告、给客户看的摘要：放入业务文件区对应交付或最终版目录。
- 正式文书已生成后，如校正影响文书内容，提示用户是否同步修订，不自动改文书。

## 九、旧文件迁移记录

旧文件迁移记录保存在 `【自定义工作目录】/_系统记录/迁移执行记录.md` 和 `【自定义工作目录】/_系统记录/总索引.md`。

新合同、新产品法务事项、新监管合规事项、新诉讼事项、新系统记录，均按本协议的新双路径结构创建。
