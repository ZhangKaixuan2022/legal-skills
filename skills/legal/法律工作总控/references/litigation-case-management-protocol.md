# 诉讼案件管理共享协议

本协议只处理诉讼案件的运营管理动作，不重复定义案件隔离、建档、材料读取复查、来源披露、OCR 校正、缺口归档和法规核验；上述事项仍由法律工作总控其他共享协议处理。

## 一、适用范围

出现以下任务时，路由至 `诉讼案件管理`：

- 案件更新、程序进展、收到新文书或新通知。
- 传票、开庭通知、举证通知、应诉通知、缴费通知、裁判文书、公安/检察通知处理。
- 期限台账、开庭提醒、举证/答辩/上诉/缴费/提交材料期限、飞书日历提醒。
- 事实时间线、程序时间线、证据链、争议焦点或刑事阶段路线图的飞书展示。
- 案件简报、诉讼案件总览、案件组合状态、案件关闭和结案归档检查。

民事实体分析、刑事辩护策略、劳动争议专业判断、诉讼文书起草和庭审准备，仍路由对应专业 Skill。

## 二、系统记录文件

民事和刑事诉讼事项的系统记录区应补充以下管理文件；劳动争议仲裁/诉讼按事项工作区协议归入 `诉讼/民事/` 时同样适用：

```text
程序时间线.md
期限台账.md
传票与通知台账.md
案件简报.md
组合状态.md
飞书同步记录.md
结案归档记录.md
```

全局诉讼案件总览使用：

```text
【自定义工作目录】/_系统记录/诉讼案件总览.md
```

## 三、传票与通知处理

收到传票、通知、裁判文书或公安/检察机关程序文书时：

1. 先按文件读取规范读取并复查文书。
2. 抽取案号、机关、送达/签收日期、开庭/到场时间、地点、期限和待办事项。
3. 更新 `传票与通知台账.md`、`期限台账.md`、`程序时间线.md` 和 `当前状态.md`。
4. 有明确时间或截止日的，调用飞书日历创建提醒。
5. 将飞书链接或未创建原因写入 `飞书同步记录.md`。

时间、地点、案号、起算日或 OCR 字段存疑时，不创建飞书提醒；必须提示用户确认，并写入 `缺口归档.md`。

## 四、期限与飞书提醒

期限必须有来源：文书原文、用户确认、法院/机关通知、法规核验或已验证系统记录。

默认提醒：
- 开庭/到场：提前 7 天、3 天、1 天、当天。
- 举证/答辩/上诉/缴费/提交材料：提前 7 天、3 天、1 天。
- 刑事会见、阅卷、庭前会议：按文书或用户确认时间创建。

创建日历前读取 `lark-calendar` 和 `lark-shared` 规则；创建后更新 `飞书同步记录.md`。

## 五、时间线与飞书展示

本地系统记录是事实源，飞书文档或画板只是展示视图。

- `事实时间线.md` 记录案件事实，每条事实必须带来源。
- `程序时间线.md` 记录程序推进、机关/法院、文件、影响和下一步。
- 用户明确要求“案件可视化图表”“案件思维导图”“案件全套图表”或类似表达时，转入 `诉讼可视化`，默认输出全套案件图表；本协议只负责提供事实时间线、程序时间线和材料清单作为事实源。
- 需要展示时，优先生成面向用户阅读的图片化图表，再用 `lark-doc` 保存为飞书文档；不得只交付 Mermaid 代码。
- 图表类型按可读性选择：事实经过优先时间轴或泳道图，人物/主体优先关系图，争点结构优先树状图，程序推进优先流程图。
- 思维导图仅适合层级少、节点短的结构；节点过多或文字过长时必须拆成多张图，或改用树状图/泳道图/分组流程图。
- 需要飞书画板时使用 `lark-whiteboard`；覆盖已有画板前必须按其规则 dry-run 并确认。
- 飞书链接写入 `飞书同步记录.md`。

## 六、案件简报

案件简报分为内部简报、客户简报和庭前简报。生成简报前必须读取当前事项系统记录，并在输出前说明：

- 已读取材料。
- 已核验内容。
- 未核验/存疑内容。
- 法规案例检索状态。
- 需要客户判断事项。

简报保存到系统记录区 `案件简报.md`；给客户看的正式交付文件另存业务文件区。

## 七、组合状态

多案件总览或组合状态使用 `【自定义工作目录】/_系统记录/诉讼案件总览.md`。

每个案件至少记录：客户/委托人、案件名称、类型、案号、当前阶段、下一期限、风险等级、下一步动作、业务文件路径、系统记录路径、最后更新时间。

## 八、案件关闭

案件关闭时先做结案检查，再更新状态：

1. 核对是否存在未处理期限、未归档材料、未完成飞书提醒、未同步的 OCR 校正或缺口。
2. 更新 `当前状态.md`、`结案归档记录.md`、`组合状态.md` 和全局 `诉讼案件总览.md`。
3. 正式结案报告、客户移交文件和归档文书仍交给对应专业结案 Skill 或业务文件区处理。

## 九、专业 Skill 分工

- 民事阶段、诉讼文书、庭前庭审、判后分析：`民事一审诉讼` 及其子 Skill。
- 刑事阶段、会见阅卷、侦查/审查起诉/一审/二审辩护：`刑事辩护总调度` 及其子 Skill。
- 劳动争议仲裁/诉讼专业判断：`劳动争议诉讼` 及其子 Skill。
- 传票通知、期限台账、飞书提醒、程序时间线、案件简报、案件总览和关闭检查：`诉讼案件管理`。
