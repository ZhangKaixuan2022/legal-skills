# OCR 校正协议

## 核心原则

- 不修改原始 PDF、图片或证据原件。
- OCR 原文和校正文分开保存。
- 用户指出 OCR 错误后，必须沉淀到校正文，下次优先引用校正文。
- OCR 校正影响事实、材料清单或来源记录时，必须同步更新受影响记录。

## 文件存放

新事项统一放在系统记录区：

```text
_系统记录/
  <业务类型>/
    <客户或项目>/
      <事项名称>/
        OCR校正记录.md
        OCR文本/
          原文件名.ocr.md
          原文件名.corrected.md
```

OCR 校正统一写入系统记录区。

## 引用优先级

1. 原生可读文本、Word、Excel 内容。
2. 已校正 OCR：`原文件名.corrected.md`。
3. 原始 OCR：`原文件名.ocr.md`。
4. 重新 OCR。
5. 用户确认。

如果校正文与原文件可读文本冲突，提示用户确认。

## 校正记录格式

```markdown
## YYYY-MM-DD
- 文件：
- 原文件路径：
- 位置：
- OCR原识别：
- 校正后：
- 校正来源：
- 影响：
```

## 同步更新规则

用户指出 OCR 错误后，按影响范围同步：

- 影响材料关键字段：更新 `材料清单.md`。
- 影响案件事实：更新 `事实时间线.md`。
- 影响来源引用：更新 `来源验证记录.md`。
- 影响办案动作：追加 `事件记录.md`。
- 影响已生成文书或报告：提示用户是否同步修订，不自动修改正式文书。

## 不同影响的处理

```text
OCR校正影响事实 → 更新事实时间线
OCR校正只影响错别字 → 只更新校正文和校正记录
OCR校正影响文书/分析结论 → 提示用户确认后再修订文书
```

## 事件记录示例

```markdown
## 2026-05-16 — OCR校正同步
- 文件：租赁合同.pdf
- 校正内容：付款金额由“26703元”更正为“267031元”
- 已同步：材料清单.md、事实时间线.md、来源验证记录.md
- 需用户判断：是否同步修订已生成起诉状/法律服务建议书
```
