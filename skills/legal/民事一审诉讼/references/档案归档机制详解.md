# 档案归档机制详解

> **全局强制规则**：本文档仅作为历史字段映射和系统记录同步参考。当前执行以法律工作总控为准：先更新当前事项系统记录；复盘台账仅在用户确认或总控规则要求时同步。

## 一、两步归档机制

### 职责说明

- **S0→S1的推进**由**主Skill（civil-litigation-agent）自身**完成两步归档（系统记录归档 + 文件夹创建）
- 委托合同管理负责S3阶段的委托合同签订和跟踪表生成，不重复执行系统记录/文件夹创建

---

## 二、第一步：系统记录归档（优先）

### 目标文件
当前事项系统记录；复盘台账按总控确认后同步

### 基础字段（主流程S0→S1创建）

| 字段名 | 说明 | 初始值 |
|-------|------|-------|
| 客户编号 | 格式：客户名称_联系方式 | 用户输入 |
| 客户姓名/单位名称 | | 用户输入 |
| 联系方式 | | 用户输入 |
| 首次咨询日期 | | 当前日期 |
| 案件状态 | 各Skill更新 | 已咨询 |
| 承办律师 | | 潘睿 |
| 案件文件夹路径 | | 创建后填写 |
| 备注 | | |

## 五、第二步：文件夹归档

### 触发条件
系统记录归档完成后，检查客户文件夹是否存在

### 基础路径
`<当前事项业务文件路径>/`

### 如文件夹不存在，自动创建标准档案结构
详见 [民事一审Agent前置文档检查表.md](民事一审Agent前置文档检查表.md) 中的档案结构说明

### 归档完成后更新系统记录
在当前事项系统记录中更新业务文件路径；复盘台账是否同步由法律工作总控确认。

## 六、字段自动新增机制

系统记录缺少上述字段时补充；复盘台账字段变更需先确认。

操作逻辑：
1. 读取当前事项系统记录字段
2. 检查是否存在目标字段
3. 系统记录不存在则补充字段
4. 写入字段值
