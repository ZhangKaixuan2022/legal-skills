## 二、前置文档检查逻辑

```
调用Skill前 → 检查所需前置文档
    ↓
文档齐全 → 继续执行
    ↓
文档缺失 → 分析缺失原因
    ↓
    ├─ Skill衔接问题（前置Skill未生成文档）
    │   → 报告用户："检测到XX Skill未生成输出文档，请检查该Skill执行是否正常"
    │   → 配合用户修复Skill逻辑
    │
    └─ 用户未提供（需要外部材料）
        → 询问用户："需要提供XX材料，请提供路径或内容"
```

## 四、文档缺失处理策略

### 策略一：Skill衔接问题
```
检测到：【Skill名称】未生成预期输出文档
可能原因：
1. 该Skill执行中断
2. 文档生成路径不正确
3. Skill未被正确调用

建议操作：
→ 重新调用该Skill
→ 检查Skill执行日志
→ 手动指定文档路径
```

### 策略二：用户未提供材料
```
缺失文档：【文档名称】
该文档需要用户提供：
□ 请上传文件
□ 请提供文件路径
□ 请口述相关内容（我将协助整理）

是否继续不读取该文档？[是/否]
```
