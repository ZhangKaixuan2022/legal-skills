## 十、排版实现要点

### 10.1 对话区前缀格式

对话区"谈话律师："和"当事人："使用**独立run + 黑体加粗**实现：
- Run[0]：前缀标签（"谈话律师："或"当事人："），字体设为黑体，加粗（w:b + w:bCs）
- Run[1]：发言内容，字体设为宋体，不加粗

**区别于信息区**：信息区段4"谈话律师："和段6"当事人："为宋体不加粗，属于信息标签而非对话标签。

### 10.2 告知事项列表 vs 风险提示列表

两处编号列表格式不同：

| 特征 | 告知事项（段35-47） | 风险提示（段57-69） |
|------|-------------------|-------------------|
| 首行缩进 | 无 | firstLine=480（2字符） |
| Run结构 | 1个run（整体） | 3个run（序号+标题+描述） |
| 风险标题加粗 | 无 | 黑体加粗 |
| 分隔符 | 无 | 破折号" — " |

### 10.3 空行规范

文档中使用空行（空段落）作为视觉分隔：
- 标题后1个空行
- 信息区后1个空行
- 每轮问答之间1个空行
- 告知事项引言后1个空行
- 告知事项确认后2个空行
- 每个分节标题后1个空行
- 签章确认声明前1个空行
- 签章行前1个空行
- 签章行与日期行之间1个空行

**空行格式**：空段落（无run或仅有1个空run），行距1.5倍。

### 10.4 手写编号

告知事项和风险提示均使用**手写编号**（如"1. ""2. "...），不使用Word的numPr自动编号机制。

### 10.5 右对齐签章/日期

签章行和日期行使用**右对齐（jc=right）**定位，无缩进。

签章行下划线使用11个下划线字符（_），非w:u下划线属性。

日期行格式：6个半角空格 + 年 + 3个半角空格 + 月 + 3个半角空格 + 日。

### 10.6 分节标题间距

"整体案情总结"和"风险提示"分节标题的间距：
- 段前间距：12 pt（before=240 twips）
- 段后间距：6 pt（after=120 twips）
- 与正文形成视觉分层
