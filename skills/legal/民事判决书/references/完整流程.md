# 民事判决书Skill

## 功能概述

从已有案件材料中提取要素，生成一审普通程序民事判决书Word初稿。适用场景包括：
- **模拟判决书**：基于已有材料模拟法院裁判思路，生成判决书初稿
- **起草判决书**：在案件审结后，根据庭审结果和认定事实撰写正式判决书
- **教学/演练用途**：用于模拟法庭演练后的判决书撰写

支持与复盘台账衔接，读取当前事项系统记录并按需同步判决书路径。

## 排版格式规范

判决书排版与诉讼文书不同，使用仿宋三号/行距28磅，而非宋体小四/1.5倍行距。

> 详细排版标准参见 [references/判决书排版规范.md](判决书排版规范.md)
>
> 文书制作规范（当事人信息、审理经过、诉辩意见、事实认定、裁判理由、数字用法、标点符号、引用规范等）以 [references/民事裁判文书制作规范.md](民事裁判文书制作规范.md) 为权威标准

## 工作流程

### 步骤1：读取系统记录/复盘台账

**数据来源**：复盘台账（按法律工作总控确认后读取/更新）

**读取字段**：
| 字段 | 用途 |
|-----|------|
| 客户称呼 | 识别客户 |
| 案件文件夹路径 | 定位案件材料目录 |
| 法院名称 | 填写首部法院名称 |
| 案号 | 填写案号 |
| 案由 | 填写案由 |
| 当事人信息 | 填写原告/被告/第三人信息 |
| 诉讼代理人信息 | 填写代理人信息 |

**操作指令**：
1. 根据当前事项系统记录定位事项；必要时参考复盘台账
2. 读取上述字段
3. 确认案件文件夹路径存在且可访问

### 步骤2：扫描案件文件夹，识别已有文档

**案件文件夹扫描路径**：
```
<当前事项业务文件路径>/
├── 01-委托阶段/
│   ├── 案件沟通记录/
│   └── 法律服务建议书/
├── 02-诉前准备/
│   ├── 起诉状/
│   ├── 答辩状/
│   ├── 法规案例汇编/
│   └── 诉讼可视化/
├── 03-诉讼阶段/
│   ├── 代理词/
│   ├── 质证意见/
│   └── 庭审纪要/
├── 04-庭审阶段/
│   ├── 庭审纪要/
│   ├── 庭审复盘/
│   └── 案件讨论与提纲/
└── 05-裁判阶段/
    └── 判决书/
```

**识别以下文档（如存在）**：
| 文档类型 | 典型文件名 | 提取要素 |
|---------|-----------|---------|
| 起诉状 | 民事起诉状_*.docx/pdf | 原告诉请、事实与理由、当事人信息 |
| 答辩状 | 民事答辩状_*.docx/pdf | 被告答辩意见 |
| 法律服务建议书 | 法律服务建议书_*.docx/pdf | 法律关系定性、请求权基础、法律依据 |
| 初步法律分析 | 法律分析_*.docx/pdf | 争议焦点、法律适用意见 |
| 庭审纪要 | 庭审纪要_*.docx/pdf | 争议焦点、双方主张、证据质证情况 |
| 庭审复盘 | 庭审复盘_*.docx/pdf | 裁判思路、可能判项 |
| 案件讨论与提纲 | 案件讨论_*.docx/pdf | 裁判思路、判项草案 |
| 模拟法庭 | 模拟法庭_*.docx/pdf | 模拟裁判结果、判项 |

**读取规范**：
- 逐份文档完整读取全部内容
- PDF扫描件使用OCR识别，精度不足时标注 `[OCR识别不确定]`
- 禁止部分读取或仅提取摘要

### 步骤3：按7个区块逐一提取要素，标记缺失项

#### 区块1：首部

| 要素 | 数据来源 | 是否必填 |
|-----|---------|---------|
| 法院名称 | 系统记录/复盘台账 / 起诉状 | 是 |
| 文书名称 | 固定为"民事判决书" | 是 |
| 案号 | 系统记录/复盘台账 / 起诉状 | 是 |

**缺失处理**：法院名称、案号缺失时标记为【待补充】，进入步骤4向用户确认。

#### 区块2：当事人信息

**通用规则**：
1. **诉讼地位在前，姓名/名称在后，中间用冒号；姓名/名称后用逗号**
2. 全部诉讼参加人均分行写明

**自然人**（以居民身份证、户籍证明为准）：
- 姓名、性别、出生年月日、民族、职业或工作单位和职务、住所
- 职业或工作单位和职务不明确的，可以不表述
- 住所以户籍所在地为准；离开户籍所在地有经常居住地的，经常居住地为住所
- 连续两个当事人的住所相同的，应当分别表述，**不用"住所同上"**

**法人**：
- 名称、住所；另起一行写明法定代表人的姓名和职务

**其他组织**：
- 名称、住所；另起一行写明负责人的姓名和职务
- 主要办事机构所在地不明确的，注册地或登记地为住所

**个体工商户**：
- 有字号的：以营业执照登记的字号为当事人，并写明该字号经营者的基本信息（姓名、性别、出生年月日、民族、住所）
- 无字号的：写经营者信息

**个人合伙**（起字号的）：
- 在姓名之后用括号注明"系……（写明字号）合伙人"

**外国人**：
- 写明经过翻译的中文姓名，并用括号注明外文姓名和住所
- 外国自然人注明国籍全称；无国籍人注明"无国籍"
- 基本信息以护照等身份证明文件为准

**港澳台居民**：
- 姓名后写明"香港特别行政区居民""澳门特别行政区居民"或"台湾地区居民"
- 住所冠以"香港特别行政区""澳门特别行政区""台湾地区"

**曾用名**：
- 与本案有关联的，在现用名后用括号注明曾用名

**姓名/名称变更**：
- 列变更后的姓名或名称；变更前名称在查明事实部分写明

**权利义务继受**：
- 继受人为当事人；被继受人在案件由来中写明继受事实

**代表人诉讼**：
- 人数众多的，表述为"原告×××等×人（名单附后）"

**名称规范**：
- 法人、其他组织、个体工商户、个人合伙的名称写全称，以注册登记文件为准

| 要素 | 自然人 | 法人/非法人组织 | 数据来源 |
|-----|-------|----------------|---------|
| 原告姓名/名称 | 必填 | 必填 | 系统记录 / 起诉状 |
| 性别 | 必填 | 不适用 | 系统记录 / 起诉状 |
| 出生日期 | 必填 | 不适用 | 系统记录 / 起诉状 |
| 民族 | 有则填 | 不适用 | 系统记录 / 起诉状 |
| 职业/工作单位 | 有则填 | 不适用 | 系统记录 / 起诉状 |
| 住址/住所地 | 必填 | 必填 | 系统记录 / 起诉状 |
| 法定代表人/主要负责人 | 不适用 | 必填 | 系统记录 / 起诉状 |
| 被告信息 | 同上原告字段 | 同上原告字段 | 系统记录 / 起诉状 / 答辩状 |
| 第三人信息 | 同上原告字段 | 同上原告字段 | 如有则填 |

**缺失处理**：当事人基本信息缺失时标记为【待补充】，进入步骤4向用户确认。

#### 新增区块：委托诉讼代理人

**通用规则**：
1. 在当事人之后另起一行写明"委托诉讼代理人"
2. 委托诉讼代理人后用冒号，再写姓名；姓名后用逗号
3. 有两个委托诉讼代理人的，分行分别写明

**排列顺序**：
- 近亲属或本单位工作人员列第一位
- 律师或外单位人员列第二位

**本单位人员**：
- 姓名、性别，身份信息表述为"该单位（如公司、机构、委员会、厂等）工作人员"

**律师/基层法律服务工作者**：
- 姓名，所在律师事务所/法律服务所名称及执业身份
- 表述为"××律师事务所律师""××法律服务所法律工作者"
- 属于法律援助的，应当写明法律援助情况

**近亲属**：
- 姓名后用括号注明其与当事人的关系，写明住所

**社区/单位/社会团体推荐的公民**：
- 姓名、性别、住所，并在住所之后注明具体由何社区、单位、社会团体推荐

**代理人变更**：
- 裁判文书首部只列写变更后的委托诉讼代理人
- 变更事实可根据需要在案件由来中写明

#### 区块3：审理经过

**诉讼地位规则**：
1. 一审表述为"原告""被告""第三人"
2. 有多个原告、被告、第三人的，按起诉状列明的顺序写
3. 提出反诉的，在本诉称谓后用括号注明"原告（反诉被告）""被告（反诉原告）"
4. 先写原告，后写被告，再写第三人

**简称规则**：
- 诉讼参加人名称过长的，在案件由来部分第一次出现时用括号注明简称"（以下简称×××）"
- 简称应当规范，能准确反映名称特点

**案件来源写法**（8种）：
1. **新收/重新起诉**：写明起诉人
2. **发回重审**："本院于××××年××月××日作出……（写明案号）民事判决。×××不服该判决，向××××法院提起上诉。××××法院于××××年××月××日作出……（写明案号）裁定，发回重审。本院依法另行组成合议庭……"
3. **指定管辖**：写明报请上级人民法院指定管辖日期及上级法院作出管辖裁定日期
4. **移送管辖**：写明原审法院作出裁判的案号及日期、上诉人、上级法院作出裁判的案号及日期
5. 上级法院指令受理、指定审理、提级管辖的，参照发回重审格式写明原审及上诉情况

**程序转换**（简易→普通）：
- "于××××年××月××日公开/因涉及……不公开开庭审理了本案，经审理发现有不宜适用简易程序（小额诉讼程序）的情形，裁定转为普通程序，于××××年××月××日再次公开/不公开开庭审理了本案"

**中止诉讼**：
- "因……（写明中止诉讼事由），于××××年××月××日裁定中止诉讼，××××年××月××日恢复诉讼"

**不公开开庭的5种情形**：
1. 因涉及国家秘密不公开开庭
2. 因涉及个人隐私不公开开庭
3. 因涉及商业秘密，经当事人申请，决定不公开开庭
4. 因离婚，经当事人申请，决定不公开开庭
5. 法律另有规定的

**庭审情况表述**：
- 立案日期："本院于××××年××月××日立案后"
- 开庭："本院于××××年××月××日公开/因涉及……（写明不公开开庭的理由）不公开开庭审理了本案，原告×××及其诉讼代理人×××，被告×××及其诉讼代理人×××等到庭参加诉讼"
- 未出庭：写明经传票传唤，无正当理由拒不到庭或未经法庭许可中途退庭的情况

| 要素 | 数据来源 | 是否必填 |
|-----|---------|---------|
| 立案日期 | 系统记录 / 起诉状 | 是 |
| 适用程序 | 固定为"普通程序" | 是 |
| 案由 | 系统记录 / 起诉状 | 是 |
| 是否公开开庭 | 庭审纪要 / 默认"公开" | 是 |
| 不公开理由 | 庭审纪要（如涉及隐私等） | 否 |
| 到庭当事人 | 庭审纪要 | 是 |
| 未到庭说明 | 庭审纪要 | 否 |
| 案件来源 | 起诉状 / 立案信息 | 是 |
| 程序转换 | 庭审纪要 | 否 |
| 中止诉讼 | 庭审纪要 | 否 |

#### 区块4：诉辩意见

**写作规则**：
1. **禁止照抄**：不照抄起诉状、答辩状、代理词原文，应全案考虑当事人在法庭上的诉辩意见和提供的证据综合表述
2. **顺序**：先写明诉讼请求，再写事实和理由；按原告、被告、第三人的顺序依次表述
3. **变更诉请**：当事人在法庭辩论终结前变更诉讼请求或提出新的请求的，在诉称部分中写明
4. **承认事实**：被告承认原告主张的全部事实的，写明"×××承认×××主张的事实"；承认部分事实的，写明"×××承认×××主张的……事实"
5. **承认诉请**：被告承认全部诉讼请求的，写明"×××承认×××的全部诉讼请求"；承认部分诉讼请求的，写明承认的具体内容

| 要素 | 数据来源 | 是否必填 |
|-----|---------|---------|
| 原告诉讼请求 | 起诉状 | 是 |
| 原告事实与理由 | 起诉状 | 是 |
| 被告答辩意见 | 答辩状 / 庭审纪要 | 是 |
| 第三人陈述意见 | 庭审纪要 / 如有第三人 | 否 |

#### 区块5：事实认定

**证据写作规则**：
1. 当事人举证质证一般情况表述为："本案当事人围绕诉讼请求依法提交了证据，本院组织当事人进行了证据交换和质证。"
2. **证据原则上不一一列明**，可以附录全案证据或证据目录
3. 对当事人无异议的证据："对当事人无异议的证据，本院予以确认并在卷佐证"
4. 对有争议的证据：写明争议的证据名称及人民法院对争议证据认定的意见和理由
5. **法院调取的证据、鉴定意见**：经庭审质证后，按当事人是否有争议分别写明
6. **逾期提交的证据、非法证据不予采纳的**：应当说明理由
7. **证据认定和事实认定**：可以合并写，也可以分开写。分开写的，在证据审查认定之后，另起一段概括写明："根据当事人陈述和经审查确认的证据，本院认定事实如下：……"

**事实叙述规则**：
1. 认定的事实应当重点围绕当事人争议的事实展开
2. 一般按时间先后顺序叙述，或者对法律关系或请求权认定相关的事实着重叙述，其他事实可归纳、概括叙述
3. 可用"另查明"为引语叙述其他相关事实
4. **争议焦点**：有证据和事实内容的，可在当事人诉辩意见之后在争议证据和事实中写明；主要是法律适用问题的，可在本院认为部分先写明
5. **涉外案件**：应叙述查明外国法的事实

| 要素 | 数据来源 | 是否必填 |
|-----|---------|---------|
| 证据交换情况 | 庭审纪要 | 是 |
| 质证情况 | 庭审纪要 | 是 |
| 无异议证据认定 | 庭审纪要 | 是 |
| 有争议证据认定 | 庭审纪要 | 是 |
| 事实认定要点 | 庭审纪要 / 法律分析 | 是 |

**写作格式**：
```
当事人围绕诉讼请求依法提交了证据，本院组织当事人进行了证据交换和质证。对当事人无异议的证据，本院予以确认并在卷佐证。对有争议的证据和事实，本院认定如下：
1. 【事实认定要点1】；
2. 【事实认定要点2】；
……
```

#### 区块6：裁判理由与主文

**说理规则**：
1. 以"本院认为"开头，其后直接写具体意见（"本院认为"后用逗号）
2. 围绕争议焦点逐一进行分析论证，层次明确
3. **案由错误**：经审理认为立案案由不当的，以经审理确定的案由为准，但应在本院认为部分予以说明并阐明理由
4. 最后另起一段以"综上所述"引出，对当事人的诉讼请求是否支持进行评述

**法律引用规则**：
1. **法条引用格式**：准确完整写明法律名称、条款项序号和条文内容；引用法律条款中的项的，一律使用汉字不加括号（如"第一项"）；裁判文书后附相关条文
2. **引用排序**：法律及法律解释→行政法规→地方性法规→司法解释；同时引用两部以上法律的，先引用基本法律，后引用其他法律；同时引用实体法和程序法的，先引用实体法，后引用程序法
3. **指导性案例**：正在审理的案件与最高人民法院颁布的指导性案例相类似的，应当将指导性案例作为裁判理由引述，写明编号和裁判要点，但**不作为裁判依据引用**
4. **司法指导性文件**：体现的原则和精神可在理由部分予以阐述或援引
5. **禁止引用**：宪法、各级人民法院关于审判工作的指导性文件、会议纪要、各审判业务庭的答复意见、人民法院与有关部门联合下发的文件**不得作为裁判依据**（但其体现的原则和精神可在说理部分阐述）
6. **规范性文件冲突**：确需引用的规范性文件之间存在冲突，依法无法选择适用的，应依法提请有决定权的机关作出裁决，**不得自行在裁判文书中认定相关规范性法律文件的效力**

**裁判主文规则**：
1. 当事人名称使用全称
2. 内容必须明确、具体、便于执行
3. 多名当事人承担责任的，写明各当事人承担责任的形式、范围
4. 有多项给付内容的，先写明各项目的名称、金额，再写明累计金额。如："交通费……元、误工费……元、……，合计……元"
5. 当事人互负给付义务且内容相同的，另起一段写明抵付情况
6. 对于金钱给付的利息，应当明确利息计算的起止点、计息本金及利率

## 文书制作规范细则（数字·标点·引用）

以下内容依据《人民法院民事裁判文书制作规范》第五至第七部分制定，判决书生成时必须严格执行。

### 数字用法规则

1. **裁判主文序号**：使用汉字数字，例："一""二"
2. **落款日期**：使用汉字数字，例："二〇二六年五月二日"
3. **案号**：使用阿拉伯数字，例："(2025)吉0122民初3172号"
4. **其他数字用法**：按照《中华人民共和国国家标准GB/T15835－2011出版物上数字用法》执行

### 标点符号规则

1. **逗号**："被告辩称""本院认为"等词语之后用逗号
2. **冒号**："×××向本院提出诉讼请求""本院认定如下""判决如下""裁定如下"等词语之后用冒号
3. **顿号**：裁判项序号后用顿号
4. **其他标点**：除上述明确要求外，按照《中华人民共和国国家标准GB/T15834－2011标点符号用法》执行

### 引用规范

1. **法律法规名称**：引用法律、法规、司法解释应书写全称并加书名号
2. **简称规则**：法律全称太长的可以简称，简称不使用书名号。可以在第一次出现全称后使用简称，例："《中华人民共和国民事诉讼法》(以下简称民事诉讼法)"
3. **条款序号**：引用法律、法规和司法解释条文有序号的，书写序号应与法律正式文本中的写法一致
4. **公文引用**：引用公文应先用书名号引标题，后用圆括号引发文字号；引用外文应注明中文译文

### 步骤5：组装判决书 → 出稿前审查 → 统一导出 → 校验 → 保存 → 系统记录同步

#### 5.1 组装判决书文本

按照7个区块结构组装完整判决书文本，使用 [templates/一审普通程序民事判决书.md](../templates/一审普通程序民事判决书.md) 作为结构模板。

#### 5.2 生成语义 HTML 并进入出稿前审查

先将判决书正文组装为 `draft.html`，同时生成 `preflight-meta.json`，再调用 `法律文书出稿前审查`。只有审查状态为 `PASS` 或 `FIXED_PASS` 时，才进入 `法律文书模板与导出` 统一生成 `.docx`。

#### 5.3 文档校验

按 `法律文书模板与导出` 的 `health_check.py` 执行结构体检。校验不通过时，不得标记为正式交付；先修复导出问题，修复失败则保留 `draft_checked.html`、审查报告和错误摘要。

#### 5.4 文件保存

**保存路径**：
```
<当前事项业务文件路径>/民事判决书_【YYYYMMDD】.docx
```

如目录不存在，按法律工作总控确认后的当前事项业务文件区创建；不得保存到未确认事项路径。

#### 5.5 系统记录同步

**回写路径**：复盘台账（按法律工作总控确认后读取/更新）

**回写字段**：
| 字段名 | 回写内容 |
|-------|---------|
| 判决书文件路径 | <当前事项业务文件路径>/民事判决书_【YYYYMMDD】.docx |
| 判决书生成时间 | 当前日期时间 |
| 文书类型 | "民事判决书" |

**字段维护**：系统记录缺少上述字段时补充；复盘台账字段变更需先确认。

**回写异常处理**：
- 如当前事项系统记录不存在，提示建立或切换当前事项
- 如客户编号未能定位当前事项，提示用户核实
- 回写失败时保存备份并提示用户

> **强制规则**：系统记录同步必须按法律工作总控确认后的当前事项和记录路径执行；涉及复盘台账字段变更时，先向用户说明更新内容，不得静默写入。

## 注意事项

1. **判项不可编造**：AI只能根据已有材料整理判项草案，最终判项必须经用户确认
2. **审判人员信息不可编造**：审判长、审判员、书记员姓名必须用户提供
3. **诉讼费必须准确**：案件受理费金额和分担比例必须用户提供
4. **法律依据需校验**：引用法条必须通过网络检索确认准确性和时效性
5. **日期格式统一**：裁判日期使用"××××年××月××日"格式（汉字数字）
6. **案号格式**：按（年份）+法院简称+民初+序号，使用阿拉伯数字
7. **标点规范**："本院认为"后用逗号，"判决如下"后用冒号，判项序号后用顿号
8. **法条引用**：准确完整写明法律名称、条款项序号和条文内容，项用汉字不加括号
9. **禁止引用**：宪法、会议纪要、答复意见、联合发文不得作为裁判依据

## Wiki 知识库查阅规则

当处理的案件涉及具体罪名或法律领域时，必须查阅 Wiki 知识库：

1. **检查知识库**：检查 `~/wiki/wiki/` 下是否有对应知识页（刑事：`books/criminal/{罪名}.md`，民事：`books/civil/{领域}/{具体问题}.md`）
2. **读取知识卡片**：若有对应页面，读取页面顶部 `<!-- CARD_START -->` 到 `<!-- CARD_END -->` 之间的知识卡片（约800字浓缩摘要）
3. **按需深入**：卡片信息足够则直接引用；需要深入了解具体裁判策略、案例细节、司法解释原文时，读取全文对应板块
4. **融入输出**：将 Wiki 知识自然融入当前工作输出，引用时标注来源页面路径
5. **案例知识页**：同时检查 `~/wiki/wiki/cases/` 下是否有相关判例知识页可供参考
