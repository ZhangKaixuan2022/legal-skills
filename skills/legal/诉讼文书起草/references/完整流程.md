# 诉讼文书起草Skill

## 功能概述

运用"六来源体系"和请求权基础分析框架，起草各类诉讼文书，包括：
- **起诉状**：按六来源逻辑组织诉讼请求，采用请求权递进结构陈述事实与理由
- **答辩状**：提供否认、阻却、抵消、反诉四类策略框架
- **代理词**：采用六维分析框架（选题、格式、语言、说理、法律、社会效果）
- **质证意见**：六类标准质证模板，按"关联性→合法性→真实性→证明力"顺序
- **证据目录**：按请求权要件逻辑排序，含证明对象与要件对应关系
- **保全申请书**：诉讼财产保全标准格式

支持与复盘台账衔接，读取前序法律分析成果并按需同步文书路径。

## 排版格式规范（方案五）

严格执行法院要求的排版标准，包括纸张规格、页边距、字体字号、编号层级等要求。

> 📋 详细排版标准参见 [references/文书排版格式规范.md](文书排版格式规范.md)

### 起诉状与答辩状格式差异

起诉状和答辩状在标题、编号方式、文风结构上存在明显区别，需按各自规范要求起草。

> 📋 详细格式对比参见 [references/起诉状格式标准.md](起诉状格式标准.md)

- 证据清单
- 管辖法院

**证据材料读取规范**：

### 步骤2：交互收集当事人信息

当事人信息是起诉状的法定必要内容，必须逐项核实。信息不全时必须一次性列出所有缺失字段追问补全，不得留空或分多轮追问。

**重要规则：起诉状中不列示诉讼代理人信息**
- 起诉状当事人信息部分**只列原告信息和被告信息**
- 如有法定代理人则列法定代理人（如原告为未成年人、限制行为能力人等情形）
- 诉讼代理人信息**不写入起诉状正文**，仅在委托手续中体现

> 📋 详细信息字段表和交互格式参见 [templates/当事人信息收集表.md](../templates/当事人信息收集表.md)

### 步骤3：收集诉讼请求与事实理由

#### 3.1 诉讼请求

逐条列明，要求具体金额/事项，不接受模糊表述：

**违约金计算规则（统一计算方式）**：

当诉讼请求中包含违约金时，采用**统一计算方式**：

1. **基数**：以全部欠付金额为基数
2. **利率**：按合同约定的违约金标准（如每日万分之二）
3. **起算日**：从最后一期应付之日起算（如合同约定每年5月31日前支付，则最后一期应付日为合同到期年度的5月31日）
4. **截止日**：计算至起诉日（即今天）
5. **计算公式**：违约金 = 欠付金额 × 日利率 × 天数
6. **在诉讼请求中写明**：具体金额、基数、利率标准、起算日、截止日、天数

示例表述：判令被告支付违约金XX元（以欠付租金XX元为基数，按每日万分之二的标准，自XXXX年X月X日起计算至XXXX年X月X日，共XX天）

**利息、滞纳金等其他金额类请求计算规则**：
- **必须计算出截止到特定日期的具体金额**，不能只写计算公式或"计算至实际支付之日止"
- 正确格式示例：
  > "判令被告支付利息人民币XX元（自XXXX年X月X日起至XXXX年X月X日止，以欠付金额XX元为基数，按年利率X%的标准计算）"
- 起诉状中诉讼请求必须明确具体数额，后续如发生变动可在庭审中变更或补充

```
诉讼请求：
1. 请求判令被告___（具体内容+金额）；
2. 请求判令被告承担本案诉讼费用。
```

#### 3.2 事实与理由

引导用户按时间顺序提供：
- 合同/关系形成经过
- 履行过程
- 违约/侵权事实
- 损失情况及计算依据
- 协商/催告过程

#### 3.3 证据清单与完整性检验

**（一）证据清单收集**

逐条列明：证据名称、证明目的、证据来源（原件/复印件）

**（二）证据完整性检验（关键步骤）**

依据《诉讼精细化》手册，建立请求权基础要件映射审查机制，确保证据链闭合。需审查责任成立要件、责任范围要件、抗辩排除要件对应的证据是否完备。如存在证据缺漏，需生成补充材料清单。

> 📋 详细检验矩阵、补充材料清单模板和交互确认格式参见 [templates/证据完整性检验矩阵.md](../templates/证据完整性检验矩阵.md)

### 步骤4：生成起诉状（方案一升级版）

#### 4.1 起诉状结构框架

**基于"六来源体系"和请求权递进结构**：

```
民事起诉状

原告：[自然人：姓名，性别，X族，X年X月X日生，职业，工作单位，住所，联系方式]
     [法人：名称，住所，法定代表人：姓名（职务），联系方式]
被告：[自然人：姓名，性别，工作单位，住所，联系方式]
     [法人：名称，住所，法定代表人：姓名（职务），联系方式]

原告因[案由]一案，依法向贵院提起诉讼。

诉讼请求：
一、[主请求：明确金额/具体事项]；
二、[从请求：利息/违约金计算方式]；
三、请求判令被告承担本案诉讼费用。

【诉讼请求说明（内部参考，不写入正文）】
- 主体来源确认：原被告适格性依据
- 管辖来源确认：约定管辖/法定管辖依据
- 权利来源：请求权的法律基础
- 救济来源：请求的具体救济方式
- 成就来源：权利成就条件（如履行期届满）
- 内容来源：金额计算依据

事实与理由：

一、[案由主题/法律关系概述]
（一）请求权基础及构成要件
    1. 法律依据
       根据《[法律名称]》第[X]条规定："[法条全文引用]"
    2. 构成要件分析
       该请求权的构成要件包括：（1）[要件一]；（2）[要件二]；（3）[要件三]。
（二）要件事实与证据对应
    1. [要件一]：[事实描述] → 证据第1-1至1-3号
    2. [要件二]：[事实描述] → 证据第2-1至2-2号
    3. [要件三]：[事实描述] → 证据第3-1号

二、[第二项请求权的论证]（如有多项请求）
（一）请求权基础及构成要件
    ...
（二）要件事实与证据对应
    ...

三、关于损害赔偿的计算
（一）本金计算
    ...
（二）利息/违约金计算
    ...

综上所述，[总结请求权成立的核心理由]，原告的诉讼请求具有事实和法律依据，恳请贵院依法支持原告全部诉讼请求。

此致
[法院名称]

具状人：[原告姓名或名称]
[XXXX年XX月XX日]

附：
一、本起诉状副本X份
二、证据目录及证据材料
```

#### 4.2 事实与理由撰写规范

**法条援引规则（重要）**：
- **只写法条号，不写括号中的法条名称/内容说明**
- 正确示例：`依据《中华人民共和国民法典》第五百零九条、第五百七十七条、第五百八十五条之规定`
- 错误示例：`依据《中华人民共和国民法典》第五百零九条（全面履行原则）、第五百七十七条（违约责任）`
- 即：只写条文编号，括号内的说明文字一律删除

**请求权递进结构要求**（而非简单叙事）：

| 层级 | 内容 | 格式 |
|-----|------|------|
| 一级 | 案由主题/法律关系概述 | 一、二、三 |
| 二级 | 请求权基础/要件分析 | （一）（二）（三） |
| 三级 | 具体论证/证据对应 | 1. 2. 3. |
| 四级 | 细分要点 | （1）（2）（3） |

**论证结构示例**：
```
一、原告与被告之间存在有效的买卖合同关系
（一）请求权基础
    根据《民法典》第五百七十七条规定："当事人一方不履行合同义务或者履行
    合同义务不符合约定的，应当承担继续履行、采取补救措施或者赔偿损失等
    违约责任。"
（二）合同成立与生效
    1. 合同成立：原被告于X年X月X日签订《XX合同》 → 证据第1-1号
    2. 合同生效：该合同自签订之日起生效，不存在无效或可撤销情形
二、被告存在违约行为
（一）违约事实
    1. 合同约定：被告应于X年X月X日前交付货物 → 证据第1-1号第X条
    2. 实际履行：被告至今未交付 → 证据第2-1号（催告函）
（二）违约认定
    被告逾期未交付货物，构成迟延履行，符合违约责任构成要件。
```

#### 4.3 导出链路

最终本地 `.docx` 必须调用 `法律文书模板与导出` Skill。本文书起草 Skill 只负责事实、请求权基础、诉讼请求、事实与理由和证据结构；排版与导出由统一导出 Skill 接收语义 HTML 后完成。

**字体规范**：
- 标题（民事起诉状）：方正小标宋简体，居中，sz=44
- 正文段落：方正仿宋_GB2312，首行缩进640(firstLineChars=200)，两端对齐，sz=32
- 章节标题（诉讼请求/事实与理由等）：加粗，sz=32
- 当事人信息行：左对齐，sz=32

**XML 生成原则**：
- 每个段落用独立 `<w:p>` 包裹
- 格式属性写在 `<w:rPr>` 中，不混用内联样式
- 特殊字符必须用 XML 实体：`&lt;` `&gt;` `&amp;`

## 质证环节操作规范

### 触发条件

- 【客户编号】+ "质证"
- 【客户编号】+ "质证意见"
- 【客户编号】+ "对方证据"

### 证据获取规则

**1. 证据来源确认**

质证环节所需材料应当来自前述诉讼流程中已生成的文档：
- 案件沟通记录（含对方主张）
- 起诉状/答辩状（含证据清单）
- 初步法律分析（含请求权基础及要件事实）
- 案例检索报告（含有利裁判规则）

**2. 证据路径询问机制**

如前述环节未提供对方证据材料，必须向用户询问：

```
请提供对方当事人提交的证据材料：

【证据材料路径】
- 对方起诉状/答辩状路径：___
- 对方证据清单路径：___
- 对方证据材料文件夹路径：___

【证据形式说明】
- 书证：请提供PDF/图片文件路径
- 音频证据：请提供音频文件路径，我将提取主要内容
- 视频证据：请提供视频文件路径，我将提取主要内容
- 电子数据：请提供原始载体截图或公证文书路径
```

**3. 音视频证据处理**

用户提供的音频、视频证据，由用户提取主要内容后告知：

```
【音频/视频证据内容提取】

证据名称：___
时长：___
主要内容：
1. ...
2. ...
3. ...

关键时间节点：
- 00:00-01:30：...
- 01:30-03:00：...
```

**4. 证据材料读取规范**

获取对方证据材料路径后，读取时必须遵守以下规范：

- **逐份证据完整读取全部内容**（禁止摘要、禁止仅提取关键信息、禁止跳过任何页面）
- **PDF文件处理**：
  - 可复制文本PDF：直接全文读取
  - 扫描件PDF（图片型PDF）：使用pdf Skill进行OCR文字识别，读取全部内容
  - OCR精度不足时：在对应位置标注`[OCR识别不确定]`，提示用户人工核对
- **禁止部分读取**：无论文件页数多少，必须逐页完整读取

### 质证意见生成（方案三升级版）

质证按"关联性 → 合法性 → 真实性 → 证明力"顺序进行，提供六类标准质证模板（无异议、逾期举证异议、关联性异议、合法性异议、真实性异议、证明力异议）。针对不同证据类型（书证、物证、视听资料、电子数据、证人证言、鉴定意见等）有具体质证要点。

> 📋 详细质证模板、句式库和各类证据质证要点参见 [templates/质证标准模板库.md](../templates/质证标准模板库.md)

### 瑕疵证据补强规则

**不能单独作为认定案件事实依据的证据**（《民事证据规定》第90条）：
1. 当事人的陈述——需其他证据佐证
2. 无民事行为能力人或者限制民事行为能力人所作的与其年龄、智力状况或者精神健康状况不相当的证言
3. 与一方当事人或者其代理人有利害关系的证人陈述的证言
4. 存有疑点的视听资料、电子数据
5. 无法与原件、原物核对的复制件、复制品

**质证策略运用**：
- **攻击端**：指出对方证据属于上述瑕疵证据类型，论证其缺乏补强证据，不能单独采信
- **防御端**：对我方瑕疵证据提前准备补强证据链，确保不被孤立否定

### 证明妨害的质证运用

**识别证明妨害情形**：
1. 对方控制书证无正当理由拒不提交（已申请书证提出命令的）
2. 对方故意毁灭证据或使证据灭失
3. 对方妨碍对方当事人使用证据

**质证策略运用**：
- 向法庭申请适用证明妨害规则
- 法律后果：人民法院可以认定申请人所主张的书证内容为真实（《民事证据规定》第95条）
- 在质证意见中明确指出对方的证明妨害行为，请求法庭作出不利推定

### 电子数据证据质证专项要点

生成质证意见时，对电子数据类证据应额外关注以下专项质证要点（索引至质证标准模板库中的电子数据部分）：

**按类型的质证焦点**：

| 电子数据类型 | 真实性质证焦点 | 关联性质证焦点 | 合法性质证焦点 |
|------------|-------------|-------------|-------------|
| 聊天记录 | 身份确认、对话完整性、原始载体 | 对话内容与待证事实的关联 | 是否侵害隐私权 |
| 电子交易记录 | 平台认证、时间戳、防篡改验证 | 交易与争议事实的关联 | 获取途径是否合法 |
| 电子邮件 | 发件人身份、邮件头完整性 | 邮件内容与待证事实的关联 | 是否经合法途径获取 |
| 网页信息 | 公证保全时效、页面可修改性 | 网页内容与待证事实的关联 | 网页抓取方式是否合法 |
| 电子文件 | 文件元数据（创建/修改时间）、哈希值 | 文件内容与待证事实的关联 | 文件来源是否合法 |

详见 [templates/质证标准模板库.md](../templates/质证标准模板库.md) 电子数据部分。

## 注意事项

1. **当事人信息完整性**：信息不全时必须追问，不得擅自留空或填"不详"（被告信息除外）
2. **信息真实性**：所有填写信息必须基于案件事实，不得虚构
3. **管辖正确性**：必须确认管辖法院正确，避免程序错误
4. **签名要求**：生成的起诉状需由当事人签名或盖章后提交法院
5. **副本份数**：起诉状副本数量 = 被告人数 + 1（法院留存）
6. **时效注意**：注意诉讼时效，避免超期
