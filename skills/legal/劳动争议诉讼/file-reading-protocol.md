# 劳动争议诉讼文件读取规范

## 规范目的

确保所有劳动争议诉讼相关Skill在处理用户上传文件时遵循统一标准，优先使用技术手段读取文件内容，严禁跳过文件读取或失败后不告知用户。

---

## 一、文件读取四级兜底策略（强制执行）

### 第一级：专用Skill读取（优先）

**适用范围**：所有常见办公文档格式

| 文件格式 | 专用Skill | 调用方式 |
|---------|----------|---------|
| PDF | pdf Skill | `Skill:pdf` |
| Word (.doc/.docx) | docx Skill | `Skill:docx` |
| Excel (.xls/.xlsx) | xlsx Skill | `Skill:xlsx` |
| PowerPoint (.ppt/.pptx) | pptx Skill | `Skill:pptx` |

**操作要求**：
1. 首先检测已安装的专用Skill
2. 调用对应Skill读取文件内容
3. 获取完整文本内容后进行处理

### 第二级：通用read_file工具读取（兜底）

**适用场景**：专用Skill读取失败或不可用

**操作要求**：
1. 使用`read_file`工具直接读取文件
2. 支持文本文件、Markdown文件等
3. 对于二进制文件可能返回原始内容

### 第三级：OCR识别（针对扫描PDF/图片）

**适用场景**：
- 扫描件PDF（图片型PDF）
- 图片格式证据材料（.jpg/.png）
- 手写材料照片

**OCR三级质量评估标准**：

| 等级 | 判定标准 | 处理方式 |
|-----|---------|---------|
| **可用级** | 识别结果整体连贯、文字通顺，仅个别字符存疑 | 正常输出，存疑处标注`[OCR识别不确定]` |
| **部分可用级** | 识别结果部分段落可读、部分段落乱码或缺失 | 输出可读部分，不可读部分标注`[OCR识别失败：该段落无法可靠识别]`，并告知用户具体哪些页面/段落无法识别 |
| **不可用级** | 识别结果大面积乱码、字符碎片、语义不连贯，无法支撑准确阅读 | **禁止输出OCR结果**，直接告知用户识别失败并提供替代方案 |

### 第四级：用户确认（最终兜底）

**触发条件**：以上三级全部失败

**标准话术模板**：

```
【文件读取失败告知】

无法读取您上传的文件【文件名】，已尝试以下方式：
1. ✗ 专用文档读取工具
2. ✗ 通用文件读取工具
3. ✗ OCR文字识别

可能原因：
- 文件格式特殊或加密
- 文件损坏或权限受限
- 扫描件清晰度不足

建议解决方案：
1. 提供更清晰的扫描件（建议300dpi以上）
2. 提供可复制文本的PDF版本
3. 手动输入关键内容
4. 尝试转换为其他格式（如Word/TXT）

【询问】是否继续处理其他内容，或提供替代文件？
- 回复"继续"：跳过此文件，继续处理其他内容
- 回复"重试"：重新尝试读取
- 提供新路径：读取替代文件
```

## 二、读取准确性强制复查（强制执行）

文件成功读取后，**必须在进入后续处理前**执行以下复查流程。任何材料（证据材料、法律文书、合同、表格等）均适用，不得跳过。

### 复查步骤一：关键数据提取与校验

从读取内容中自动提取以下关键数据点，以结构化列表形式输出供用户确认：

| 数据类型 | 提取内容 | 校验方式 |
|---------|---------|---------|
| 当事人信息 | 姓名、身份证号、联系方式 | 交叉比对多处出现是否一致 |
| 日期时间 | 合同日期、事件日期、期限 | 检查日期格式合理性和前后一致性 |
| 金额数字 | 借款金额、赔偿金额、合同标的额 | 大小写金额是否一致、计算是否正确 |
| 案号信息 | 法院案号、仲裁案号 | 格式是否符合标准案号规则 |
| 地址信息 | 住所地、送达地址 | 多处出现是否一致 |
| 表格数据 | 行列数量、合计数 | 表格是否完整、合计数是否正确 |

### 复查步骤二：内容完整性检查

- 页数/段落数是否与原文件一致（如可判断）
- 是否存在明显的内容截断、断页、空白页
- 表格是否存在错位、合并单元格丢失
- 图片/印章/签名区域是否被正确识别或标注

### 复查步骤三：OCR结果专项复核（仅OCR读取时）

- 数字复核：金额、日期、电话号码、身份证号等数字序列重点检查
- 专有名词复核：当事人姓名、公司名称、法律术语是否被正确识别
- 上下文语义校验：识别结果是否符合法律文书/证据的语义逻辑
- 存疑标记：对无法确认准确性的内容标注 `[待用户确认]`

### 复查步骤四：输出读取复查摘要

**必须**向用户输出以下格式的复查摘要，**用户确认后才继续后续流程**：

```
【读取复查摘要】

文件名：xxx
读取方式：专用Skill读取 / read_file工具 / OCR识别
页数/段落数：xxx

【关键数据提取】
- 当事人：xxx
- 日期：xxx
- 金额：xxx
- 案号：xxx
- ...（根据文件内容提取）

【存疑项】（如有）
- [待用户确认] xxx（说明存疑原因）

【完整性评估】
- 评估结果：完整 / 存在缺失（具体说明）

请确认以上读取结果是否准确：[确认无误] [需要修正]
```

## 三、法律法规援引校验（强制执行）

当读取的材料中包含法律法规条文，或后续处理需要输出/引用法律法规时，**必须通过网络检索进行校验**，确保援引准确且条文现行有效。

### 校验步骤一：法条援引准确性验证

- 对输出中引用的每一条法律法规，通过网络检索验证：
  - 法律名称是否准确完整（如《中华人民共和国民法典》而非简写）
  - 条文编号是否正确（条、款、项、目对应关系）
  - 条文内容是否与官方发布版本一致，禁止凭记忆引用

### 校验步骤二：法条时效性验证

- 检查所引用的法律法规是否为**现行有效**版本：
  - 确认该法律法规未被废止、修改或替代
  - 如有新旧版本，必须引用最新生效版本
  - 标注生效日期和施行日期
  - 如引用已废止/失效条文（用于历史分析），必须明确标注`[已失效]`或`[已被XX替代]`

### 校验步骤三：输出法规校验摘要

在涉及法律法规引用的输出中，附带以下校验信息：

```
【法规援引校验】
- 引用法规：《XXX》第X条第X款
- 校验状态：现行有效 / 已修订（最新版为XXX）/ 已废止
- 检索来源：xxx
- 校验时间：YYYY-MM-DD
```

### 绝对禁止

- 未经网络检索验证就直接引用法律条文
- 引用已废止或失效的法规而不标注
- 凭记忆默写法条内容而不核实原文

## 四、绝对禁止行为

### ❌ 严禁以下行为：

1. **跳过文件不读取**
   - 用户明确提供了文件路径，必须尝试读取
   - 不得以任何理由忽略用户上传的文件

2. **读取失败后直接输出结果**
   - 文件读取失败时，必须先告知用户
   - 未经用户确认不得继续生成结果

3. **不读取文件就自行网络检索**
   - 严禁用网络搜索替代读取用户文件
   - 网络检索只能作为补充，不能替代文件读取

4. **部分读取或摘要替代**
   - 必须完整读取文件全部内容
   - 禁止仅读取前几页或提取摘要

5. **跳过复查直接使用读取结果**
   - 文件读取成功后，必须执行准确性复查
   - 未经用户确认复查摘要，不得进入后续处理流程
   - 关键数据（姓名、金额、日期、案号）必须提取校验

6. **未经校验引用法律法规**
   - 输出中涉及的法律法规必须通过网络检索验证
   - 禁止凭记忆引用法条，必须核实条文原文
   - 必须确认法规现行有效，废止法规需标注

## 五、不同类型文件处理规范

### 3.1 证据材料文件

**常见格式**：PDF、图片、扫描件

**处理流程**：
1. 优先使用pdf Skill读取
2. 扫描件使用OCR识别（三级质量评估）
3. 完整读取全部内容，逐页处理
4. 对证据进行分类和初步评估

### 3.2 法律文书文件

**常见格式**：.doc/.docx、.pdf

**处理流程**：
1. 优先使用docx Skill读取Word文档
2. PDF格式使用pdf Skill读取
3. 提取文书结构、法律依据、诉讼请求等关键信息

### 3.3 Excel表格文件

**常见格式**：.xls/.xlsx

**处理流程**：
1. 使用xlsx Skill读取
2. 读取全部工作表内容
3. 注意表头识别和数据完整性

### 3.4 图片证据

**常见格式**：.jpg/.png/.jpeg

**处理流程**：
1. 直接读取图片文件
2. 如需文字内容，使用OCR识别
3. 描述图片内容（如证据照片、聊天记录截图等）

## 六、批量文件处理规范

### 4.1 文件夹批量读取

当用户提供文件夹路径时：

1. 扫描文件夹内所有文件
2. 按文件类型分类处理
3. 逐一读取每个文件内容
4. 汇总输出所有文件内容清单

### 4.2 压缩包文件

当用户提供.zip/.rar等压缩包时：

1. 先解压压缩包
2. 按上述规范处理解压后的文件
3. 保持文件结构层次

---

## 七、错误处理与日志记录

### 5.1 读取失败记录

每次文件读取失败必须记录：
- 文件路径
- 尝试的读取方式
- 失败原因
- 时间戳

### 5.2 用户告知义务

无论何种原因导致文件读取失败，必须：
1. 明确告知用户读取失败
2. 说明已尝试的读取方式
3. 分析可能的原因
4. 提供替代解决方案
5. 询问用户是否继续

---

## 八、执行检查清单

在处理用户上传文件前，逐项确认：

- [ ] 已检测文件格式
- [ ] 已选择正确的读取方式
- [ ] 已尝试专用Skill读取（如适用）
- [ ] 失败后已尝试read_file工具
- [ ] 扫描件已尝试OCR识别
- [ ] OCR结果已进行三级质量评估
- [ ] 全部失败后已告知用户
- [ ] 已获得用户确认后再继续
- [ ] 读取成功后已执行关键数据提取与校验
- [ ] 已检查内容完整性（无截断、缺页、错位）
- [ ] OCR结果已进行专项复核（如适用）
- [ ] 已向用户输出读取复查摘要
- [ ] 已获得用户对复查摘要的确认
- [ ] 输出中的法律法规已通过网络检索验证
- [ ] 已确认所有引用法规为现行有效版本
- [ ] 已废止/失效法规已明确标注
