# 法律Wiki知识库查询Skill

## 一、功能概述

### （一）Skill定位

本Skill是法律专业知识查询的统一入口，为刑事辩护、民事诉讼、法律咨询等工作提供权威、结构化的知识支持。

### （二）核心功能

| 功能 | 说明 |
|-----|------|
| **罪名知识查询** | 查询刑事罪名的构成要件、量刑标准、辩护要点 |
| **民事领域查询** | 查询民事法律领域的规则、案例、实务要点 |
| **案例知识查询** | 查询典型判例的裁判要点、参考价值 |
| **交叉引用跳转** | 支持知识页之间的关联跳转 |

### （三）输出模式

| 模式 | 触发条件 | 输出内容 |
|-----|---------|---------|
| **知识卡片模式**（默认） | 未指定"深入" | 返回`<!-- CARD_START -->`到`<!-- CARD_END -->`之间的浓缩摘要（约800字） |
| **全文深入模式** | 用户指定"深入"或"全文" | 返回指定板块或完整全文 |

## 二、Wiki目录结构

### （一）索引页

| 索引页 | 路径 | 用途 |
|-------|------|------|
| 刑事知识索引 | `~/wiki/wiki/_index/刑事知识索引.md` | 定位刑事罪名知识页 |
| 民事知识索引 | `~/wiki/wiki/_index/民事知识索引.md` | 定位民事领域知识页 |

### （二）刑事知识页

| 类型 | 路径格式 | 示例 |
|-----|---------|------|
| 罪名各论 | `~/wiki/wiki/books/criminal/crimes/{罪名}.md` | `~/wiki/wiki/books/criminal/crimes/诈骗罪.md` |
| 刑法总论 | `~/wiki/wiki/books/criminal/doctrine/{专题}.md` | `~/wiki/wiki/books/criminal/doctrine/共犯认定.md` |
| 量刑专题 | `~/wiki/wiki/books/criminal/sentencing/{专题}.md` | `~/wiki/wiki/books/criminal/sentencing/自首认定.md` |
| 程序专题 | `~/wiki/wiki/books/criminal/procedure/{专题}.md` | `~/wiki/wiki/books/criminal/procedure/追诉时效.md` |
| 类型化专题 | `~/wiki/wiki/books/criminal/topics/{类型}/{专题}.md` | `~/wiki/wiki/books/criminal/topics/诈骗类/借款型诈骗认定.md` |
| 证据专题 | `~/wiki/wiki/books/criminal/evidence/{专题}.md` | `~/wiki/wiki/books/criminal/evidence/证据证明标准.md` |
| 案例知识 | `~/wiki/wiki/cases/criminal/{案例}.md` | `~/wiki/wiki/cases/criminal/张某某诈骗案.md` |

### （三）民事知识页

| 类型 | 路径格式 | 示例 |
|-----|---------|------|
| 领域知识 | `~/wiki/wiki/books/civil/{领域}/{具体问题}.md` | `~/wiki/wiki/books/civil/合同纠纷/违约金调整.md` |
| 案例知识 | `~/wiki/wiki/cases/civil/{案例}.md` | `~/wiki/wiki/cases/civil/某某公司合同纠纷案.md` |

## 三、查询流程

### （一）标准查询流程

```
用户查询请求
    │
    ├─→ 1. 解析查询意图（罪名/领域/关键词）
    │
    ├─→ 2. 读取对应索引页定位目标页
    │       - 刑事 → 读取刑事知识索引.md
    │       - 民事 → 读取民事知识索引.md
    │
    ├─→ 3. 读取目标知识页
    │
    ├─→ 4. 提取知识卡片（默认）
    │       或读取全文（深入模式）
    │
    ├─→ 5. 检查交叉引用和相关案例
    │
    └─→ 6. 输出查询结果
```

### （二）查询示例

**示例一：查询诈骗罪知识**
```
用户：查Wiki 诈骗罪

执行流程：
1. 读取 ~/wiki/wiki/_index/刑事知识索引.md
2. 定位到 books/criminal/crimes/诈骗罪.md
3. 读取诈骗罪.md
4. 提取 <!-- CARD_START --> 到 <!-- CARD_END --> 之间的知识卡片
5. 检查 cases/criminal/ 下是否有相关判例
6. 输出知识卡片 + 相关案例推荐
```

**示例二：深入查询（指定板块）**
```
用户：查Wiki 诈骗罪 深入 辩护策略

执行流程：
1-3. 同上
4. 读取全文，定位"辩护策略"板块
5. 输出该板块完整内容
```

## 五、交叉引用处理

### （一）自动识别交叉引用

知识页中常见的交叉引用标记：
- `参见：[相关罪名]`
- `相关阅读：[具体知识点]`
- `对比：[相似概念]`

### （二）交叉引用输出

在返回知识卡片时，自动列出交叉引用链接：

```
【相关知识点】
- 参见：盗窃罪（区分要点）
- 相关阅读：非法占有目的认定
- 对比：合同诈骗罪 vs 诈骗罪
```

## 六、与其他Skill的协作

### （一）被调用场景

| 调用方Skill | 调用场景 |
|------------|---------|
| 刑事辩护总调度 | 案件涉及具体罪名时自动查询 |
| 侦查阶段辩护 | 需要罪名构成要件、量刑标准时 |
| 审查起诉阶段辩护 | 需要辩护策略、司法解释时 |
| 庭前准备 | 需要案例支持、法律分析时 |
| 庭审与庭后工作 | 需要类案参考、裁判规则时 |

### （二）调用方式

其他Skill可通过以下方式使用本Skill：
1. **显式调用**：用户明确说"查Wiki XXX"
2. **隐式调用**：Skill检测到具体罪名/领域时自动查询

## 七、错误处理

### （一）知识页不存在

```
⚠️ 未找到对应知识页

【查询关键词】：XXX
【搜索路径】：~/wiki/wiki/books/criminal/crimes/XXX.md

建议：
1. 确认关键词是否正确
2. 查阅索引页获取准确名称
3. 如确实缺失，可记录补充需求
```

### （二）索引页读取失败

```
⚠️ 索引页读取失败

【路径】：~/wiki/wiki/_index/刑事知识索引.md

处理方式：
1. 检查Wiki目录是否存在
2. 如索引页缺失，尝试直接搜索知识页
```
