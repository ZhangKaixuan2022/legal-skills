# 诉讼可视化Skill

## 功能概述

根据《诉讼精细化》办案手册第八节要求，运用Mermaid图表语法，将复杂的案件法律关系、时间脉络、争议焦点、要件分析、攻防对抗和证据链条转化为直观的可视化图表。按总控当前事项记录和复盘台账衔接，生成法官版和律师版两种报告，帮助律师和客户快速理解案件结构和关键问题。

## 依赖Skill

本Skill需要调用 **mermaid-diagrams** Skill 来生成Mermaid图表代码。

## 触发条件

- 【客户编号】+ "可视化"
- 【客户编号】+ "诉讼可视化"
- 【客户编号】+ "制作图表"

3. 获取"法律意见书文件路径"，读取请求权分析和争点清单
4. 获取"沟通记录文件路径"，读取证据初步清单

## 第二部分：图表类型说明

本Skill支持6种Mermaid图表类型，用于不同的可视化需求：

| 图表类型 | Mermaid语法 | 主要用途 |
|---------|-------------|---------|
| 法律关系图 | flowchart LR | 展示当事人间的法律关系、权利义务流向 |
| 法律事实时间图 | timeline | 展示案件关键时间节点和事件脉络 |
| 争点树 | mindmap | 展示争议焦点的树形结构和状态标注 |
| 要件分析图 | graph TD | 展示请求权→构成要件→证据的层级关系 |
| 攻防对抗图 | graph LR | 左右对照展示我方主张与对方抗辩 |
| 证据链图谱 | graph TD | 按证明对象分组展示证据关系 |

> 📖 详细语法示例和标注规范参见 [references/六类Mermaid图表完整说明.md](六类Mermaid图表完整说明.md)

## 第三部分：工作流程

### 步骤1：读取前置文档与系统记录

1. 从当前事项系统记录读取客户信息、案由、当事人信息
2. 读取案件沟通记录（获取客户信息、案件事实、证据初步清单）
3. 读取法律服务建议书（获取请求权路径、构成要件分析、争点清单、证据评估）

### 步骤2：分析法律关系结构

**分析要素：**
- **当事人识别**：原告、被告、第三人、担保人、利害关系人等
- **法律关系类型**：合同关系、侵权关系、担保关系、不当得利、无因管理等
- **权利义务流向**：谁向谁主张什么权利、承担什么义务
- **关键要素标注**：金额、期限、违约事实、损害后果等

**输出：** 法律关系结构分析表

### 步骤3：提取关键时间节点

**时间要素：**
- 合同签订日
- 履行期限/违约发生日
- 催告/协商日
- 重要沟通节点
- 起诉准备日
- 其他影响诉讼时效的日期

**输出：** 关键时间节点列表

### 步骤4：构建争点树结构

基于法律服务建议书中的争点清单：

1. **识别主争点**：案件核心争议
2. **分解子争点**：
   - 事实争议焦点
   - 法律争议焦点
   - 证据争议焦点
3. **关联要件事实**：每个子争点对应的要件事实
4. **关联证据**：每个要件事实的证据支撑
5. **标注状态**：
   - 共识要件（✅绿色）
   - 争议要件（🔴红色）
   - 待确认要件（🟡黄色）
   - 证据状态：✅充分 / ⚠️不足 / ❌缺失

**输出：** 争点清单（树形结构）

### 步骤5：构建要件分析结构

基于法律服务建议书中的请求权基础分析：

1. **确定请求权**：从法律服务建议书获取推荐的请求权路径
2. **列出构成要件**：每个请求权的全部构成要件
3. **关联要件事实**：每个要件对应的具体事实
4. **关联证据**：支撑每个要件事实的证据
5. **标注状态**：✅满足 / ⚠️待证 / ❌不满足

**输出：** 要件分析表

### 步骤6：构建攻防对抗结构

1. **整理我方主张**：基于请求权基础
2. **预测对方抗辩**：基于法律服务建议书中的抗辩权预判
3. **关联争议焦点**：每对攻防对应的争议焦点编号

**输出：** 攻防对照表

### 步骤7：构建证据链图谱

1. **按证明对象分组**：
   - 合同成立
   - 付款/履行事实
   - 违约事实
   - 损害事实
   - 因果关系
   
2. **标注每个证据节点**：
   - 证据名称
   - 证据类型
   - 原件状态
   - 三性评价

3. **标注证据关系**：
   - 印证关系（实线箭头）
   - 补强关系（虚线箭头）
   - 矛盾关系（双向虚线）

**输出：** 证据链关系表

### 步骤8：生成Mermaid图表

调用 **mermaid-diagrams** Skill，根据分析结果生成以下图表：

**全部图表类型：**
1. 法律关系图（flowchart LR）
2. 法律事实时间图（timeline）
3. 争点树（mindmap）
4. 要件分析图（graph TD）
5. 攻防对抗图（graph LR）
6. 证据链图谱（graph TD）

**法官版图表（不含策略分析）：**
1. 法律关系图
2. 法律事实时间图
3. 争点树（去除"对我方有利/不利"等标注）

**律师版图表（全部图表+策略标注）：**
- 全部6种图表
- 包含策略标注和风险提示
- 薄弱环节用红色高亮
- 证据缺失处标注补强建议

## 第四部分：排版与嵌入规范

**Mermaid图表处理原则：**
- 每个图表同时导出 `.mmd` 源文件和 `.png` 图片

## 第五部分：输出双版本

### 5.1 法官版：诉讼可视化报告（法庭提交版）

**特点：**
- 纯图表 + 客观说明文字
- 不含任何策略分析、主观评价或"对我方有利/不利"标注
- 仅包含：法律关系图、时间轴图、争点树（客观版）
- 不含攻防对抗图、要件分析图、证据链图谱

**文件名：** `诉讼可视化报告_【客户姓名】_【YYYYMMDD】.docx`

> 📋 详细模板参见 [templates/法官版报告框架.md](../templates/法官版报告框架.md)

### 5.2 律师版：诉讼可视化分析报告（律师内部版）

**特点：**
- 包含全部6种图表类型
- 包含策略标注和风险提示
- 薄弱环节用红色高亮
- 证据缺失处标注补强建议
- 包含"对我方有利/不利"等分析性标注

**文件名：** `诉讼可视化分析报告_【客户姓名】_【YYYYMMDD】.docx`

## 第六部分：文件保存（飞书云空间）

### 6.1 飞书上传流程

**步骤1：创建飞书文件夹**

在飞书云空间中创建以客户姓名命名的文件夹：

```bash
lark-cli drive folders create --data '{"name": "【客户姓名】", "folder_token": ""}'
```

> 注意：`folder_token` 为空字符串时，将在"我的空间"根目录下创建文件夹。如系统记录/复盘台账中已记录该客户的「飞书文件夹token」，则跳过创建步骤，直接使用已有文件夹。

**步骤2：上传文件到飞书**

将本地生成的所有文件逐一上传到飞书文件夹：

```bash
# 上传 .mmd 源文件和 .png 图片
lark-cli drive +upload --file ./法律关系分析图_【客户姓名】.mmd --folder-token <文件夹token>
lark-cli drive +upload --file ./法律关系分析图_【客户姓名】.png --folder-token <文件夹token>
# ...其余图表文件同理

# 上传 Word 报告
lark-cli drive +upload --file ./诉讼可视化报告_【客户姓名】_【YYYYMMDD】.docx --folder-token <文件夹token>
lark-cli drive +upload --file ./诉讼可视化分析报告_【客户姓名】_【YYYYMMDD】.docx --folder-token <文件夹token>
```

**步骤3：记录飞书文件信息**

上传成功后，记录返回的 `file_token`，用于后续系统记录同步和其他Skill引用。

**步骤4：同步系统记录**

文件上传完成后，将 `file_token`、输出文件名和保存位置写入当前事项系统记录；如需同步复盘台账，按法律工作总控规则确认后再更新。

## 第七部分：系统记录同步逻辑

> **强制规则**：当前事项系统记录必须同步；复盘台账仅在用户确认或总控规则要求时更新。

### 7.1 同步位置

当前事项系统记录；复盘台账按需确认后同步。

### 7.2 同步字段

生成可视化报告后，更新以下字段：

| 字段名 | 回写内容 |
|-------|---------|
| 案件状态 | "已可视化" |
| 飞书文件夹token | 飞书云空间中客户文件夹的 folder_token |
| 可视化报告file_token（法官版） | 法官版报告在飞书中的 file_token |
| 可视化分析报告file_token（律师版） | 律师版报告在飞书中的 file_token |
| 可视化完成时间 | 当前日期时间 |

### 7.3 字段维护

当前事项系统记录中没有上述字段时，在系统记录中补充；复盘台账不静默扩展字段。

### 7.4 同步异常处理

- 如当前事项系统记录不存在，先提示建立或切换当前事项
- 如复盘台账需要更新但未经确认，先提示用户确认
- 同步失败时写入当前事项的缺口/待办记录并提示用户

## 第十部分：注意事项

1. **图表准确性**：可视化图表基于已有信息生成，随着案件进展可能需要更新

2. **保密义务**：
   - 法官版可对外提交
   - 律师版仅限内部使用，请勿对外提供

3. **客户沟通**：可视化图表是与客户沟通的有效工具，建议使用律师版与客户讨论策略

4. **庭审使用**：法官版图表可在庭审中使用，帮助法官理解复杂案情

5. **Mermaid限制**：部分复杂图表可能需要手动调整布局以达到最佳展示效果

6. **前序Skill依赖**：本Skill依赖初步法律分析Skill的输出，建议先完成初步法律分析再执行可视化
