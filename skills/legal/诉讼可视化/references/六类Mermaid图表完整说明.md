# 六类Mermaid图表完整说明

> **来源**: 诉讼可视化Skill - 第二部分：图表类型说明
> **用途**: 详细描述6种诉讼可视化图表的用途、Mermaid语法示例和标注规范

---

## 2.1 法律关系图（flowchart）

使用 flowchart 语法展示当事人之间的法律关系：

```mermaid
flowchart LR
    A[原告:张三<br/>出借人] -->|借款合同<br/>金额:100万| B[被告:李四<br/>借款人]
    B -->|违约<br/>逾期未还| C[违约事实<br/>本金+利息]
    B -->|连带责任保证| D[担保人:王五]
    D -->|担保范围<br/>本金+利息+违约金| C
```

**图表要素：**
- 各方当事人及其法律地位
- 法律关系类型及核心内容
- 权利义务流向
- 关键金额、期限等要素

## 2.2 法律事实时间图（timeline）

使用 timeline 语法展示案件关键时间线：

```mermaid
timeline
    title 案件关键时间节点
    2023-01-15 : 签订借款合同
               : 约定借款100万,期限6个月
    2023-01-20 : 支付借款
               : 银行转账100万至李四账户
    2023-07-15 : 借款到期
               : 李四未按期还款
    2023-08-01 : 发送催款函
               : 书面催告,要求10日内还款
    2024-01-10 : 提起诉讼
               : 诉讼时效内起诉
```

**图表要素：**
- 关键时间节点
- 对应事件描述
- 与诉讼时效的关系提示

## 2.3 争点树（mindmap）

将争议焦点以树形结构展示，按层级标注状态：

```mermaid
mindmap
  root((主争点：合同履行纠纷))
    事实争议
      共识：合同已签订 ✅
      争议：付款金额争议 🔴
        要件事实：实际付款100万
          证据：转账凭证 ✅充分
        要件事实：对方主张仅收到80万
          证据：收据 ⚠️不足
    法律争议
      共识：合同有效 ✅
      争议：违约金计算标准 🔴
        要件事实：约定日万分之五
          证据：合同条款 ✅充分
        要件事实：是否过高
          证据：无 ❌缺失
    证据争议
      待确认：微信记录真实性 🟡
```

**节点状态标注：**
- 共识要件（双方无争议）→ ✅ 绿色
- 争议要件（双方有分歧）→ 🔴 红色
- 待确认要件 → 🟡 黄色
- 证据状态：✅充分 / ⚠️不足 / ❌缺失

## 2.4 要件分析图（graph TD）

展示请求权→构成要件→要件事实→证据支撑的层级关系：

```mermaid
graph TD
    A[请求权: 违约损害赔偿] --> B1[要件1: 合同有效成立]
    A --> B2[要件2: 违约事实]
    A --> B3[要件3: 损害事实]
    A --> B4[要件4: 因果关系]
    B1 --> C1[证据: 合同原件 ✅]
    B2 --> C2[证据: 催告函 ✅]
    B2 --> C3[证据: 微信记录 ⚠️待补强]
    B3 --> C4[证据: 损失计算书 ❌缺失]
    B4 --> C5[证据: 鉴定报告 ⚠️待补强]
```

**要件状态标注：**
- ✅ 满足（证据充分）
- ⚠️ 待证（证据不足，需补强）
- ❌ 不满足（证据缺失）

## 2.5 攻防对抗图（graph LR）

左右对照结构展示我方主张与对方可能抗辩：

```mermaid
graph LR
    subgraph 我方主张
        A1[主张1: 合同有效]
        A2[主张2: 对方违约]
        A3[主张3: 损失XXX元]
    end
    subgraph 对方可能抗辩
        B1[抗辩1: 合同无效]
        B2[抗辩2: 不可抗力]
        B3[抗辩3: 损失过高]
    end
    A1 -.争点1.-> B1
    A2 -.争点2.-> B2
    A3 -.争点3.-> B3
```

**图表要素：**
- 左侧：我方主张（基于请求权基础）
- 右侧：对方可能抗辩（基于抗辩权分析）
- 中间连线：标注争议焦点编号

## 2.6 证据链图谱（graph TD）

按证明对象分组，展示证据之间的关系：

```mermaid
graph TD
    subgraph 证明对象1: 合同成立
        E1[合同原件<br/>书证/原件/三性✅]
        E2[签约录像<br/>视听资料/原件/三性✅]
    end
    subgraph 证明对象2: 付款事实
        E3[银行流水<br/>书证/原件/三性✅]
        E4[收据<br/>书证/复印件/真实性⚠️]
        E5[微信转账截图<br/>电子数据/原件/关联性⚠️]
    end
    subgraph 证明对象3: 违约事实
        E6[催告函<br/>书证/原件/三性✅]
        E7[EMS签收记录<br/>书证/原件/三性✅]
    end
    E1 -->|印证| E2
    E3 -->|印证| E5
    E4 -.补强.-> E3
    E4 <-.矛盾.-> E5
    E6 -->|印证| E7
```

**证据节点信息：**
- 证据名称
- 证据类型（书证/物证/视听资料/电子数据/证人证言）
- 原件状态（原件/复印件）
- 三性评价（✅通过 / ⚠️存疑 / ❌不通过）

**关系类型：**
- 实线箭头 `-->|印证|`：多个证据指向同一事实
- 虚线箭头 `-.补强.->`：一个证据增强另一证据的证明力
- 红色双向虚线 `<-.矛盾.->`：两个证据互相矛盾
