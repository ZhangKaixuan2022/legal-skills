# 调查取证与证据管理Skill

## 功能概述

本Skill定位在民事一审诉讼体系的第4.5环节（介于"法规案例检索"与"诉讼文书起草"之间），承接初步法律分析的请求权要件清单和案件沟通记录的证据初步清单，系统化指引证据收集、调查取证、证人管理和财产保全全流程，输出证据材料和文书供诉讼文书起草使用。

### 核心功能

1. **证据缺口分析**：依据请求权构成要件清单，逐项核查证据完备性
2. **取证方案制定**：生成律师内部取证方案和交当事人的补充材料清单
3. **调查令申请**：生成工商、不动产、银行、车辆四类调查令申请书
4. **证人管理**：生成调查笔录、证人出庭申请书
5. **财产保全**：提供保全决策支持，生成诉前/诉讼保全申请书
6. **期限管理**：计算举证期限，生成证据提交清单

## 触发方式（双模式）

### 模式一：被动触发

用户发送客户编号 + 提及以下关键词时自动触发：
- 【客户编号】+ "调查取证"
- 【客户编号】+ "证据收集"
- 【客户编号】+ "调查令"
- 【客户编号】+ "财产保全"
- 【客户编号】+ "证人"
- 【客户编号】+ "补充材料"
- 【客户编号】+ "取证方案"

### 模式二：主动询问

当律师向AI提供证据材料路径（如"这是当事人提供的证据材料：/path/to/folder"）时：

1. 检测到用户提供了证据文件/文件夹路径
2. 读取该路径下的材料，识别证据类型和内容
3. 读取当前事项系统记录中该案件的法律服务建议书路径，提取请求权要件清单
4. 比对已有证据与所需证据，判断是否存在缺口
5. 若存在缺口 → 主动提示："检测到以下证据缺口：[缺口清单]，是否开启调查取证流程？"
6. 用户确认后启动完整流程

## 第二部分：核心工作流程

### 步骤1：读取前序文档与系统记录衔接

**操作指令**：

1. **读取当前事项系统记录**：根据客户编号读取客户/案件信息，获取客户称呼、联系方式、案件沟通记录路径、法律服务建议书路径

2. **读取案件沟通记录**：
   - 提取"证据初步清单"附表
   - 记录已有证据的名称、类型、原件/复印件状态、证明对象

3. **读取法律服务建议书**：
   - 提取"请求权构成要件清单"
   - 提取"证据评估"结论
   - 提取"争点清单"

4. **证据材料读取规范**（如用户主动提供证据材料路径时适用）：
   - **逐份证据完整读取全部内容**（禁止摘要、禁止仅提取关键信息、禁止跳过任何页面）
   - **PDF文件处理**：
     - 可复制文本PDF：直接全文读取
     - 扫描件PDF（图片型PDF）：使用pdf Skill进行OCR文字识别，读取全部内容
   - **OCR识别结果质量评估（三级处理，严格执行）**：
     - 【可用级】识别结果整体连贯、文字通顺，仅个别字符存疑 → 正常输出，存疑处标注`[OCR识别不确定]`
     - 【部分可用级】识别结果部分段落可读、部分段落乱码或缺失 → 输出可读部分，不可读部分标注`[OCR识别失败：该段落无法可靠识别]`，并向用户说明哪些页面/段落无法识别，建议提供清晰扫描件或文字版PDF
     - 【不可用级】识别结果整体呈现大面积乱码、字符碎片、语义不连贯，无法支撑准确阅读 → **禁止输出OCR结果**，直接告知用户："该文件为扫描件，OCR识别准确率不足以支撑可靠读取，建议：(1)提供更清晰的扫描件；(2)提供可复制文本的PDF版本；(3)手动输入关键内容"
   - **禁止部分读取**：无论文件页数多少，必须逐页完整读取

5. **信息整合提示**：
   > "已读取【客户称呼】的案件档案。现有证据X份，请求权要件Y项。开始证据缺口分析。"

### 步骤2：证据缺口分析

依据请求权构成要件清单，逐项核查证据完备性。

#### 2.1 证据缺口分析表

**输出格式**（Word表格）：

| 序号 | 请求权要件 | 所需证据类型 | 已有证据 | 证据状态 | 缺口描述 | 建议取证方式 | 优先级 |
|-----|-----------|------------|---------|---------|---------|------------|-------|
| 1 | | | | | | | |
| 2 | | | | | | | |

#### 2.2 证据状态分类

| 状态标识 | 状态名称 | 说明 |
|---------|---------|------|
| ✅ | 充分 | 已有直接证据且形式完整 |
| ⚠️ | 不足 | 有证据但证明力弱或形式有瑕疵 |
| ❌ | 缺失 | 完全没有证据 |
| 🔍 | 待核实 | 需要进一步确认真实性或原件 |

#### 2.3 建议取证方式分类（八种方法）

| 编号 | 取证方式 | 适用场景 |
|-----|---------|---------|
| 1 | 向当事人收集 | 证据由当事人掌握，需补充提供 |
| 2 | 发送核实函 | 向相关方发函核实事实 |
| 3 | 向对方发函 | 要求对方提供其掌握的证据 |
| 4 | 向第三方取证 | 向银行、工商等机构调取 |
| 5 | 固定证人证言 | 制作调查笔录 |
| 6 | 网络检索 | 通过公开信息检索固定 |
| 7 | 公证保全 | 对易灭失证据进行公证固定 |
| 8 | 申请调查令 | 向法院申请调查令调取 |

#### 2.4 优先级判定标准

| 优先级 | 判定标准 |
|-------|---------|
| 高 | 核心要件缺失证据，直接影响请求权成立 |
| 中 | 次要要件证据不足，影响证明力 |
| 低 | 补强证据，有则更好无亦可 |

### 步骤4：调查取证申请文书生成

根据取证方案中需要"申请调查令"或"申请法院调查取证"的项目，区分调查对象生成对应文书。

#### 4.1 调查令申请书（区分四类）

**4.1.1 工商版调查令申请书**

- **调查对象**：市场监督管理局/工商行政管理局
- **调查内容**：企业基本信息、股东信息、变更记录、年报信息
- **法律依据**：《民事诉讼法》第67条、各地调查令规定
- **模板路径**：`templates/调查令申请书_工商版.md`

**人工协作提示**：
> "调查令申请书（工商版）已生成。请持调查令前往【工商部门名称】调取档案，完成后将材料扫描件保存至案件文件夹并告知路径。"

**4.1.2 不动产版调查令申请书**

- **调查对象**：不动产登记中心
- **调查内容**：权属信息、抵押登记、查封信息、交易记录
- **模板路径**：`templates/调查令申请书_不动产版.md`

**4.1.3 银行版调查令申请书**

- **调查对象**：商业银行
- **调查内容**：账户信息、资金流水、冻结状态
- **模板路径**：`templates/调查令申请书_银行版.md`

**4.1.4 车辆版调查令申请书**

- **调查对象**：车辆管理所
- **调查内容**：车辆登记信息、抵押状态、违章记录
- **模板路径**：`templates/调查令申请书_车辆版.md`

#### 4.2 调查取证申请书

- **适用场景**：律师无法自行取证且不适用调查令的情况
- **申请法院依职权调查取证**
- **法律依据**：《民事诉讼法》第67条第2款
- **模板路径**：`templates/调查取证申请书.md`

**文书共性要求**：
每份文书均包含：
1. 申请人信息
2. 申请事项
3. 与请求权要件的关联论证（说明该证据对证明哪个要件具有必要性）
4. 法律依据

#### 步骤4a：书证提出命令

**适用场景**：待证事实的关键证据由对方当事人控制，我方无法自行取得。

**操作要点**：
1. 申请条件：明确书证名称或内容、书证由对方控制的理由、书证与待证事实的关联性
2. 申请时机：举证期限届满前向法院书面申请
3. 法律后果：对方无正当理由拒不提交的，人民法院可以认定申请人所主张的书证内容为真实（《民事证据规定》第48条）
4. 文书生成：详见 `templates/书证提出命令申请书.md`

> **法律依据**：《民事诉讼法解释》第112条、《民事证据规定》第45-48条

---

#### 步骤4b：公证证据保全

**适用场景**：证据可能灭失或日后难以取得，需通过公证方式固定证据。

**与诉前/诉讼保全的区别**：公证保全由公证机构实施，不受诉讼程序约束，可在纠纷发生初期即时启动。

**操作要点**：
1. 保全方式：公证员现场监督下的拍照、录像、复制、下载、截屏等
2. 电子数据保全：保存原始存储介质，由公证员见证提取过程，记录哈希值
3. 网络证据保全：公证员现场操作或监督操作，记录完整访问路径
4. 隐名公证的合法性边界：取证行为本身不得违反法律禁止性规定，不得侵害他人合法权益
5. 效力：经公证的证据具有推定真实的效力，但有相反证据足以推翻的除外

> **法律依据**：《公证法》第36条、《民事诉讼法解释》第93条第7项

---

#### 步骤4c：证据合法性红线检验

**适用场景**：收集证据前或收集证据后，检验取证方式是否合法。

**三类非法证据排除情形**（《民事证据规定》第68条精神）：
1. 严重侵害他人合法权益的方法取得——如非法侵入住宅、窃取他人隐私
2. 违反法律禁止性规定的方法取得——如违反法定程序
3. 严重违背公序良俗的方法取得

**注意**：民事诉讼中非法证据排除标准低于刑事诉讼，"偷录偷拍"在未侵害他人合法权益的前提下一般可采纳。

详见 `references/证据合法性检验矩阵.md`

---

#### 步骤4d：电子数据证据收集规范

**电子数据范围**：网页信息、即时通信记录（微信/QQ等）、电子邮件、电子交易记录、用户注册信息、电子文件（文档/图片/音视频）。

**收集要点**：
1. 原始载体保存：优先保存原始存储介质（手机、硬盘、U盘等），不得直接在原始载体上操作
2. 完整性校验：对电子数据进行哈希值（MD5/SHA256）计算并记录
3. 提取过程记录：记录提取时间、地点、操作人、操作步骤
4. 公证保全：重要电子数据建议通过公证方式固定（参见步骤4b）
5. 第三方平台数据：向平台运营商申请调取（结合步骤4调查令）

详见 `references/电子数据证据收集与保全指南.md`

### 步骤6：财产保全

#### 6.1 财产线索调查（人工协作模式）

**提示律师需查询的内容和平台**：

| 查询对象 | 查询平台/方式 | 查询内容 |
|---------|-------------|---------|
| 企业信息 | 企查查/天眼查/国家企业信用信息公示系统 | 股权、对外投资、关联企业、资产状况 |
| 不动产 | 不动产登记中心（需调查令） | 房产权属、抵押、查封 |

| 公开信息 | 裁判文书网、执行信息公开网 | 涉诉情况、被执行记录 |

**操作流程**：
1. 生成《财产线索查询指引》
2. 提示律师："请完成以上查询，将结果文件保存并告知路径"
3. 律师提供路径后，读取文件自动生成《财产线索清单》

#### 6.2 保全决策支持

**根据财产线索清单，进行保全必要性评估**：

| 评估维度 | 判断标准 |
|---------|---------|
| 紧急程度 | 对方是否有转移、隐匿、毁损财产的行为或可能 |
| 保全金额 | 不超过诉讼请求金额 |
| 担保能力 | 申请人能否提供担保（保险保函/银行保函/担保公司/自有财产） |
| 保全时机 | 诉前保全（须30日内起诉）vs 诉讼保全 |

**输出**：
- 保全建议（建议保全/暂不保全/视情况而定）
- 建议保全的具体财产和理由

### 步骤7：证据提交与期限管理

#### 7.1 举证期限计算

- 根据案件受理日期，计算举证期限
- 提示：举证期限可由法院指定，也可由当事人协商经法院认可

#### 7.2 不受举证期限约束的情形

- 新证据
- 反驳证据
- 当事人申请法院调查收集的证据
- 法院依职权调查的证据
- 补充或补正诉讼请求所需证据
- 逾期提交但涉案件基本事实的证据

#### 7.3 逾期举证风险预警

- 法院可不予采纳
- 对方可要求赔偿因逾期举证造成的损失
- 法院可训诫、罚款

## 第五部分：注意事项

### 5.1 证据收集注意事项

1. **合法性原则**：取证方式必须合法，不得采用非法手段获取证据
2. **及时性原则**：对易灭失证据及时采取保全措施
3. **完整性原则**：收集证据应全面，避免选择性收集
4. **原件优先**：尽量收集原件，复印件需核对无误

### 5.2 调查令使用注意事项

1. **适用范围**：各地对调查令适用范围有不同规定，需确认当地规定
2. **使用期限**：调查令通常有有效期限，需在期限内使用
3. **持令人**：一般由代理律师持令调查
4. **结果反馈**：调取的材料应及时归档并告知法院

### 5.3 证人证言注意事项

1. **独立性**：证人应独立作证，不得与其他证人串供
2. **真实性**：如实记录，不得诱导、威胁或利诱
3. **程序规范**：调查笔录需被调查人确认签字
4. **出庭准备**：提前告知证人庭审流程和注意事项

### 5.4 财产保全注意事项

1. **必要性论证**：诉前保全需充分论证紧急性
2. **担保准备**：提前准备担保材料
3. **线索准确**：财产线索应尽量准确具体
4. **时效把握**：诉前保全后30日内必须起诉

### 5.5 举证期限注意事项

1. **及时提交**：在举证期限内提交证据
2. **逾期后果**：逾期举证可能面临证据不被采纳的风险
3. **申请延期**：确有困难可申请延期举证
4. **新证据**：符合法定情形的"新证据"不受举证期限限制
