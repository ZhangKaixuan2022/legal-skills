# 电子数据证据收集与保全指南

本文档依据《最高人民法院关于民事诉讼证据的若干规定》及《最高人民法院关于互联网法院审理案件若干问题的规定》，系统梳理各类电子数据证据的收集与保全规范。

---

## 一、电子数据证据范围

根据《最高人民法院关于民事诉讼证据的若干规定》第十四条，电子数据包括下列信息、电子文件：

1. 网页、博客、微博客等网络平台发布的信息
2. 手机短信、电子邮件、即时通信、通讯群组等网络应用服务的通信信息
3. 用户注册信息、身份认证信息、电子交易记录、通信记录、登录日志等信息
4. 文档、图片、音频、视频、数字证书、计算机程序等电子文件
5. 其他以数字化形式存储、处理、传输的能够证明案件事实的信息

---

## 二、通用收集与保全原则

### 2.1 原始载体优先原则

- **核心要求**：优先保存原始存储介质（手机、电脑硬盘、U盘、服务器等）
- **禁止行为**：不得在原始载体上直接进行编辑、删除、修改等操作
- **例外情形**：原始载体不便封存时，应制作具有唯一性标识的备份

### 2.2 完整性校验原则

- **哈希值计算**：对电子数据进行MD5或SHA256哈希值计算并记录
- **校验时机**：提取时计算一次，提交前再次校验
- **记录保存**：将哈希值与电子数据一并保存，作为完整性证明

### 2.3 过程记录原则

- **记录要素**：提取时间、地点、操作人、操作步骤、设备信息
- **见证要求**：重要电子数据提取应有见证人在场或公证员见证
- **同步录像**：必要时对提取过程进行录像记录

### 2.4 及时保全原则

- **时效性**：电子数据易被篡改、删除，应及时固定
- **公证保全**：重要电子数据建议通过公证方式固定
- **区块链存证**：可考虑使用可信的区块链存证平台

## 三、分类收集与保全规范

### 3.1 即时通信记录（微信/QQ/钉钉等）

#### 保全方法

1. **公证保全（推荐）**
   - 公证员现场操作手机，逐条展示聊天记录
   - 对关键对话进行截屏或录屏
   - 将截屏/录屏文件刻录光盘或存入U盘
   - 公证处出具保全证据公证书

2. **自行保全**
   - 使用另一台设备对聊天界面进行录像
   - 展示聊天双方头像、昵称、完整对话内容
   - 展示聊天时间、消息状态（已发送/已读）
   - 保存原始手机，不得删除聊天记录

#### 注意事项

| 注意要点 | 具体要求 |
|---------|---------|
| 身份确认 | 通过备注名称、转账记录、共同好友等方式确认对方身份 |
| 完整性 | 展示完整对话，不得选择性截取片段 |
| 时间连续性 | 确保聊天记录的时间顺序完整 |
| 原始载体 | 保存原始手机，作为核对依据 |
| 微信转账 | 申请微信支付电子凭证以确认收款方身份 |

#### 证明力增强

- 申请对方当事人确认聊天记录真实性
- 结合其他证据（转账记录、证人证言等）佐证身份
- 申请法院调取腾讯公司后台数据（需调查令）

---

### 3.2 电子邮件

#### 保全方法

1. **公证保全**
   - 公证员登录邮箱，展示邮件列表
   - 逐封打开重要邮件，展示发件人、收件人、时间、内容
   - 对邮件进行截屏或打印
   - 保存邮件为.eml或.msg格式

2. **自行保全**
   - 登录邮箱，对重要邮件进行截屏
   - 下载邮件原件保存
   - 记录邮箱登录时间、IP地址

#### 注意事项

| 注意要点 | 具体要求 |
|---------|---------|
| 邮件头信息 | 保存完整的邮件头信息（发件人、收件人、抄送、时间戳、邮件ID等） |
| 发件人验证 | 核实发件人邮箱地址，注意辨别相似邮箱（如将"l"替换为"1"） |
| 附件保全 | 邮件附件应一并保存，记录附件名称和大小 |
| 服务器日志 | 必要时申请法院调取邮件服务器日志 |

---

### 3.3 电子交易记录（银行/支付宝/微信支付等）

#### 保全方法

1. **银行交易记录**
   - 通过银行柜台打印盖章的交易流水
   - 通过网银/手机银行导出交易明细（建议公证）
   - 申请法院向银行调取（需调查令）

2. **支付宝交易记录**
   - 登录支付宝网页版，导出账单
   - 申请支付宝电子凭证
   - 公证保全交易记录页面

3. **微信支付交易记录**
   - 通过微信钱包导出账单
   - 申请微信支付电子凭证
   - 公证保全交易记录

#### 注意事项

| 注意要点 | 具体要求 |
|---------|---------|
| 完整性 | 交易记录应包含完整流水号、时间、金额、对方账户信息 |
| 身份关联 | 确认交易账户与当事人的关联性 |
| 原始凭证 | 保存原始电子凭证，不得篡改 |
| 对方信息 | 通过申请电子凭证获取对方实名信息 |

---

### 3.4 网页信息

#### 保全方法

1. **公证保全（推荐）**
   - 公证员操作浏览器访问目标网页
   - 对网页内容进行截屏并打印
   - 记录完整的URL和访问时间
   - 必要时对网页操作过程进行录像

2. **可信时间戳保全**
   - 使用可信时间戳服务中心进行网页保全
   - 获取时间戳认证证书
   - 保存网页截图和认证证书

3. **区块链存证**
   - 使用区块链存证平台固定网页内容
   - 获取存证哈希值和存证证书

#### 注意事项

| 注意要点 | 具体要求 |
|---------|---------|
| 时效性 | 网页内容可随时修改，应尽早保全 |
| URL完整 | 记录完整的URL地址，包括参数 |
| 访问时间 | 精确记录访问时间（精确到分钟） |
| 多页保全 | 多页内容应逐页保全 |
| 动态内容 | 对可能变化的动态内容（如评论区）应特别说明 |

---

### 3.5 电子文件（文档/图片/视频/音频等）

#### 保全方法

1. **原始存储介质保全**
   - 保存原始存储介质（硬盘、U盘、SD卡等）
   - 对存储介质进行封存
   - 记录封存时间、地点、见证人

2. **文件提取保全**
   - 计算文件哈希值（MD5/SHA256）
   - 将文件复制到专用存储介质
   - 再次计算复制后文件的哈希值进行比对
   - 记录提取过程和哈希值

3. **公证保全**
   - 公证员见证文件提取过程
   - 计算并记录文件哈希值
   - 将文件刻录光盘或存入U盘封存

#### 注意事项

| 注意要点 | 具体要求 |
|---------|---------|
| 文件属性 | 记录文件创建时间、修改时间、访问时间、文件大小 |
| 原始载体 | 不得在原始载体上修改、删除文件 |
| 哈希值 | 计算并记录文件哈希值，作为完整性校验依据 |
| 病毒查杀 | 提取前进行病毒查杀，避免污染 |
| 格式兼容 | 保存文件时应注意格式兼容性 |

## 四、电子数据提交规范

### 4.1 提交形式

| 电子数据类型 | 提交形式 | 说明 |
|-------------|---------|------|
| 即时通信记录 | 打印件+原始载体 | 打印关键对话页面，携带原始手机备查 |
| 电子邮件 | 打印件+电子文件 | 打印邮件内容，同时提交.eml或.msg文件 |
| 电子交易记录 | 打印件+电子文件 | 银行流水需加盖银行印章 |
| 网页信息 | 打印件+公证书 | 提交公证保全的网页截图 |
| 电子文件 | 存储介质+哈希值 | 提交刻录光盘或U盘，附哈希值记录 |

### 4.2 提交清单

提交电子数据证据时，应附以下材料：

1. **证据清单**：列明电子数据名称、类型、页数/时长、证明对象
2. **来源说明**：说明电子数据的来源、提取时间、提取方式
3. **完整性证明**：提供哈希值校验记录或公证书
4. **原始载体**：携带原始存储介质备查（如手机、硬盘）

## 五、电子数据真实性审查要点

### 5.1 形式审查

- [ ] 电子数据是否有原始载体
- [ ] 电子数据是否有完整性校验记录
- [ ] 电子数据的提取过程是否有记录
- [ ] 电子数据是否经过公证或其他可信方式保全

### 5.2 内容审查

- [ ] 电子数据内容是否完整，有无剪辑、删改痕迹
- [ ] 电子数据的时间、地点、人物是否与案件事实相符
- [ ] 电子数据与其他证据是否能够相互印证
- [ ] 电子数据的生成、存储、传输环境是否可靠

### 5.3 质疑与反驳

对对方提交的电子数据提出质疑时，可从以下角度入手：

1. **真实性质疑**：主张电子数据经过篡改、伪造
2. **关联性质疑**：主张电子数据与案件事实无关
3. **合法性质疑**：主张电子数据通过非法手段获取
4. **完整性质疑**：主张电子数据不完整，经过选择性截取

---

## 六、法律依据汇编

1. 《中华人民共和国民事诉讼法》第六十六条第二款（电子数据作为独立证据种类）
2. 《最高人民法院关于民事诉讼证据的若干规定》第十四条至第二十三条（电子数据范围、审查判断规则）
3. 《最高人民法院关于互联网法院审理案件若干问题的规定》第十一条（电子数据真实性认定）
4. 《人民法院在线诉讼规则》第十六条至第二十条（电子数据提交、质证规则）
